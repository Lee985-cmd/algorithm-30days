# 🎯 归并排序的分治思想 - 超详解

## 一、什么是分治法？

### 1.1 生活中的分治

**场景：整理 1000 张扑克牌**

```
问题：把 1000 张牌按数字从小到大排好

笨方法：
- 一张一张比较
- 需要比较约 50 万次
- 累死也排不完 😫

聪明方法（分治）：
1. 分成 2 堆，每堆 500 张
2. 每堆再分成 2 堆，每堆 250 张
3. 继续分...直到每堆只有 1 张
4. 然后两两合并（这时候很简单）
5. 最后得到有序的 1000 张牌

这就是分治！
```

### 1.2 分治三步曲

```
第 1 步：分解（Divide）
       把大问题拆成小问题
       
第 2 步：解决（Conquer）
       小问题递归求解
       （小到一定程度直接解决）
       
第 3 步：合并（Combine）
       把小问题的解合并成大问题的解
```

**形象理解：**

```
                    大问题
                      ↓
              ┌───────┴───────┐
              ↓               ↓
           中问题          中问题
              ↓               ↓
         ┌────┴────┐     ┌────┴────┐
         ↓    ↓    ↓     ↓    ↓    ↓
       小小  小小  小小  小小  小小  小小
       问题  问题  问题  问题  问题  问题
       
       解决！解决！解决！解决！解决！解决！
         ↓    ↓    ↓     ↓    ↓    ↓
         └────┴────┘     └────┴────┘
              ↓               ↓
           合并！            合并！
              ↓               ↓
              └───────┬───────┘
                      ↓
                   合并！
                      ↓
                  大问题解决！
```

## 二、归并排序的分治实现

### 2.1 问题描述

```
输入：[5, 2, 8, 1, 9, 3, 7, 4]
输出：[1, 2, 3, 4, 5, 7, 8, 9]

要求：用分治法排序
```

### 2.2 分解过程（Divide）

```
原始数组：[5, 2, 8, 1, 9, 3, 7, 4]

第 1 次分解：
┌───────────────┴───────────────┐
[5, 2, 8, 1]                [9, 3, 7, 4]

第 2 次分解：
┌─────┴─────┐             ┌─────┴─────┐
[5, 2]    [8, 1]        [9, 3]    [7, 4]

第 3 次分解：
┌──┴──┐   ┌──┴──┐      ┌──┴──┐   ┌──┴──┐
[5] [2]  [8] [1]      [9] [3]  [7] [4]

现在每堆只有 1 个元素
已经有序了！（单个元素天然有序）

分解结束！✓
```

**代码实现分解：**

```javascript
function mergeSort(arr) {
    // 分解的终止条件
    if (arr.length <= 1) {
        return arr;  // 只有一个元素，已经有序
    }
    
    // 分解步骤
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);      // 左半部分
    const right = arr.slice(mid);        // 右半部分
    
    // 递归分解
    const sortedLeft = mergeSort(left);
    const sortedRight = mergeSort(right);
    
    // 合并（下一步）
    return merge(sortedLeft, sortedRight);
}
```

### 2.3 合并过程（Combine）⭐难点

**问题：如何合并两个有序数组？**

```
left = [2, 5]
right = [1, 8]

目标：合并成 [1, 2, 5, 8]

怎么做？
```

**双指针法：**

```javascript
function merge(left, right) {
    const result = [];
    let i = 0;  // 指向 left
    let j = 0;  // 指向 right
    
    // 两个数组都有元素时
    while (i < left.length && j < right.length) {
        // 谁小要谁
        if (left[i] <= right[j]) {
            result.push(left[i]);
            i++;
        } else {
            result.push(right[j]);
            j++;
        }
    }
    
    // 把剩下的放进去
    while (i < left.length) {
        result.push(left[i]);
        i++;
    }
    
    while (j < right.length) {
        result.push(right[j]);
        j++;
    }
    
    return result;
}
```

## 三、为什么归并排序是分治的完美例子？

### 3.1 完美符合分治三要素

```
✓ 分解：从中间切，分成两个子数组
✓ 解决：递归排序子数组
✓ 合并：合并两个有序数组

每一步都清晰明了！
```

### 3.2 分解是平衡的

```
每次都对半分：
[8 个元素] → [4 个] + [4 个]
[4 个元素] → [2 个] + [2 个]
[2 个元素] → [1 个] + [1 个]

这样保证树的高度是 log n
效率最高！
```

**对比不平衡的分治：**

```
❌ 糟糕的分法：
[8 个元素] → [1 个] + [7 个]
[7 个元素] → [1 个] + [6 个]
...

这样退化成了 O(n²)
和冒泡排序一样慢！
```

### 3.3 子问题是独立的

```
排序左半部分 和 排序右半部分
互不干扰

可以并行处理！
（实际工程中很有用）
```

## 四、时间复杂度分析

### 4.1 画递归树

```
                    [8 个元素]        ← 第 0 层
                   /          \
              [4 个]          [4 个]    ← 第 1 层
              /    \          /    \
           [2]     [2]      [2]     [2]  ← 第 2 层
           / \     / \      / \     / \
         [1][1]  [1][1]   [1][1]  [1][1] ← 第 3 层

树的高度：log₂8 = 3
```

### 4.2 计算每层的工作量

```
第 0 层：分解 1 次，合并 8 个元素 → 工作量 8
第 1 层：分解 2 次，合并 8 个元素 → 工作量 8
第 2 层：分解 4 次，合并 8 个元素 → 工作量 8
第 3 层：分解 8 次，不需要合并   → 工作量 8

总工作量：8 + 8 + 8 + 8 = 32

推广到 n 个元素：
- 树高：log₂n
- 每层工作量：n
- 总工作量：n × log n

时间复杂度：O(n log n) ✓
```

### 4.3 严格推导（可选）

```
设 T(n) 是排序 n 个元素的时间

分解：切成两半 → O(1)
解决：递归排序 → 2T(n/2)
合并：合并两个数组 → O(n)

递归式：
T(n) = 2T(n/2) + O(n)

根据主定理：
a = 2, b = 2, f(n) = O(n)
log_b(a) = log₂2 = 1

f(n) = O(n) = Θ(n^log_b(a)) = Θ(n¹)

属于主定理情况 2：
T(n) = Θ(n^log_b(a) × log n)
     = Θ(n × log n)
     = O(n log n)

证毕！✓
```

## 五、空间复杂度

### 5.1 为什么需要 O(n) 额外空间？

```
因为合并时需要临时数组！

merge([2,5], [1,8])
需要一个 result 数组来存结果

不能原地合并吗？
→ 很难，会很复杂
→ 所以归并排序不是原地排序
```

### 5.2 空间复杂度分析

```
每一层合并都需要临时数组：
第 0 层：需要 n 的空间
第 1 层：需要 n 的空间（但可以复用）
...

实际上只需要一份 n 的空间

空间复杂度：O(n)
```

## 六、归并排序的优缺点

### 优点 ✅

```
✓ 时间复杂度稳定 O(n log n)
  最好、最坏、平均都是 O(n log n)
  
✓ 稳定排序
  相同的元素，相对顺序不变
  
✓ 适合链表
  可以在 O(1) 额外空间内完成
  
✓ 可以并行化
  左右两边可以同时排序
```

### 缺点 ❌

```
❌ 需要 O(n) 额外空间
  不是原地排序
  
❌ 对小数组不一定最优
  递归有开销
  
❌ 实现相对复杂
  比插入排序难写
```

## 八、完整代码（背诵版）

```javascript
/**
 * 归并排序 - 分治法的经典应用
 * 时间复杂度：O(n log n)
 * 空间复杂度：O(n)
 */
function mergeSort(arr) {
    // 终止条件
    if (arr.length <= 1) {
        return arr;
    }
    
    // 分解
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);
    const right = arr.slice(mid);
    
    // 递归解决 + 合并
    return merge(mergeSort(left), mergeSort(right));
}

/**
 * 合并两个有序数组
 */
function merge(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    // 双指针比较
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i]);
            i++;
        } else {
            result.push(right[j]);
            j++;
        }
    }
    
    // 处理剩余元素
    return [...result, ...left.slice(i), ...right.slice(j)];
}

// 测试
console.log(mergeSort([5, 2, 8, 1, 9]));  
// [1, 2, 5, 8, 9]

console.log(mergeSort([3, 1, 4, 1, 5, 9, 2, 6]));  
// [1, 1, 2, 3, 4, 5, 6, 9]
```

## 十、总结

### 分治思想的核心

```
1. 大问题拆成小问题
2. 小问题独立解决
3. 小问题的解合并成大问题的解

关键：
✓ 分解要平衡
✓ 子问题要独立
✓ 合并要高效
```

### 归并排序的要点

```
✓ 分解：从中间切
✓ 解决：递归排序
✓ 合并：双指针法

时间复杂度：O(n log n)
空间复杂度：O(n)
稳定性：稳定
```
