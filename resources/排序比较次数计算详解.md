# 🧮 排序比较次数是怎么算出来的？

## 一、问题的由来

### 场景回顾

```
问题：把 1000 张牌从小到大排好

笨方法（一张一张比较）：
- 第 1 张和后面 999 张比较
- 第 2 张和后面 998 张比较
- 第 3 张和后面 997 张比较
- ...
- 第 999 张和最后 1 张比较

总比较次数 = ?
```

---

## 二、详细推导过程

### 方法 1：逐个分析

```
假设有 n = 1000 张牌

第 1 轮：
拿第 1 张牌，和后面所有牌比较
需要比较：999 次

第 2 轮：
拿第 2 张牌，和后面所有牌比较
需要比较：998 次

第 3 轮：
拿第 3 张牌，和后面所有牌比较
需要比较：997 次

...

第 i 轮：
拿第 i 张牌，和后面所有牌比较
需要比较：n - i 次

...

第 998 轮：
拿第 998 张牌，和后面比较
需要比较：2 次

第 999 轮：
拿第 999 张牌，和最后 1 张比较
需要比较：1 次

第 1000 轮：
只剩 1 张，不用比较了
需要比较：0 次
```

### 总次数计算

```
总次数 = 999 + 998 + 997 + ... + 2 + 1 + 0

这是一个等差数列！

首项 a₁ = 999
末项 aₙ = 0
项数 n = 999
```

---

## 三、等差数列求和公式

### 公式推导

```
等差数列求和公式：
S = n × (a₁ + aₙ) / 2

其中：
- n 是项数
- a₁ 是首项
- aₙ 是末项

代入我们的数据：
n = 999
a₁ = 999
aₙ = 0

S = 999 × (999 + 0) / 2
  = 999 × 999 / 2
  = 998001 / 2
  = 499000.5
  ≈ 499,000 次
  ≈ 50 万次 ✓
```

### 为什么是这个公式？

**高斯的巧妙方法：**

```
传说高斯小时候老师出了这道题：
1 + 2 + 3 + ... + 99 + 100 = ?

高斯发现：
(1 + 100) + (2 + 99) + (3 + 98) + ... 
= 101 + 101 + 101 + ... 
= 101 × 50
= 5050

规律：
首尾配对，每对的和都相等！

应用到我们的问题：
999 + 0 = 999
998 + 1 = 999
997 + 2 = 999
...

共有 999/2 = 499.5 对
每对的和 = 999

总和 = 999 × 499.5 = 499,000.5 ≈ 50 万 ✓
```

## 四、一般化公式

### n 个元素的比较次数

```
对于 n 个元素：

总次数 = (n-1) + (n-2) + ... + 2 + 1
       = n × (n-1) / 2

验证：
n = 1000 时
总次数 = 1000 × 999 / 2
       = 999000 / 2
       = 499,500 次 ✓
```

### 为什么是 O(n²)？

```
总次数 = n × (n-1) / 2
       = (n² - n) / 2
       = n²/2 - n/2

当 n 很大时：
- n²/2 占主导
- n/2 可以忽略

所以时间复杂度 = O(n²)

这就是为什么冒泡排序、选择排序、插入排序
都是 O(n²) 的原因！
```

## 五、直观理解

### 画图法

```
画一个 n×n 的方格：

┌────────────────────────┐
│  ×  │  √  │  √  │  √  │  ← 第 1 行：比较 n-1 次
├─────┼─────┼─────┼─────┤
│     │  ×  │  √  │  √  │  ← 第 2 行：比较 n-2 次
├─────┼─────┼─────┼─────┤
│     │     │  ×  │  √  │  ← 第 3 行：比较 n-3 次
├─────┼─────┼─────┼─────┤
│     │     │     │  ×  │  ← 第 n 行：比较 0 次
└────────────────────────┘

√ 表示需要比较的位置
× 表示自己和自己，不用比

数一下 √ 的个数：
第 1 行：n-1 个
第 2 行：n-2 个
...
第 n-1 行：1 个
第 n 行：0 个

总和 = (n-1) + (n-2) + ... + 1 + 0
     = n(n-1)/2
```

### 三角形面积法

```
把比较次数画成三角形：

    *
   **
  ***
 ****
*****

底 = n-1
高 = n-1

三角形面积 = 底 × 高 / 2
          = (n-1) × (n-1) / 2
          ≈ n²/2

所以是 O(n²)！
```

## 六、实际计算示例

### 小规模验证

**n = 5 的情况：**

```
数组：[5, 2, 8, 1, 9]

用冒泡排序（一张一张比较）：

第 1 轮：
5 vs 2 ✗ 交换 → [2, 5, 8, 1, 9]
5 vs 8 ✓
8 vs 1 ✗ 交换 → [2, 5, 1, 8, 9]
8 vs 9 ✓
比较次数：4 次

第 2 轮：
2 vs 5 ✓
5 vs 1 ✗ 交换 → [2, 1, 5, 8, 9]
5 vs 8 ✓
比较次数：3 次

第 3 轮：
2 vs 1 ✗ 交换 → [1, 2, 5, 8, 9]
2 vs 5 ✓
比较次数：2 次

第 4 轮：
1 vs 2 ✓
比较次数：1 次

总次数：4+3+2+1 = 10 次

用公式验证：
n(n-1)/2 = 5×4/2 = 10 ✓
```

### 中等规模

**n = 100 的情况：**

```
总次数 = 100 × 99 / 2
       = 9900 / 2
       = 4950 次

约 5000 次！
```

### 大规模

**n = 1000 的情况：**

```
总次数 = 1000 × 999 / 2
       = 999000 / 2
       = 499,500 次

约 50 万次！✓
```

**n = 10000 的情况：**

```
总次数 = 10000 × 9999 / 2
       = 99990000 / 2
       = 49,995,000 次

约 5000 万次！

看到了吗？
n 增加 10 倍
比较次数增加 100 倍！

这就是 O(n²) 的可怕之处！
```

## 七、对比分治法

### 归并排序的比较次数

```
归并排序：O(n log n)

n = 1000 时：
比较次数 ≈ 1000 × log₂1000
        ≈ 1000 × 10
        ≈ 10,000 次

而笨方法：≈ 500,000 次

差距：500,000 / 10,000 = 50 倍！

归并排序快了 50 倍！🚀
```

### 更直观的对比

```
n = 10000 时：

笨方法（O(n²)）：
5000 万次
假设每秒 100 万次
需要 50 秒

归并排序（O(n log n)）：
约 13 万次
只需要 0.13 秒

差距巨大！
```

## 八、为什么是"约"50 万次？

### 精确计算

```
精确值：499,500 次

说"约 50 万次"是因为：
1. 方便记忆
2. 数量级正确
3. 不需要太精确

就像说"中国有 14 亿人口"
实际上是 14.12 亿左右
但说"约 14 亿"就够了
```

### 不同情况

```
最好情况（已经有序）：
有些优化算法只需要 n-1 次比较
但还是 O(n²)

最坏情况（完全倒序）：
就是 n(n-1)/2 次

平均情况：
也是约 n²/2 次

所以都说"约 n²/2 次"
```

## 九、动手练习

### 练习 1：手动计算

```
n = 10 时：
总次数 = ?

答案：
10 × 9 / 2 = 45 次
```

### 练习 2：找规律

```
n = 2:  2×1/2 = 1 次
n = 3:  3×2/2 = 3 次
n = 4:  4×3/2 = 6 次
n = 5:  5×4/2 = 10 次
n = 6:  ?

答案：6×5/2 = 15 次
```

### 练习 3：反向计算

```
如果比较了 55 次
那么有多少个元素？

解方程：
n(n-1)/2 = 55
n² - n = 110
n² - n - 110 = 0

用求根公式：
n = (1 + √(1+440)) / 2
n = (1 + 21) / 2
n = 11

验证：11×10/2 = 55 ✓
```

## 十、总结

### 核心公式

```
n 个元素的比较次数：
总次数 = n × (n-1) / 2

时间复杂度：O(n²)
```

### 推导过程

```
1. 第 1 个和后面 n-1 个比较
2. 第 2 个和后面 n-2 个比较
3. ...
4. 总次数 = (n-1) + (n-2) + ... + 1
5. 等差数列求和 = n(n-1)/2
```

### 直观理解

```
- 三角形面积：底×高/2
- 方格图：数格子
- 高斯配对：首尾相加

都得到相同的结果！
```

### 实际应用

```
✓ 理解为什么 O(n²) 慢
✓ 知道为什么要用 O(n log n)
✓ 明白算法优化的重要性

n=1000 时：
O(n²) = 50 万次
O(n log n) = 1 万次

差了 50 倍！
```
