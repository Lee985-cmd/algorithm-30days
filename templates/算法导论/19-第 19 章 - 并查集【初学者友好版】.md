# 📚 算法导论 Part V - 第 19 章：朋友圈管理大师 - 并查集

## 19.1 什么是并查集？为什么这么厉害？

### 生活中的例子

**场景 1：微信好友分组**

```
你有好多好友：张三、李四、王五...

想把他们分成不同的圈子：
- 大学同学圈：张三、李四、赵六
- 高中同学圈：王五、钱七、孙八
- 工作同事圈：周九、吴十

操作：
1️⃣ 查找：张三在哪个圈？→ 大学同学圈
2️⃣ 合并：大学同学圈和工作同事圈合并了！
   → 现在是一个大圈

这就是并查集的思想！🎯
```

**场景 2：帮派管理**

```
江湖上有好多帮派：
- 少林派（掌门：方丈）
- 武当派（掌门：道长）
- 峨眉派（掌门：师太）

每个弟子都有一个"老大"
老大的老大还是老大
直到掌门

操作：
1️⃣ 查找：小和尚的掌门是谁？→ 方丈
2️⃣ 合并：少林和武当合并了！
   → 方丈成为整个大帮派的掌门

这也是并查集！✨
```

### 说人话定义

```
并查集 = 管理不相交集合的数据结构

支持两个操作：

1️⃣ Find（查找）
   问：你在哪个集合里？
   或者：咱俩在不在一个集合里？
   
2️⃣ Union（合并）
   把两个集合合并成一个

就这么简单！
但非常有用！💡
```

## 19.2 基础实现 - 用树来表示

### 核心思想

```
用树来表示集合：
- 每棵树代表一个集合
- 树的根节点代表集合的代表元
- 每个节点指向它的父节点
- 根节点的父节点是自己

例如：
集合 1：大学同学
    张三 (根)
    / | \
  李四 王五 赵六

集合 2：工作同事
    周九 (根)
    / \
  吴十 郑十一

查找：李四的老大是谁？
→ 一直往上找：李四 → 张三
→ 张三是根，所以老大是张三

合并：把两个集合合并
→ 让一个根指向另一个根
→ 比如：周九的父亲 = 张三
→ 现在张三是整个大集合的老大！
```

### 代码实现（基础版）

```javascript
/**
 * 并查集 - 基础版本
 */
class UnionFind {
    constructor(n) {
        // parent[i] 表示 i 的父节点
        this.parent = new Array(n);
        
        // 初始化：每个节点都是自己的父节点
        for (let i = 0; i < n; i++) {
            this.parent[i] = i;
        }
    }
    
    /**
     * Find - 查找元素所在的集合（根节点）
     */
    find(x) {
        // 如果自己是根，返回自己
        if (this.parent[x] === x) {
            return x;
        }
        
        // 否则，继续往上找
        return this.find(this.parent[x]);
    }
    
    /**
     * Union - 合并两个集合
     */
    union(x, y) {
        // 找到两个元素的根
        const rootX = this.find(x);
        const rootY = this.find(y);
        
        // 如果已经在同一个集合，不需要合并
        if (rootX === rootY) {
            return;
        }
        
        // 否则，让一个根指向另一个根
        this.parent[rootX] = rootY;
    }
    
    /**
     * connected - 判断两个元素是否在同一个集合
     */
    connected(x, y) {
        return this.find(x) === this.find(y);
    }
}

// 测试
const uf = new UnionFind(10);  // 10 个人，编号 0-9

// 合并一些关系
uf.union(0, 1);  // 0 和 1 是一个圈的
uf.union(1, 2);  // 1 和 2 是一个圈的
// 现在 0, 1, 2 都在一个圈里了！

console.log(uf.connected(0, 2));  // true ✅ 在一个圈里
console.log(uf.connected(0, 3));  // false ❌ 不在一个圈里
```

### 详细执行过程

```
初始化 5 个人（0-4）：
parent = [0, 1, 2, 3, 4]
         ↑  ↑  ↑  ↑  ↑
索引：   0  1  2  3  4

每个人都是自己的老大

═══════════════════════════════════
union(0, 1)
═══════════════════════════════════
find(0) = 0
find(1) = 1

让 0 的老大变成 1
parent[0] = 1

现在：
parent = [1, 1, 2, 3, 4]

树结构：
1 (根)
↑
0

═══════════════════════════════════
union(1, 2)
═══════════════════════════════════
find(1) = 1
find(2) = 2

让 1 的老大变成 2
parent[1] = 2

现在：
parent = [1, 2, 2, 3, 4]

树结构：
2 (根)
↑
1
↑
0

═══════════════════════════════════
find(0)
═══════════════════════════════════
从 0 开始往上找：
0 → parent[0]=1 → parent[1]=2 → parent[2]=2（根）

返回：2

所以 0 的老大是 2！
0, 1, 2 都在一个集合里！✅
```

## 19.3 优化 1：路径压缩

### 问题

```
看刚才的树：
2 (根)
↑
1
↑
0

如果要找 0 的老大：
0 → 1 → 2
要走 3 步！

如果有更多节点：
10 → 9 → 8 → ... → 2 → 1
要走 10 步！

太慢了！❌

能不能优化？
→ 可以！路径压缩！
```

### 核心思想

```
路径压缩：
在查找的过程中
把路上的所有节点都直接连到根上

就像：
第一次找 0 的老大：
0 → 1 → 2

找到后，告诉 0 和 1：
"你们的老大都是 2！别绕弯子了！"

下次再找：
0 → 2（直接到！）
1 → 2（直接到！）

快多了！✅
```

### 代码实现（路径压缩）

```javascript
class UnionFindOptimized {
    constructor(n) {
        this.parent = new Array(n);
        for (let i = 0; i < n; i++) {
            this.parent[i] = i;
        }
    }
    
    /**
     * Find - 带路径压缩
     */
    find(x) {
        if (this.parent[x] === x) {
            return x;
        }
        
        // 递归查找根节点
        const root = this.find(this.parent[x]);
        
        // 路径压缩：直接指向根
        this.parent[x] = root;
        
        return root;
    }
    
    union(x, y) {
        const rootX = this.find(x);
        const rootY = this.find(y);
        
        if (rootX !== rootY) {
            this.parent[rootX] = rootY;
        }
    }
    
    connected(x, y) {
        return this.find(x) === this.find(y);
    }
}

// 测试
const uf = new UnionFindOptimized(5);

uf.union(0, 1);
uf.union(1, 2);

console.log(uf.find(0));  
// 第 1 次调用：0 → 1 → 2，返回 2
// 同时路径压缩：parent[0]=2, parent[1]=2

console.log(uf.find(0));  
// 第 2 次调用：0 → 2，直接返回 2！
// 快多了！✅
```

## 19.4 优化 2：按秩合并

### 问题

```
合并两个集合时：
随便让一个根指向另一个根

可能的问题：
如果把小树挂到大树上
树会很高！

例如：
大树：1000 个节点，高度 10
小树：1 个节点

合并：让小树的根指向大树的根
→ 高度变成 11

反过来：
让大树的根指向小树的根
→ 高度变成 1001！❌

怎么避免？
→ 按秩合并！
```

### 核心思想

```
秩（rank）：树的高度的近似值

合并时：
总是把矮的树挂到高的树上

这样树的高度增长最慢！

规则：
1️⃣ 如果 rank[x] < rank[y]
   → x 挂到 y 上，rank[y] 不变
   
2️⃣ 如果 rank[x] > rank[y]
   → y 挂到 x 上，rank[x] 不变
   
3️⃣ 如果 rank[x] == rank[y]
   → 随便挂，被挂的那个 rank+1
```

### 完整优化版代码

```javascript
/**
 * 并查集 - 完全优化版（路径压缩 + 按秩合并）
 */
class UnionFindBest {
    constructor(n) {
        this.parent = new Array(n);
        // rank[i] 表示以 i 为根的树的高度（近似值）
        this.rank = new Array(n).fill(0);
        
        for (let i = 0; i < n; i++) {
            this.parent[i] = i;
        }
    }
    
    find(x) {
        if (this.parent[x] === x) {
            return x;
        }
        
        // 路径压缩
        const root = this.find(this.parent[x]);
        this.parent[x] = root;
        
        return root;
    }
    
    /**
     * Union - 按秩合并
     */
    union(x, y) {
        const rootX = this.find(x);
        const rootY = this.find(y);
        
        if (rootX === rootY) return;
        
        // 按秩合并：矮的挂到高的上
        if (this.rank[rootX] < this.rank[rootY]) {
            // x 的树更矮，挂到 y 上
            this.parent[rootX] = rootY;
        } else if (this.rank[rootX] > this.rank[rootY]) {
            // y 的树更矮，挂到 x 上
            this.parent[rootY] = rootX;
        } else {
            // 一样高，随便挂，被挂的秩 +1
            this.parent[rootY] = rootX;
            this.rank[rootX]++;
        }
    }
    
    connected(x, y) {
        return this.find(x) === this.find(y);
    }
}

// 测试
const uf = new UnionFindBest(10);

uf.union(0, 1);
uf.union(2, 3);
uf.union(4, 5);

uf.union(0, 2);  // 合并 {0,1} 和 {2,3}
uf.union(0, 4);  // 合并 {0,1,2,3} 和 {4,5}

// 现在 {0,1,2,3,4,5} 都在一个集合里了！

console.log(uf.connected(1, 5));  // true ✅
```

## 19.5 时间复杂度分析（震惊！）

```
两种优化都用上后：

时间复杂度：O(α(n))

其中 α 是反阿克曼函数

这是什么概念？

α(n) ≤ 4，对于所有实际的 n！

也就是说：
不管 n 有多大（就算是宇宙中的原子数）
操作次数都不会超过 4 次！

这跟 O(1) 几乎一样！

比哈希表还快！🚀

空间复杂度：O(n)
```

## 19.6 实际应用

### 应用 1：判断图的连通性

```javascript
/**
 * 判断无向图是否连通
 */
function isGraphConnected(n, edges) {
    const uf = new UnionFindBest(n);
    
    // 把所有边连接起来
    for (let [u, v] of edges) {
        uf.union(u, v);
    }
    
    // 检查是否所有点都在一个集合里
    const root = uf.find(0);
    for (let i = 1; i < n; i++) {
        if (uf.find(i) !== root) {
            return false;
        }
    }
    
    return true;
}

// 测试
const edges1 = [[0,1], [1,2], [2,3]];
console.log(isGraphConnected(4, edges1));  // true ✅

const edges2 = [[0,1], [2,3]];
console.log(isGraphConnected(4, edges2));  // false ❌
```

### 应用 2：朋友圈问题

```javascript
/**
 * LeetCode 547. 朋友圈数量
 */
function findCircleNum(M) {
    const n = M.length;
    const uf = new UnionFindBest(n);
    
    // 遍历朋友关系
    for (let i = 0; i < n; i++) {
        for (let j = i + 1; j < n; j++) {
            if (M[i][j] === 1) {
                uf.union(i, j);
            }
        }
    }
    
    // 数有多少个根节点
    let count = 0;
    for (let i = 0; i < n; i++) {
        if (uf.parent[i] === i) {
            count++;
        }
    }
    
    return count;
}

// 测试
const M1 = [
    [1,1,0],
    [1,1,0],
    [0,0,1]
];
console.log(findCircleNum(M1));  // 2 ✅ (0,1 是朋友，2 单独)
```
