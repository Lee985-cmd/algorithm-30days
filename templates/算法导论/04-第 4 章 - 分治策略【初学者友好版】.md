## 👀 分治法的三个经典例子

### 例子 1：归并排序（复习）

```
问题：排序 [38, 27, 43, 3, 9, 82, 10]

分治三步：

1. 分解：从中间切开
   [38, 27, 43, 3, 9, 82, 10]
         ↓
   [38, 27, 43, 3]  [9, 82, 10]
   
2. 解决：递归排序两边
   [3, 27, 38, 43]  [9, 10, 82]
   
3. 合并：合并两个有序数组
   [3, 9, 10, 27, 38, 43, 82] ✓

时间复杂度：O(n log n)
```

### 例子 2：快速排序

```
问题：排序 [10, 80, 30, 90, 40, 50, 70]

分治三步：

1. 分解：选 pivot=70，分成两半
   小于 70: [10, 30, 40, 50]
   大于 70: [80, 90]
   
2. 解决：递归排序两边
   [10, 30, 40, 50]  [80, 90]
   
3. 合并：... 不需要！已经有序了
   [10, 30, 40, 50, 70, 80, 90] ✓

时间复杂度：平均 O(n log n)
```

### 例子 3：最大子数组问题（重点！）

**问题描述：**

```
你是一家公司的分析师，手头有连续 n 天的股价变化：

第 1 天：+10 元
第 2 天：-5 元
第 3 天：+20 元
第 4 天：-15 元
第 5 天：+30 元

问：在哪几天买入和卖出能获得最大收益？

注意：必须先买后卖，不能做空！
```

**暴力解法（O(n²)）：**

```javascript
// 试试所有可能的买卖组合
function maxProfitBruteForce(prices) {
    let maxProfit = -Infinity;
    
    // 枚举所有买入日期 i
    for (let i = 0; i < prices.length; i++) {
        // 枚举所有卖出日期 j (j > i)
        for (let j = i + 1; j < prices.length; j++) {
            const profit = prices[j] - prices[i];
            maxProfit = Math.max(maxProfit, profit);
        }
    }
    
    return maxProfit;
}

// 太慢了！n 天要试 n×(n-1)/2 次
// n=1000 时要试 50 万次！❌
```

**分治解法（O(n log n)）：**

```javascript
/**
 * 分治法求最大子数组和
 * 
 * 关键洞察：
 * 最大子数组要么在：
 * 1. 左半边
 * 2. 右半边
 * 3. 跨越中点
 */
function maxSubArray(nums) {
    return maxSubArrayHelper(nums, 0, nums.length - 1);
}

function maxSubArrayHelper(nums, left, right) {
    // 基本情况：只有一个元素
    if (left === right) {
        return nums[left];
    }
    
    // 1. 分解：找中点
    const mid = Math.floor((left + right) / 2);
    
    // 2. 解决：递归求左右两边的最大子数组
    const leftMax = maxSubArrayHelper(nums, left, mid);
    const rightMax = maxSubArrayHelper(nums, mid + 1, right);
    
    // 3. 合并：求跨越中点的最大子数组
    const crossMax = maxCrossingSum(nums, left, mid, right);
    
    // 返回三者中的最大值
    return Math.max(leftMax, rightMax, crossMax);
}

/**
 * 求跨越中点的最大子数组和
 */
function maxCrossingSum(nums, left, mid, right) {
    // 从中点向左的最大和
    let leftSum = -Infinity;
    let sum = 0;
    for (let i = mid; i >= left; i--) {
        sum += nums[i];
        if (sum > leftSum) leftSum = sum;
    }
    
    // 从中点向右的最大和
    let rightSum = -Infinity;
    sum = 0;
    for (let i = mid + 1; i <= right; i++) {
        sum += nums[i];
        if (sum > rightSum) rightSum = sum;
    }
    
    // 合并
    return leftSum + rightSum;
}

// 测试
const changes = [10, -5, 20, -15, 30];
console.log('最大收益:', maxSubArray(changes));
// 输出：40（从第 1 天持有到第 5 天）✓

// 复杂度：O(n log n) 比 O(n²) 快多了！🚀
```

## ⏱️ 主定理 - 复杂度分析神器

### 什么是主定理？

```
主定理就像一个公式：
只要你的递归长这样：
T(n) = a·T(n/b) + f(n)

就能直接套公式算出复杂度！

其中：
a = 子问题个数
b = 缩小的倍数
f(n) = 分解和合并的代价
```

### 三种情况（背下来！）

```
情况 1：f(n) 很小
如果 f(n) = O(n^d)，其中 d < log_b(a)
那么 T(n) = Θ(n^(log_b(a)))

例子：归并排序
T(n) = 2T(n/2) + O(n)
a=2, b=2, d=1
log_2(2) = 1 = d
所以 T(n) = Θ(n log n)

情况 2：f(n) 刚刚好
如果 f(n) = Θ(n^(log_b(a)))
那么 T(n) = Θ(n^(log_b(a)) · log n)

例子：还是归并排序
a=2, b=2, log_2(2)=1
f(n)=O(n)=O(n^1)
所以 T(n) = Θ(n log n)

情况 3：f(n) 很大
如果 f(n) = Ω(n^d)，其中 d > log_b(a)
那么 T(n) = Θ(f(n))

例子：
T(n) = T(n/2) + O(n)
a=1, b=2, d=1
log_2(1)=0 < 1
所以 T(n) = Θ(n)
```

### 记忆技巧

```
把 T(n) 想象成一棵树：

情况 1：叶子最重
大部分工作在叶子节点
T(n) = Θ(叶子数) = Θ(n^(log_b(a)))

情况 2：每层一样重
每层工作量差不多
T(n) = Θ(层数 × 每层工作) = Θ(n^(log_b(a)) · log n)

情况 3：根最重
大部分工作在根节点
T(n) = Θ(根的工作) = Θ(f(n))
```
