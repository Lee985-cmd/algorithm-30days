## 👀 堆的数组表示

### 关键公式（背下来！）

```
对于索引为 i 的节点（从 0 开始）：

┌──────────────────────┐
│ 父节点：PARENT(i)    │
│ = ⌊(i - 1) / 2⌋     │
└──────────────────────┘

┌──────────────────────┐
│ 左孩子：LEFT(i)      │
│ = 2i + 1            │
└──────────────────────┘

┌──────────────────────┐
│ 右孩子：RIGHT(i)     │
│ = 2i + 2            │
└──────────────────────┘
```


## 🛠️ 核心操作：MAX-HEAPIFY（下沉）

### 功能说明

```
MAX-HEAPIFY 的作用：
假设某个节点变"小"了，让它"下沉"到合适的位置

就像：
班长突然长矮了，要往后调座位 😄
```

### 一步一步演示

```
修复以 index=1 为根的子树

初始状态：
[16, 4, 10, 8, 7, 9, 3, 2, 4, 1]
         ↑
      i=1, arr[1]=4（这个太小了，要下沉）

步骤 1: 比较 4, 8, 7
        左孩子 LEFT(1)=3, arr[3]=8
        右孩子 RIGHT(1)=4, arr[4]=7
        最大的是 8
        
        [16, 4, 10, 8, 7, 9, 3, 2, 4, 1]
              ↑        ↑
              i        largest=3
        
        交换 arr[1] 和 arr[3]
        [16, 8, 10, 4, 7, 9, 3, 2, 4, 1]
                       ↑
                       i=3

步骤 2: 比较 4, 2, 4
        左孩子 LEFT(3)=7, arr[7]=2
        右孩子 RIGHT(3)=8, arr[8]=4
        最大的是 4（右孩子）
        
        [16, 8, 10, 4, 7, 9, 3, 2, 4, 1]
                       ↑              ↑
                       i              largest=8
        
        交换 arr[3] 和 arr[8]
        [16, 8, 10, 4, 7, 9, 3, 2, 4, 1]
                                  ↑
                                  i=8

步骤 3: i=8 是叶子节点
        结束 ✓

最终：[16, 8, 10, 4, 7, 9, 3, 2, 4, 1]
```

### 代码实现

```javascript
/**
 * MAX-HEAPIFY - 让节点"下沉"到合适位置
 * 
 * @param {number[]} arr - 堆数组
 * @param {number} i - 当前节点索引
 * @param {number} heapSize - 堆大小
 */
function maxHeapify(arr, i, heapSize) {
    const left = 2 * i + 1;      // 左孩子
    const right = 2 * i + 2;     // 右孩子
    let largest = i;             // 假设当前节点最大
    
    // 如果左孩子更大
    if (left < heapSize && arr[left] > arr[largest]) {
        largest = left;
    }
    
    // 如果右孩子更大
    if (right < heapSize && arr[right] > arr[largest]) {
        largest = right;
    }
    
    // 如果最大值不是当前节点
    if (largest !== i) {
        // 交换
        [arr[i], arr[largest]] = [arr[largest], arr[i]];
        
        // 递归处理被交换的子树
        maxHeapify(arr, largest, heapSize);
    }
    // 否则，以 i 为根的子树已经是最大堆
}

// ⏱️ 时间复杂度：O(log n)
// 因为树的高度是 log n，最多下沉这么多次
```

### ❗ 常见错误

```javascript
// ❌ 错误 1：忘记检查边界
if (arr[left] > arr[largest]) {  // 漏了 left < heapSize
    // 可能数组越界！
}

// ❌ 错误 2：没有递归调用
if (largest !== i) {
    [arr[i], arr[largest]] = [arr[largest], arr[i]];
    // 忘了 maxHeapify(arr, largest, heapSize)!
    // 下面的子树可能还不满足堆性质！
}

// ❌ 错误 3：搞错左右孩子公式
const left = 2 * i;      // ❌ 应该是 2*i + 1
const right = 2 * i + 1; // ❌ 应该是 2*i + 2
```

## 🏗️ 建堆（Build Max Heap）

### 核心思想

```
从一个乱序数组构建最大堆

方法：
从最后一个非叶子节点开始，往前遍历
对每个节点调用 maxHeapify

为什么从后往前？
因为叶子节点天然满足堆性质（没有孩子）
只需要修复非叶子节点
```

### 代码实现

```javascript
/**
 * 构建最大堆
 * 
 * @param {number[]} arr - 输入数组
 */
function buildMaxHeap(arr) {
    const heapSize = arr.length;
    
    // 从最后一个非叶子节点开始
    // 向前遍历到根节点
    for (let i = Math.floor(arr.length / 2) - 1; i >= 0; i--) {
        maxHeapify(arr, i, heapSize);
    }
    
    return arr;
}

// ⏱️ 时间复杂度：O(n)（不是 O(n log n)，很神奇！）
```

### 为什么是 O(n) 而不是 O(n log n)？

```
直观理解：

大部分节点在底层，高度很小
只有少数节点在顶层，高度大

详细分析：
- 底层（高度 0）：n/2 个节点，每个 O(1)
- 倒数第二层（高度 1）：n/4 个节点，每个 O(1)
- ...
- 顶层（高度 log n）：1 个节点，O(log n)

总时间：Σ (层节点数 × 层高)
      = Σ (n/2^h × h)
      = n × Σ (h/2^h)
      = O(n)  ← 这是个收敛级数！

所以建堆是 O(n)，不是 O(n log n)！✨
```

## 🎯 堆排序算法

### 完整流程

```javascript
/**
 * 堆排序
 * 
 * @param {number[]} arr - 待排序数组
 * @returns {number[]} 排好序的数组
 */
function heapSort(arr) {
    // 第 1 步：建堆
    buildMaxHeap(arr);
    
    // 第 2 步：反复提取最大值
    for (let i = arr.length - 1; i > 0; i--) {
        // 将当前最大值（根节点）放到最后
        [arr[0], arr[i]] = [arr[i], arr[0]];
        
        // 堆大小减 1
        // 并对新的根节点调用 maxHeapify
        maxHeapify(arr, 0, i);
    }
    
    return arr;
}

// ⏱️ 时间复杂度：O(n log n)
// 空间复杂度：O(1) 原地排序
```

## 🎫 优先队列（堆的重要应用）

### 什么是优先队列？

```
普通队列：先进先出（FIFO）
排队买票，先来先服务

优先队列：优先级高的先出
急诊室看病，病重的先看！🏥
```

### 最大优先队列实现

```javascript
/**
 * 最大优先队列
 */
class MaxPriorityQueue {
    constructor() {
        this.heap = [];
    }
    
    /**
     * 返回最大值
     */
    maximum() {
        if (this.heap.length === 0) return null;
        return this.heap[0];
    }
    
    /**
     * 提取并删除最大值
     */
    extractMax() {
        if (this.heap.length === 0) return null;
        
        const max = this.heap[0];
        // 把最后一个元素移到根
        this.heap[0] = this.heap.pop();
        
        // 如果不是空堆，修复堆性质
        if (this.heap.length > 0) {
            maxHeapify(this.heap, 0, this.heap.length);
        }
        
        return max;
    }
    
    /**
     * 增加某个元素的键值
     */
    increaseKey(i, key) {
        if (key < this.heap[i]) {
            throw new Error('新键值必须更大');
        }
        
        this.heap[i] = key;
        
        // 向上浮
        while (i > 0 && this.heap[Math.floor((i - 1) / 2)] < this.heap[i]) {
            const parent = Math.floor((i - 1) / 2);
            [this.heap[i], this.heap[parent]] = [this.heap[parent], this.heap[i]];
            i = parent;
        }
    }
    
    /**
     * 插入新元素
     */
    insert(key) {
        this.heap.push(key);
        // 从最后一个位置开始上浮
        this.increaseKey(this.heap.length - 1, key);
    }
}

// 🧪 测试
const pq = new MaxPriorityQueue();
pq.insert(10);
pq.insert(20);
pq.insert(5);
console.log(pq.maximum());  // 20
console.log(pq.extractMax());  // 20
console.log(pq.maximum());  // 10
```
