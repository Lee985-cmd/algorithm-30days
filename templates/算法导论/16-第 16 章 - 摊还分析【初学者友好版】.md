## 🔑 核心概念

### 什么是摊还分析？

```
目的：分析一系列操作的总代价

关键思想：
虽然某些操作代价高
但其他操作代价低
平均下来每个操作的代价很低

三种方法：
1. 聚合分析：算总价，再平均
2. 记账法：预先存钱，以后花
3. 势能法：存势能，释放时用
```

## 🎯 经典例子：动态数组

### 问题描述

```javascript
/**
 * 动态数组（类似 JavaScript Array）
 */
class DynamicArray {
    constructor() {
        this.array = new Array(1);  // 初始容量 1
        this.size = 0;               // 当前元素个数
        this.capacity = 1;           // 当前容量
    }
    
    /**
     * 在末尾添加元素
     */
    push(val) {
        // 如果满了，扩容
        if (this.size === this.capacity) {
            this.resize(this.capacity * 2);
        }
        
        this.array[this.size++] = val;
    }
    
    /**
     * 扩容操作
     */
    resize(newCapacity) {
        const newArray = new Array(newCapacity);
        
        // 复制旧数组的所有元素
        for (let i = 0; i < this.size; i++) {
            newArray[i] = this.array[i];
        }
        
        this.array = newArray;
        this.capacity = newCapacity;
    }
}
```

### 摊还分析

```
问题：push 操作的时间复杂度是多少？

直觉回答：O(n)，因为可能要复制所有元素
❌ 错了！这是最坏情况，不是摊还代价

正确答案：O(1) 摊还时间 ✨

为什么？看下面的分析！
```

## 📊 方法 1：聚合分析

```
考虑 n 次 push 操作的总代价：

假设初始容量是 1

第 1 次 push: O(1)
第 2 次 push: O(2)  ← 扩容，复制 1 个元素
第 3 次 push: O(1)
第 4 次 push: O(4)  ← 扩容，复制 3 个元素
第 5-7 次 push: O(1)
第 8 次 push: O(8)  ← 扩容，复制 7 个元素
...

观察规律：
扩容发生在第 2, 4, 8, 16, ... 次 push
每次扩容的代价是当前的元素个数

总代价 = Σ(i=1 to log n) 2^i
      = 1 + 2 + 4 + 8 + ... + n
      = 2n - 1
      = O(n)

所以 n 次 push 的总代价是 O(n)
摊还到每次：O(n) / n = O(1) ✓

结论：每次 push 的摊还代价是 O(1)！✨
```

## 📊 方法 2：记账法（银行家方法）

```
想象每个操作可以"存钱"：

策略：
- 每次 push 收 3 块钱
- 实际 push 花 1 块
- 剩下 2 块存起来，以后扩容用

花钱过程：

第 1 次 push:
收入 3 块，花 1 块，存 2 块
余额：2 块

第 2 次 push（要扩容）:
收入 3 块
扩容要花 2 块（复制 1 个元素）
实际 push 花 1 块
之前存的 2 块正好用来扩容！
余额：3 块

第 3 次 push:
收入 3 块，花 1 块，存 2 块
余额：5 块

第 4 次 push（要扩容）:
收入 3 块
扩容要花 4 块（复制 3 个元素）
实际 push 花 1 块
之前存的 5 块够用了！
余额：3 块

...

关键：每次扩容前，存的钱都够用！
所以摊还代价是 O(1) ✓
```

### 图解记账过程

```
操作序列：push A, B, C, D, E, F, G, H

余额变化：
初始：0

push A: +3-1=2   余额：2
push B: +3-2-1=0 余额：2  （扩容花 2 块）
push C: +3-1=2   余额：4
push D: +3-4-1=-2 余额：2 （扩容花 4 块，之前存了 4 块）
push E: +3-1=2   余额：4
push F: +3-1=2   余额：6
push G: +3-1=2   余额：8
push H: +3-8-1=-6 余额：2（扩容花 8 块，之前存了 8 块）

看到没？每次扩容前都有足够的钱！✨
```

## 📊 方法 3：势能法（最难但最强）

### 核心思想

```
势能法就像弹簧：
- 压缩时储存势能
- 释放时消耗势能

定义势能函数 Φ(D):
- 数据结构的状态 → 一个数字
- 势能增加：存能量
- 势能减少：放能量

摊还代价 = 实际代价 + ΔΦ
         = 实际代价 + (Φ新 - Φ旧)
```

### 应用到动态数组

```javascript
// 定义势能函数
Φ(D) = 2 × (size - capacity/2)

解释：
- size = 当前元素个数
- capacity = 当前容量
- 当 size = capacity/2 时，Φ = 0
- 当 size = capacity 时（满了），Φ = capacity（势能最大）
```

### 分析过程

```
情况 1: 不扩容

假设第 i 次 push 不需要扩容：
- 实际代价：cᵢ = 1
- size 增加 1
- ΔΦ = 2 × ((size+1) - capacity/2) - 2 × (size - capacity/2)
     = 2

摊还代价：ĉᵢ = cᵢ + ΔΦ = 1 + 2 = 3 = O(1)


情况 2: 扩容

假设第 i 次 push 需要扩容（从 capacity 到 2×capacity）：
- 实际代价：cᵢ = size + 1 ≈ capacity + 1（复制 + 插入）
- 扩容前：size = capacity
- 扩容后：size = capacity + 1, capacity' = 2×capacity

ΔΦ = 2 × ((capacity+1) - (2×capacity)/2) - 2 × (capacity - capacity/2)
   = 2 × (capacity+1 - capacity) - 2 × (capacity/2)
   = 2 - capacity

摊还代价：ĉᵢ = cᵢ + ΔΦ
         = (capacity + 1) + (2 - capacity)
         = 3 = O(1)

两种情况的摊还代价都是 O(1)！✨
```

### 常见问题

**Q: 摊还分析和平均分析有什么区别？**

```
✅ 标准答案：

摊还分析：
- 针对最坏情况的一系列操作
- 保证每个操作的平均代价
- 确定性分析

平均分析：
- 针对随机输入
- 期望时间复杂度
- 概率分析

例子：
- 动态数组 push：摊还 O(1)
- 快速排序：平均 O(n log n)
```
