## 🎯 最小值和最大值

### 同时找最小和最大

```javascript
/**
 * 同时找最小值和最大值
 * 
 * @param {number[]} arr - 输入数组
 * @returns {[number, number]} [最小值，最大值]
 */
function findMinMax(arr) {
    if (arr.length === 0) return [null, null];
    
    let min = arr[0];
    let max = arr[0];
    
    // 成对处理元素
    for (let i = 1; i < arr.length; i += 2) {
        if (i + 1 < arr.length) {
            // 先比较这一对
            let small, large;
            if (arr[i] < arr[i + 1]) {
                small = arr[i];
                large = arr[i + 1];
            } else {
                small = arr[i + 1];
                large = arr[i];
            }
            
            // 更新最小和最大
            if (small < min) min = small;
            if (large > max) max = large;
        } else {
            // 最后一个元素
            if (arr[i] < min) min = arr[i];
            if (arr[i] > max) max = arr[i];
        }
    }
    
    return [min, max];
}

// 🧪 测试
const arr = [3, 1, 7, 2, 9, 4, 6];
const [min, max] = findMinMax(arr);
console.log('最小值:', min);  // 1
console.log('最大值:', max);  // 9

// ⏱️ 时间复杂度：O(n)
// 比较次数：约 3n/2 次（比单独找省）
```

## 🔍 快速选择算法（重点！）

### 核心思想

```
利用快速排序的 partition 思想：

1. 选一个 pivot
2. 分区：左边 ≤ pivot，右边 ≥ pivot
3. 如果 pivot 的位置正好是 k，返回
4. 如果 pivot 在 k 左边，在右边继续找
5. 如果 pivot 在 k 右边，在左边继续找

关键：只处理一边，不用两边都处理！
所以比快速排序快！⚡
```

### 一步一步演示

```
找第 4 小的元素（k=4）
数组：[10, 4, 5, 8, 6, 11, 26]

第 1 轮：选 pivot=26
partition 后：[10, 4, 5, 8, 6, 11, 26]
                      ↑
                   pivot 索引=6

pivot 是第 7 小（索引 6+1）
我们要找第 4 小
所以在左边继续找

第 2 轮：在 [10, 4, 5, 8, 6, 11] 中找第 4 小
选 pivot=11
partition 后：[10, 4, 5, 8, 6, 11]
                      ↑
                   pivot 索引=5

pivot 是第 6 小
还是在左边找

第 3 轮：在 [10, 4, 5, 8, 6] 中找第 4 小
选 pivot=6
partition 后：[4, 5, 6, 8, 10]
                  ↑
               pivot 索引=2

pivot 是第 3 小（索引 2+1）
我们要找第 4 小
所以在右边找

第 4 轮：在 [8, 10] 中找第 1 小（因为前面已经有 3 个了）
选 pivot=10
partition 后：[8, 10]
              ↑
           pivot 索引=0

pivot 是第 1 小 ✓
返回 8

所以第 4 小的元素是 8！
```

### 代码实现

```javascript
/**
 * 快速选择算法 - 找第 k 小的元素
 * 
 * @param {number[]} arr - 输入数组
 * @param {number} k - 找第 k 小（从 1 开始）
 * @returns {number} 第 k 小的元素
 */
function quickSelect(arr, k) {
    if (k < 1 || k > arr.length) return null;
    
    return quickSelectHelper(arr, 0, arr.length - 1, k);
}

function quickSelectHelper(arr, low, high, k) {
    if (low === high) {
        return arr[low];
    }
    
    // 分区
    const pivotIndex = partition(arr, low, high);
    
    // pivot 是第几个小的（从 1 开始）
    const rank = pivotIndex - low + 1;
    
    if (rank === k) {
        // 正好是第 k 个
        return arr[pivotIndex];
    } else if (k < rank) {
        // 在左边找
        return quickSelectHelper(arr, low, pivotIndex - 1, k);
    } else {
        // 在右边找，注意 k 要减去左边的个数
        return quickSelectHelper(arr, pivotIndex + 1, high, k - rank);
    }
}

// partition 函数和快速排序一样
function partition(arr, low, high) {
    const pivot = arr[high];
    let i = low - 1;
    
    for (let j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            [arr[i], arr[j]] = [arr[j], arr[i]];
        }
    }
    
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
    return i + 1;
}

// 🧪 测试
const arr = [10, 4, 5, 8, 6, 11, 26];
for (let k = 1; k <= arr.length; k++) {
    console.log(`第${k}小的元素:`, quickSelect([...arr], k));
}
// 输出：
// 第 1 小的元素：4
// 第 2 小的元素：5
// 第 3 小的元素：6
// 第 4 小的元素：8
// 第 5 小的元素：10
// 第 6 小的元素：11
// 第 7 小的元素：26

// ⏱️ 平均时间复杂度：O(n)
// 最坏时间复杂度：O(n²)
```

## ⏱️ 复杂度分析

### 为什么平均是 O(n)？

```
直观理解：

每次 partition 后，只处理一边
期望情况下，每次排除一半

T(n) = T(n/2) + O(n)
     = O(n) + O(n/2) + O(n/4) + ...
     = O(n × (1 + 1/2 + 1/4 + ...))
     = O(n × 2)
     = O(n)

几何级数求和！✨
```

### 数据对比

```
找第 k 小的元素：

方法 1：先排序再取
时间复杂度：O(n log n)

方法 2：快速选择
平均：O(n)
最坏：O(n²)

数据量 n=10000:
O(n log n) ≈ 130000 次
O(n) ≈ 10000 次

快了 13 倍！🚀
```
