## 🔑 核心概念

### 什么是散列表？

```
散列表 = 数组 + 哈希函数

数组：一排连续的储物柜 [0, 1, 2, ..., m-1]
哈希函数：给你分配柜子的规则
键值对：你存的东西（key → value）
```

### 哈希函数的作用

```javascript
/**
 * 简单的哈希函数示例
 * 
 * @param {string} key - 键（字符串）
 * @param {number} tableSize - 表的大小
 * @returns {number} 哈希值（数组索引）
 */
function simpleHash(key, tableSize) {
    let hash = 0;
    
    // 把字符串转成数字
    for (let i = 0; i < key.length; i++) {
        hash += key.charCodeAt(i);  // 字符的 ASCII 码
    }
    
    // 取模，保证在数组范围内
    return hash % tableSize;
}

// 🧪 测试
console.log(simpleHash("apple", 10));   // 输出 0-9 之间的数
console.log(simpleHash("banana", 10));  // 输出 0-9 之间的数
```

## 💥 哈希冲突（重点！）

### 问题描述

```
两个不同的 key，hash 到了同一个位置：

key1 = "apple"  → hash = 37
key2 = "pear"   → hash = 37

37 号柜子只能放一个，怎么办？❓
```

### 解决方案 1：拉链法（Chaining）

```
每个柜子其实是个链表：

37 号柜 → ["apple": 苹果] → ["pear": 梨] → null

查找时：
1. 算出 hash=37
2. 走到 37 号柜
3. 遍历链表找到 "apple"

优点：
✓ 简单易懂
✓ 表满了也能继续加

缺点：
✗ 链表长了就慢（退化成 O(n)）
```

**代码实现：**

```javascript
class HashTableChaining {
    constructor(size = 16) {
        this.size = size;
        this.buckets = new Array(size);
        // 每个桶初始化成一个空数组（当作链表）
        for (let i = 0; i < size; i++) {
            this.buckets[i] = [];
        }
    }
    
    _hash(key) {
        let hash = 0;
        for (let i = 0; i < key.length; i++) {
            hash += key.charCodeAt(i);
        }
        return hash % this.size;
    }
    
    /**
     * 插入键值对
     */
    put(key, value) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        // 查找是否已存在
        for (let i = 0; i < bucket.length; i++) {
            if (bucket[i][0] === key) {
                bucket[i][1] = value;  // 更新
                return;
            }
        }
        
        // 不存在就添加
        bucket.push([key, value]);
    }
    
    /**
     * 获取值
     */
    get(key) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let i = 0; i < bucket.length; i++) {
            if (bucket[i][0] === key) {
                return bucket[i][1];
            }
        }
        
        return null;  // 没找到
    }
    
    /**
     * 删除
     */
    remove(key) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let i = 0; i < bucket.length; i++) {
            if (bucket[i][0] === key) {
                bucket.splice(i, 1);
                return true;
            }
        }
        
        return false;
    }
}

// 🧪 测试
const ht = new HashTableChaining(10);
ht.put("apple", "苹果");
ht.put("pear", "梨");
console.log(ht.get("apple"));  // "苹果"
```

### 解决方案 2：开放地址法（Open Addressing）

```
冲突了就找下一个空位：

37 号满了？试试 38 号
38 号也满了？试试 39 号
...直到找到空位

线性探测：
hash(key) → 37 满了 → 38 → 39 → ...

二次探测：
hash(key) → 37 满了 → 37+1² → 37+2² → ...

双重哈希：
再用另一个哈希函数算一次
```

**代码实现（线性探测）：**

```javascript
class HashTableLinearProbing {
    constructor(size = 16) {
        this.size = size;
        this.keys = new Array(size);
        this.values = new Array(size);
    }
    
    _hash(key) {
        let hash = 0;
        for (let i = 0; i < key.length; i++) {
            hash += key.charCodeAt(i);
        }
        return hash % this.size;
    }
    
    put(key, value) {
        let index = this._hash(key);
        
        // 如果冲突了，找下一个空位
        while (this.keys[index] !== undefined) {
            if (this.keys[index] === key) {
                this.values[index] = value;  // 更新
                return;
            }
            index = (index + 1) % this.size;  // 循环找
        }
        
        // 找到空位
        this.keys[index] = key;
        this.values[index] = value;
    }
    
    get(key) {
        let index = this._hash(key);
        
        while (this.keys[index] !== undefined) {
            if (this.keys[index] === key) {
                return this.values[index];
            }
            index = (index + 1) % this.size;
        }
        
        return null;
    }
}
```

## ⏱️ 复杂度分析

### 平均情况（运气好）

```
装填因子 α = n / m （元素个数 / 桶数）

α < 0.7 时：
- 查找：O(1)
- 插入：O(1)
- 删除：O(1)

超级快！⚡
```

### 最坏情况（倒霉透顶）

```
所有元素都 hash 到同一个位置：

变成一条长链表：
Bucket 37: A → B → C → D → ... → Z

查找：O(n) ❌
插入：O(n) ❌
删除：O(n) ❌

完全退化成链表！
```

### 如何避免最坏情况？

```
1. 设计好的哈希函数
   → 让元素均匀分布
   
2. 控制装填因子
   → α > 0.7 时就扩容
   
3. 使用更好的冲突解决方法
   → 拉链法 or 双重哈希
```

## 🎯 好的哈希函数标准

### 要求

```
✓ 一致性：相同的 key 总是得到相同的 hash
✓ 均匀性：尽量均匀分布在各个桶
✓ 高效性：计算要快
✓ 低碰撞：尽量减少冲突
```

### 常见哈希函数

```javascript
// 1. 除法哈希
function hash1(key, m) {
    return key % m;
}

// 2. 乘法哈希
function hash2(key, m) {
    const A = 0.6180339887;  // 黄金比例
    return Math.floor(m * ((key * A) % 1));
}

// 3. 字符串哈希（多项式滚动）
function hashString(str, m) {
    let hash = 0;
    const p = 31;  // 质数
    
    for (let i = 0; i < str.length; i++) {
        hash = (hash * p + str.charCodeAt(i)) % m;
    }
    
    return hash;
}

// 4. JavaScript 内置的 hashCode
function jsHashCode(str) {
    let hash = 0;
    for (let i = 0; i < str.length; i++) {
        const char = str.charCodeAt(i);
        hash = ((hash << 5) - hash) + char;
        hash = hash & hash;  // 转成 32 位整数
    }
    return hash;
}
```
