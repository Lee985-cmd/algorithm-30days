## 🔑 核心概念

### 什么是生成树？

```
生成树 = 连接所有顶点的最小边集

特点：
✓ 包含所有顶点
✓ 是一棵树（没有环）
✓ 有 n-1 条边（n 是顶点数）

最小生成树（MST）：
所有生成树中，边的权重和最小的那个
```

### 图解 MST

```
原图：
    A --4-- B
    | \     |
   2|  \3   |5
    |   \   |
    C --1-- D

MST（权重和=6）：
    A       B
    | \     
   2|  \3   
    |   \   
    C --1-- D

选中的边：A-C(2), A-D(3), C-D(1)
总权重：2+3+1=6 ✓
```

## 🎯 Kruskal 算法（克鲁斯卡尔）

### 💡 核心思想

```
Kruskal 算法就像拼拼图：

1. 把所有边按权重从小到大排序
2. 从最小的边开始选
3. 如果这条边不会形成环，就选它
4. 重复直到选了 n-1 条边

贪心策略：永远选当前最小的安全边！✨
```

### 代码实现

```javascript
/**
 * Kruskal 算法求 MST
 * 
 * @param {Array} edges - 边数组 [[u, v, weight]]
 * @param {number} n - 顶点数
 * @returns {Array} MST 的边
 */
function kruskal(edges, n) {
    // 步骤 1: 按权重排序
    edges.sort((a, b) => a[2] - b[2]);
    
    // 步骤 2: 初始化并查集
    const uf = new UnionFind(n);
    
    const mst = [];
    let totalWeight = 0;
    
    // 步骤 3: 贪心选择
    for (const [u, v, weight] of edges) {
        // 如果 u 和 v 不在同一个集合（不会形成环）
        if (uf.find(u) !== uf.find(v)) {
            mst.push([u, v, weight]);
            totalWeight += weight;
            uf.union(u, v);  // 合并集合
            
            // 已经选了 n-1 条边，完成
            if (mst.length === n - 1) break;
        }
    }
    
    return { mst, totalWeight };
}

/**
 * 并查集（Union-Find）
 */
class UnionFind {
    constructor(n) {
        this.parent = Array.from({ length: n }, (_, i) => i);
        this.rank = new Array(n).fill(0);
    }
    
    find(x) {
        if (this.parent[x] !== x) {
            this.parent[x] = this.find(this.parent[x]);  // 路径压缩
        }
        return this.parent[x];
    }
    
    union(x, y) {
        const rootX = this.find(x);
        const rootY = this.find(y);
        
        if (rootX === rootY) return false;
        
        // 按秩合并
        if (this.rank[rootX] < this.rank[rootY]) {
            this.parent[rootX] = rootY;
        } else if (this.rank[rootX] > this.rank[rootY]) {
            this.parent[rootY] = rootX;
        } else {
            this.parent[rootY] = rootX;
            this.rank[rootX]++;
        }
        
        return true;
    }
}

// 🧪 测试
const edges = [
    [0, 1, 4],
    [0, 2, 2],
    [0, 3, 3],
    [1, 3, 5],
    [2, 3, 1]
];

const result = kruskal(edges, 4);
console.log('MST 的边:', result.mst);
console.log('总权重:', result.totalWeight);
// 输出：
// MST 的边：[[2,3,1], [0,2,2], [0,3,3]]
// 总权重：6 ✓
```

### 一步一步演示

```
初始图：
边列表：[(0,1,4), (0,2,2), (0,3,3), (1,3,5), (2,3,1)]

步骤 1: 排序
[(2,3,1), (0,2,2), (0,3,3), (0,1,4), (1,3,5)]

步骤 2: 选边
选 (2,3,1): 2 和 3 不连通 → 选！MST=[(2,3,1)]
选 (0,2,2): 0 和 2 不连通 → 选！MST=[(2,3,1), (0,2,2)]
选 (0,3,3): 0 和 3 已连通（通过 2）→ 跳过！❌
选 (0,1,4): 0 和 1 不连通 → 选！MST=[(2,3,1), (0,2,2), (0,1,4)]

已经选了 3 条边（n-1），完成！✅
总权重：1+2+4=7
```

## 🎯 Prim 算法（普里姆）

### 💡 核心思想

```
Prim 算法就像滚雪球：

1. 随便选一个起点
2. 找离当前树最近的点，加进来
3. 重复直到所有点都加进来

贪心策略：永远选离得最近的！✨
```

### 代码实现

```javascript
/**
 * Prim 算法求 MST
 * 
 * @param {Array} graph - 邻接矩阵
 * @param {number} n - 顶点数
 * @returns {Object} MST 的边和总权重
 */
function prim(graph, n) {
    const inMST = new Array(n).fill(false);  // 是否在 MST 中
    const minDist = new Array(n).fill(Infinity);  // 到 MST 的最短距离
    const parent = new Array(n).fill(-1);  // 父节点
    
    // 从节点 0 开始
    minDist[0] = 0;
    
    let totalWeight = 0;
    const mstEdges = [];
    
    // 需要选 n 个点
    for (let count = 0; count < n; count++) {
        // 找不在 MST 中且距离最小的点
        let u = -1;
        let minVal = Infinity;
        
        for (let i = 0; i < n; i++) {
            if (!inMST[i] && minDist[i] < minVal) {
                minVal = minDist[i];
                u = i;
            }
        }
        
        if (u === -1) break;  // 没有更多点了
        
        // 把 u 加入 MST
        inMST[u] = true;
        totalWeight += minVal;
        
        if (parent[u] !== -1) {
            mstEdges.push([parent[u], u, graph[parent[u]][u]]);
        }
        
        // 更新相邻点的距离
        for (let v = 0; v < n; v++) {
            if (graph[u][v] > 0 && !inMST[v] && graph[u][v] < minDist[v]) {
                minDist[v] = graph[u][v];
                parent[v] = u;
            }
        }
    }
    
    return { mstEdges, totalWeight };
}

// 🧪 测试
const graph = [
    [0, 4, 2, 3],
    [4, 0, 0, 5],
    [2, 0, 0, 1],
    [3, 5, 1, 0]
];

const result = prim(graph, 4);
console.log('MST 的边:', result.mstEdges);
console.log('总权重:', result.totalWeight);
// 输出：
// MST 的边：[[0,2,2], [2,3,1], [0,1,4]]
// 总权重：7 ✓
```

### 一步一步演示

```
从节点 0 开始：

初始状态：
inMST = [F, F, F, F]
minDist = [0, ∞, ∞, ∞]

第 1 轮：选节点 0
inMST = [T, F, F, F]
更新邻居：
- 节点 1: minDist[1] = 4, parent[1] = 0
- 节点 2: minDist[2] = 2, parent[2] = 0
- 节点 3: minDist[3] = 3, parent[3] = 0

第 2 轮：选节点 2（距离最小=2）
inMST = [T, F, T, F]
totalWeight = 2
更新邻居：
- 节点 3: minDist[3] = min(3, 1) = 1, parent[3] = 2

第 3 轮：选节点 3（距离最小=1）
inMST = [T, F, T, T]
totalWeight = 2+1 = 3
MST 边：[(2,3,1)]
更新邻居：无

第 4 轮：选节点 1（距离=4）
inMST = [T, T, T, T]
totalWeight = 3+4 = 7
MST 边：[(2,3,1), (0,1,4)]

完成！✅
```

## ⏱️ 复杂度对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| **Kruskal** | O(E log E) | O(V) | 稀疏图（边少） |
| **Prim** | O(V²) | O(V) | 稠密图（边多） |

**优化版本：**
- Prim + 堆优化：O(E log V)
- Kruskal + 并查集：O(E log E)

### 常见问题

**Q: Kruskal 和 Prim 哪个更好？**

```
✅ 标准答案：

看情况！

Kruskal 优点：
1. 适合稀疏图（E << V²）
2. 实现简单（排序 + 并查集）
3. 不需要知道图的结构

Prim 优点：
1. 适合稠密图（E ≈ V²）
2. 堆优化后很快
3. 从一个点开始，适合在线算法

工程实践：
- 边少用 Kruskal
- 边多用 Prim
```
