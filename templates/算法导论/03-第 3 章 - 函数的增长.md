# 📚 算法导论第 3 章 - 函数的增长

## 3.1 渐近记号

### O 记号（大 O）- 上界

```
定义：
O(g(n)) = { f(n) : 存在正常数 c 和 n₀，
           使得对所有 n ≥ n₀，有 0 ≤ f(n) ≤ cg(n) }

说人话：
f(n) 的增长速度"不超过"g(n)
g(n) 是 f(n) 的上界

例如：
2n² + 3n + 5 = O(n²)
因为当 n 很大时，主要项是 n²
```

### Ω 记号 - 下界

```
定义：
Ω(g(n)) = { f(n) : 存在正常数 c 和 n₀，
           使得对所有 n ≥ n₀，有 0 ≤ cg(n) ≤ f(n) }

说人话：
f(n) 的增长速度"至少是"g(n)
g(n) 是 f(n) 的下界

例如：
2n² + 3n + 5 = Ω(n²)
因为增长速度不会低于 n²
```

### Θ 记号 - 紧确界

```
定义：
Θ(g(n)) = { f(n) : 存在正常数 c₁, c₂ 和 n₀，
           使得对所有 n ≥ n₀，有 0 ≤ c₁g(n) ≤ f(n) ≤ c₂g(n) }

说人话：
f(n) 的增长速度"等于"g(n)
g(n) 是 f(n) 的紧确界

例如：
2n² + 3n + 5 = Θ(n²)
因为增长速度就是 n² 级别
```

## 3.2 常见的时间复杂度

### 从快到慢排序

```
O(1)      < 常数级          最快
O(log n)  < 对数级
O(n)      < 线性级
O(n log n)< 线性对数级
O(n²)     < 平方级
O(n³)     < 立方级
O(2ⁿ)     < 指数级          最慢
```

### 例子对比

```javascript
// O(1) - 常数时间
function constant(n) {
    return 1;
}

// O(log n) - 对数时间（二分查找）
function logarithmic(n) {
    let count = 0;
    while (n > 1) {
        n /= 2;
        count++;
    }
    return count;
}

// O(n) - 线性时间
function linear(n) {
    let sum = 0;
    for (let i = 0; i < n; i++) {
        sum += i;
    }
    return sum;
}

// O(n²) - 平方时间（冒泡排序）
function quadratic(n) {
    let count = 0;
    for (let i = 0; i < n; i++) {
        for (let j = 0; j < n; j++) {
            count++;
        }
    }
    return count;
}

// O(2ⁿ) - 指数时间（递归斐波那契）
function exponential(n) {
    if (n <= 1) return 1;
    return exponential(n-1) + exponential(n-2);
}
```

## 3.3 增长率比较

### 数值对比表

```
n       | log n | n     | n log n | n²      | 2ⁿ
--------|-------|-------|---------|---------|------------------
10      | 3     | 10    | 33      | 100     | 1024
100     | 7     | 100   | 664     | 10000   | 1.26×10³⁰
1000    | 10    | 1000  | 9966    | 1000000 | 天文数字
10000   | 13    | 10000 | 132877  | 100000000| 无法计算
```

### 实际意义

```
O(log n):
  n=100 万，只需约 20 次操作 ✓

O(n):
  n=100 万，需要 100 万次操作 ✓

O(n log n):
  n=100 万，需要约 2000 万次操作 ✓

O(n²):
  n=100 万，需要 1 万亿次操作 ❌ 太慢！

O(2ⁿ):
  n=100，完全无法计算 ❌ 不可能！
```
