## 2.1 插入排序详解

### 问题描述

```
输入：n 个数的一个序列 (a₁, a₂, ..., aₙ)
输出：序列的一个排列 (a'₁, a'₂, ..., a'ₙ)，满足 a'₁ ≤ a'₂ ≤ ... ≤ a'ₙ
```

### 算法实现

```javascript
/**
 * 插入排序
 * @param {number[]} arr - 待排序数组
 */
function insertionSort(arr) {
    // 从第二个元素开始（第一个元素已经是有序的）
    for (let j = 1; j < arr.length; j++) {
        const key = arr[j];     // 当前要插入的元素
        let i = j - 1;          // 已排序区域的最后一个元素
        
        // 将比 key 大的元素向后移动
        while (i >= 0 && arr[i] > key) {
            arr[i + 1] = arr[i];
            i--;
        }
        
        // 将 key 插入到正确位置
        arr[i + 1] = key;
    }
    
    return arr;
}

// 测试
console.log(insertionSort([5, 2, 4, 6, 1, 3]));
// 输出：[1, 2, 3, 4, 5, 6] ✓
```

### 详细执行过程

```
初始状态：[5, 2, 4, 6, 1, 3]

j=1, key=2:
  [5, 2, 4, 6, 1, 3]
   ↑  ↑
   i  j
  5 > 2，移动 5 → [5, 5, 4, 6, 1, 3]
  i=-1，停止
  插入 2 → [2, 5, 4, 6, 1, 3] ✓

j=2, key=4:
  [2, 5, 4, 6, 1, 3]
      ↑  ↑
      i  j
  5 > 4，移动 5 → [2, 5, 5, 6, 1, 3]
  2 < 4，停止
  插入 4 → [2, 4, 5, 6, 1, 3] ✓

j=3, key=6:
  [2, 4, 5, 6, 1, 3]
         ↑  ↑
         i  j
  5 < 6，不移动
  直接插入 → [2, 4, 5, 6, 1, 3] ✓

j=4, key=1:
  [2, 4, 5, 6, 1, 3]
            ↑  ↑
            i  j
  依次移动 6,5,4,2 → [2, 4, 5, 6, 6, 3]
                   → [2, 4, 5, 5, 6, 3]
                   → [2, 4, 4, 5, 6, 3]
                   → [2, 2, 4, 5, 6, 3]
  i=-1，停止
  插入 1 → [1, 2, 4, 5, 6, 3] ✓

j=5, key=3:
  [1, 2, 4, 5, 6, 3]
               ↑  ↑
               i  j
  依次移动 6,5,4 → [1, 2, 4, 5, 6, 6]
                → [1, 2, 4, 5, 5, 6]
                → [1, 2, 4, 4, 5, 6]
  2 < 3，停止
  插入 3 → [1, 2, 3, 4, 5, 6] ✓

最终结果：[1, 2, 3, 4, 5, 6] ✓
```

## 2.2 循环不变式与算法正确性

### 什么是循环不变式？

```
循环不变式（Loop Invariant）是一个性质：
- 在循环开始时成立
- 在每次迭代后保持成立
- 在循环终止时，能证明算法的正确性

类似于数学归纳法！
```

### 插入排序的循环不变式

```
在第 j 次迭代开始时：
子数组 A[0..j-1] 包含原来在位置 0..j-1 的元素，
且已经排好序。

证明三步骤：

1. 初始化（Initialization）：
   j = 1 时，A[0..0] 只有一个元素 [5]
   显然有序 ✓

2. 保持（Maintenance）：
   假设某次迭代前 A[0..j-1] 有序
   插入 A[j] 后，A[0..j] 仍然有序 ✓

3. 终止（Termination）：
   当 j = n 时，循环结束
   A[0..n-1] 有序，即整个数组有序 ✓

证毕！
```

## 2.3 归并排序 - 分治法的例子

### 分治法思想

```
分治法（Divide and Conquer）：

1. 分解（Divide）：
   将原问题分解为若干个规模较小的子问题

2. 解决（Conquer）：
   递归地求解这些子问题
   如果子问题足够小，直接求解

3. 合并（Combine）：
   将子问题的解合并成原问题的解
```

### 归并排序实现

```javascript
/**
 * 归并排序
 */
function mergeSort(arr) {
    if (arr.length <= 1) {
        return arr;
    }
    
    // 分解
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);
    const right = arr.slice(mid);
    
    // 递归解决
    const sortedLeft = mergeSort(left);
    const sortedRight = mergeSort(right);
    
    // 合并
    return merge(sortedLeft, sortedRight);
}

/**
 * 合并两个有序数组
 */
function merge(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i]);
            i++;
        } else {
            result.push(right[j]);
            j++;
        }
    }
    
    // 添加剩余元素
    return result.concat(left.slice(i)).concat(right.slice(j));
}

// 测试
const arr = [38, 27, 43, 3, 9, 82, 10];
console.log('排序前:', arr);
console.log('排序后:', mergeSort(arr));
// 输出：[3, 9, 10, 27, 38, 43, 82] ✓
```

### 归并排序的执行过程

```
归并排序 [38, 27, 43, 3, 9, 82, 10]：

第 1 层：分解为 [38, 27, 43] 和 [3, 9, 82, 10]

第 2 层：
  [38, 27, 43] → [38] 和 [27, 43]
  [3, 9, 82, 10] → [3, 9] 和 [82, 10]

第 3 层：
  [27, 43] → [27] 和 [43]
  [3, 9] → [3] 和 [9]
  [82, 10] → [82] 和 [10]

现在都是单元素，开始合并：

合并 [27] 和 [43] → [27, 43]
合并 [38] 和 [27, 43] → [27, 38, 43]

合并 [3] 和 [9] → [3, 9]
合并 [82] 和 [10] → [10, 82]
合并 [3, 9] 和 [10, 82] → [3, 9, 10, 82]

最后合并：
[27, 38, 43] 和 [3, 9, 10, 82] → [3, 9, 10, 27, 38, 43, 82] ✓
```

### 时间复杂度分析

```
归并排序的时间复杂度：T(n) = 2T(n/2) + O(n)

展开递归树：
第 0 层：cn
第 1 层：2 × c(n/2) = cn
第 2 层：4 × c(n/4) = cn
...
第 log n 层：n × c(1) = cn

总共有 log n + 1 层
总时间 = cn × (log n + 1) = O(n log n)

对比插入排序：
- 插入排序：O(n²)
- 归并排序：O(n log n)

n=1000 时：
- 插入排序：1000² = 1,000,000 次操作
- 归并排序：1000 × log₂1000 ≈ 10000 次操作

归并排序快 100 倍！✓
```
