## 🔑 核心概念

### 流网络的基本概念

```
流网络 = 有向图 + 容量限制

特点：
✓ 每条边有容量 c(u,v) ≥ 0
✓ 有源点 s（只出不进）
✓ 有汇点 t（只进不出）
✓ 其他点：流入 = 流出（流量守恒）

流量 f 满足：
1. 容量限制：f(u,v) ≤ c(u,v)
2. 流量守恒：流入 = 流出
```

### 图解流网络

```
       10/10        5/10
    s ------→ a ------→ t
     \        |        ^
   6/6 \      | 4/4    | 10/10
        \     v        /
         ----→ b -----+
            4/10

解释：
s→a: 流量 10，容量 10（满了）
s→b: 流量 6，容量 6（满了）
a→b: 流量 4，容量 4（满了）
a→t: 流量 5，容量 10（没满）
b→t: 流量 10，容量 10（满了）

总流量 = 从 s 流出的总量 = 10+6 = 16 ✓
```

## 🎯 Ford-Fulkerson 算法

### 💡 核心思想

```
Ford-Fulkerson 算法就像找路：

1. 找一条从 s 到 t 的路（增广路）
   这条路上的所有边都有剩余容量
   
2. 往这条路灌水
   灌的量 = 路上最小的剩余容量
   
3. 更新剩余容量
   正向边减少，反向边增加
   
4. 重复直到找不到增广路

关键：允许"反悔"（通过反向边）✨
```

### 残量网络

```
原始网络：边的容量

残量网络：还剩多少容量可以用

定义：
- 正向边：c_f(u,v) = c(u,v) - f(u,v)
- 反向边：c_f(v,u) = f(u,v)

反向边的作用：
允许"反悔"，把已经分配的水收回来
重新分配到更需要的地方
```

### 代码实现

```javascript
/**
 * Ford-Fulkerson 算法求最大流
 * 
 * @param {Array} capacity - 容量矩阵
 * @param {number} s - 源点
 * @param {number} t - 汇点
 * @returns {number} 最大流量
 */
function fordFulkerson(capacity, s, t) {
    const n = capacity.length;
    
    // 残量网络（初始等于容量）
    const residual = capacity.map(row => [...row]);
    
    let maxFlow = 0;
    
    // 不断找增广路
    while (true) {
        // BFS 找增广路
        const parent = new Array(n).fill(-1);
        const queue = [s];
        parent[s] = s;
        
        while (queue.length > 0) {
            const u = queue.shift();
            
            for (let v = 0; v < n; v++) {
                if (parent[v] === -1 && residual[u][v] > 0) {
                    parent[v] = u;
                    queue.push(v);
                    
                    if (v === t) break;  // 找到汇点了
                }
            }
        }
        
        // 没有找到增广路
        if (parent[t] === -1) break;
        
        // 计算这条增广路能增加的流量
        let pathFlow = Infinity;
        let v = t;
        while (v !== s) {
            const u = parent[v];
            pathFlow = Math.min(pathFlow, residual[u][v]);
            v = u;
        }
        
        // 更新残量网络
        v = t;
        while (v !== s) {
            const u = parent[v];
            residual[u][v] -= pathFlow;  // 正向边减少
            residual[v][u] += pathFlow;  // 反向边增加
            v = u;
        }
        
        // 累加流量
        maxFlow += pathFlow;
    }
    
    return maxFlow;
}

// ⏱️ 时间复杂度：O(E × maxFlow)
// 空间复杂度：O(V²)
```

### 一步一步演示

```
初始网络：
    10      10
s ------→ a ------→ t
 \        |        ^
  \       | 4      | 10
   \      v        /
    ----→ b -----+
       6

第 1 次 BFS：找到 s→a→t
路径容量：min(10, 10) = 10
更新残量：
s→a: 10-10=0
a→t: 10-10=0
maxFlow = 10

第 2 次 BFS：找到 s→b→t
路径容量：min(6, 10) = 6
更新残量：
s→b: 6-6=0
b→t: 10-6=4
maxFlow = 10+6 = 16

第 3 次 BFS：找到 s→a→b→t
路径容量：min(0, 4, 4) = 0？不对！

等等！还有反向边！
残量网络中：
a→s: 10（反向边，可以反悔）
b→a: 4（反向边）

实际路径：s→b→a→t
但 a→t 已经是 0 了，走不通

所以找不到增广路了！

最终最大流 = 16 ✓
```

### 改进版本

时间复杂度：O(VE²)

## ⏱️ 复杂度分析

### Ford-Fulkerson

```
时间复杂度：O(E × maxFlow)

为什么？
- 每次至少增加 1 单位流量
- 最多增加 maxFlow 次
- 每次找增广路 O(E)

如果 maxFlow 很大，会很慢！❌
```

### Edmonds-Karp

```
时间复杂度：O(VE²)

证明复杂，记住结论：
- 最多找 O(VE) 条增广路
- 每条增广路 BFS 是 O(E)

总时间：O(VE²) ✓
```

## 🎯 实际应用

### 应用 1: 二分图匹配

```javascript
/**
 * 用最大流解决二分图匹配
 * 
 * 问题：n 个男生，m 个女生，某些人可以配对
 * 问：最多能配成多少对？
 */
function maxBipartiteMatching(adj, n, m) {
    // 建图：
    // 源点 s = 0
    // 男生 1..n
    // 女生 n+1..n+m
    // 汇点 t = n+m+1
    
    const size = n + m + 2;
    const capacity = Array.from({ length: size }, () => new Array(size).fill(0));
    
    // s 到每个男生，容量 1
    for (let i = 1; i <= n; i++) {
        capacity[0][i] = 1;
    }
    
    // 每个女生到 t，容量 1
    for (let j = 1; j <= m; j++) {
        capacity[n + j][size - 1] = 1;
    }
    
    // 男生到可配对的女生，容量 1
    for (const [boy, girl] of adj) {
        capacity[boy][n + girl] = 1;
    }
    
    // 跑最大流
    return fordFulkerson(capacity, 0, size - 1);
}
```

### 常见问题

**Q: 最大流最小割定理是什么？**

```
✅ 标准答案：

定理：最大流 = 最小割

解释：
- 割：把图分成两部分（s 在一边，t 在另一边）
- 割的容量：从 s 那边到 t 那边的边的容量和
- 最小割：所有割中容量最小的

直观理解：
水管系统的最细处决定了总流量
这个最细处就是最小割

例子：
s --10-- a --5-- t

最大流 = 5（受限于 a→t）
最小割 = 5（割掉 a→t 这条边）
```
