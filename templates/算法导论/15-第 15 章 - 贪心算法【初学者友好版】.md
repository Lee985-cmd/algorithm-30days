## 🔑 核心思想

### 贪心算法的特点

```
两个关键要素：

1. 贪心选择性质
   → 通过局部最优选择达到全局最优
   → 不需要考虑子问题的解
   
2. 最优子结构
   → 问题的最优解包含子问题的最优解
   → 这点和 DP 一样

vs 动态规划：

DP:    自底向上，考虑所有可能，选最优
贪心：自顶向下，只看眼前，直接选

贪心更快，但不一定对！⚠️
```

## 🎯 经典例题 1：找零钱问题

### 问题描述

```
你要给顾客找零 n 元钱
硬币面值：1 元、5 角、1 角、5 分、1 分
问：最少需要多少个硬币？
```

### 贪心策略

```javascript
/**
 * 找零钱问题（贪心算法）
 * 
 * @param {number} amount - 要找零的金额（单位：分）
 * @returns {number} 最少硬币数
 */
function makeChange(amount) {
    // 硬币面值（单位：分）
    const coins = [100, 50, 10, 5, 1];
    
    let count = 0;
    
    // 贪心策略：优先用大面值
    for (let coin of coins) {
        // 能用这个面值就用
        while (amount >= coin) {
            amount -= coin;
            count++;
        }
    }
    
    return count;
}

// 🧪 测试
console.log(makeChange(126));  // 输出：4
// 解释：1 元×1 + 2 角×1 + 1 分×1 = 4 个硬币 ✓
```

### 一步一步演示

```
找零 126 分：

步骤 1: 用 100 分（1 元）
       126 - 100 = 26
       count = 1

步骤 2: 用 50 分？不行，26 < 50
       跳过

步骤 3: 用 10 分（1 角）
       26 - 10 = 16
       16 - 10 = 6
       count = 3

步骤 4: 用 5 分
       6 - 5 = 1
       count = 4

步骤 5: 用 1 分
       1 - 1 = 0
       count = 5

总共 5 个硬币：1 元×1 + 1 角×2 + 5 分×1 + 1 分×1
```

### ⚠️ 贪心不一定对！

```
如果硬币面值是：[1, 3, 4]
要找零 6 元：

贪心策略：
4 + 1 + 1 = 3 个硬币

但最优解是：
3 + 3 = 2 个硬币 ❌

所以贪心不是永远正确！
必须证明后才能用！
```

## 🎯 经典例题 2：活动选择问题

### 问题描述

```
有 n 个活动，每个活动有开始时间 sᵢ 和结束时间 fᵢ
同一时间只能参加一个活动
问：最多能参加多少个活动？
```

### 贪心策略

```javascript
/**
 * 活动选择问题
 * 
 * @param {number[]} starts - 开始时间数组
 * @param {number[]} finishes - 结束时间数组
 * @returns {number[]} 选中的活动索引
 */
function activitySelection(starts, finishes) {
    const n = starts.length;
    
    // 按结束时间排序（贪心策略：早结束的优先）
    const activities = [];
    for (let i = 0; i < n; i++) {
        activities.push({ start: starts[i], finish: finishes[i], index: i });
    }
    
    activities.sort((a, b) => a.finish - b.finish);
    
    // 贪心选择
    const selected = [activities[0].index];
    let lastFinish = activities[0].finish;
    
    for (let i = 1; i < n; i++) {
        // 如果这个活动的开始时间晚于上一个活动的结束时间
        if (activities[i].start >= lastFinish) {
            selected.push(activities[i].index);
            lastFinish = activities[i].finish;
        }
    }
    
    return selected;
}

// 🧪 测试
const starts = [1, 3, 0, 5, 8, 5];
const finishes = [2, 4, 6, 7, 9, 9];
const result = activitySelection(starts, finishes);
console.log('选中的活动:', result);  // 输出：[0, 1, 3, 4]
```

### 为什么贪心是对的？

```
直观理解：

策略：选最早结束的

为什么对？
→ 早结束的活动给后面留下更多时间
→ 能容纳更多活动

反证法：
假设最优解不选最早结束的活动 A
而选了另一个活动 B
因为 A 结束得最早，所以 B 的结束时间 ≥ A
那么用 A 替换 B，仍然可行且活动数不变
所以贪心策略能得到最优解 ✓
```

## 🎯 经典例题 3：分数背包问题

### 问题描述

```
有一个容量为 W 的背包
有 n 个物品，每个物品有价值 vᵢ 和重量 wᵢ
可以取物品的一部分（比如切一半）
问：怎么装能让总价值最大？
```

### 贪心策略

```javascript
/**
 * 分数背包问题（贪心算法）
 * 
 * @param {Array} items - 物品数组 [{value, weight}]
 * @param {number} capacity - 背包容量
 * @returns {number} 最大价值
 */
function fractionalKnapsack(items, capacity) {
    // 计算性价比（价值/重量）
    items.forEach(item => {
        item.ratio = item.value / item.weight;
    });
    
    // 按性价比排序
    items.sort((a, b) => b.ratio - a.ratio);
    
    let totalValue = 0;
    let remaining = capacity;
    
    // 贪心选择
    for (let item of items) {
        if (remaining >= item.weight) {
            // 全拿
            totalValue += item.value;
            remaining -= item.weight;
        } else {
            // 拿一部分
            totalValue += item.ratio * remaining;
            remaining = 0;
            break;
        }
    }
    
    return totalValue;
}

// 🧪 测试
const items = [
    { value: 60, weight: 10 },  // 性价比 6
    { value: 100, weight: 20 }, // 性价比 5
    { value: 120, weight: 30 }  // 性价比 4
];
console.log(fractionalKnapsack(items, 50));  // 输出：240
```

### 一步一步演示

```
背包容量：50

物品信息：
A: 价值 60，重量 10，性价比 6
B: 价值 100，重量 20，性价比 5
C: 价值 120，重量 30，性价比 4

贪心策略：优先拿性价比高的

步骤 1: 拿 A（性价比最高）
       价值 +60，剩余容量 40

步骤 2: 拿 B（性价比第二）
       价值 +100，剩余容量 20

步骤 3: 拿 C 的一部分
       只能拿 20/30 = 2/3
       价值 +120 × 2/3 = 80

总价值：60 + 100 + 80 = 240 ✓
```

## ⏱️ 复杂度分析

### 找零钱问题

```
时间复杂度：O(n)，n 是硬币种类数
空间复杂度：O(1)

超级快！⚡
```

### 活动选择问题

```
时间复杂度：O(n log n) ← 主要是排序的时间
空间复杂度：O(n)

如果用计数排序可以到 O(n)！
```

### 分数背包问题

```
时间复杂度：O(n log n) ← 也是排序
空间复杂度：O(n)
```

### 常见问题

**Q: 怎么判断一道题能不能用贪心？**

```
✅ 标准答案：

尝试用贪心，然后证明！

证明方法：
1. 反证法
   假设贪心不是最优，推出矛盾
   
2. 数学归纳法
   证明贪心选择后，剩下的子问题仍然可以用贪心

3. 交换论证
   假设最优解和贪心解不同
   通过交换元素，证明贪心解不会更差

常见能用贪心的问题：
- 活动选择
- 分数背包
- 霍夫曼编码
- 最小生成树（Prim、Kruskal）
```
