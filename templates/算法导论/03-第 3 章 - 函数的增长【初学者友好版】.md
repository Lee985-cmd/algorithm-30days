## 💡 把 O 记号想象成...

### ❌ 学术说法（让人头大）

"O(g(n)) = { f(n) : 存在正常数 c 和 n₀，使得对所有 n ≥ n₀，有 0 ≤ f(n) ≤ cg(n) }"

看完一脸懵逼对不对？😵

### ✅ 生活化比喻（秒懂！）

**O 记号就像给算法分级** 🎮

```
游戏段位：
青铜 → 白银 → 黄金 → 铂金 → 钻石

算法段位：
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)

O(n²) 就是告诉你：
"这个算法是白银水平，数据大了会很慢！"

O(n log n) 就是告诉你：
"这个算法是黄金水平，大部分情况都够用！"
```

**O 记号就像预估时间** ⏰

```
妈妈说："你收拾房间要多久？"

精确回答："5 分 23 秒" ← 不可能这么准！

模糊回答：
"大概几分钟" ← 这就是 O 记号！

O(5 分钟) = 很快
O(1 小时) = 要很久
O(1 天) = 完蛋了

O 记号不关心具体几秒，只关心是哪个级别！
```

### 为什么要忽略低阶项？

**例子：做饭的时间** 🍳

```
做一顿饭包括：
1. 洗菜切菜：3n 分钟（n=人数）
2. 炒菜：2n 分钟
3. 准备调料：5 分钟（固定）

总时间：T(n) = 3n + 2n + 5 = 5n + 5

当 n=1（1 个人吃饭）：
5×1 + 5 = 10 分钟
这时候"5 分钟准备调料"不能忽略

当 n=100（100 个人吃饭）：
5×100 + 5 = 505 分钟
这时候"5 分钟准备调料"还重要吗？
完全不重要了！主要是 5n 在主导！

当 n=10000：
5×10000 + 5 = 50005 分钟
那 5 分钟就是个渣渣！✨

所以：T(n) = 5n + 5 ≈ O(n)
```

**关键思想：**
```
n 很大时，最高次项 dominates（主导）
低阶项和常数可以忽略不计！

这就是 O 记号的精髓！
```

## 🚶 常见代码的复杂度

### O(1) - 常数级别

```javascript
// 不管数组多长，都只执行一次
function getFirst(arr) {
    return arr[0];  // 就一行代码
}

// 哈希表查找也是 O(1)
const map = new Map();
map.set('key', 'value');
map.get('key');  // O(1) 直接拿到
```

**特点：** 数据量再大，时间都一样！⚡

### O(log n) - 对数级别

```javascript
// 二分查找
function binarySearch(arr, target) {
    let left = 0;
    let right = arr.length - 1;
    
    while (left <= right) {
        const mid = Math.floor((left + right) / 2);
        
        if (arr[mid] === target) {
            return mid;  // 找到了！
        } else if (arr[mid] < target) {
            left = mid + 1;  // 去右半边找
        } else {
            right = mid - 1;  // 去左半边找
        }
    }
    
    return -1;
}

// 每次排除一半，超级快！
```

**特点：** 数据翻一倍，只多花 1 次操作！🚀

### O(n) - 线性级别

```javascript
// 遍历数组
function findMax(arr) {
    let max = arr[0];
    
    for (let i = 1; i < arr.length; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    
    return max;
}

// 有几个元素就看几次，很公平
```

**特点：** 数据翻一倍，时间也翻一倍！🏃

### O(n log n) - 线性对数级别

```javascript
// 归并排序（复习一下）
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    
    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));
    
    return merge(left, right);
}

// 切 log n 刀，每刀合并 n 个元素
// 总共：n × log n
```

**特点：** 大数据的好朋友！🚴

### O(n²) - 平方级别

```javascript
// 插入排序、冒泡排序、选择排序
function bubbleSort(arr) {
    for (let i = 0; i < arr.length; i++) {
        for (let j = 0; j < arr.length - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}

// 两层循环，n × n = n²
```

**特点：** 小数据还行，大数据完蛋！🐌

## ⏱️ 直观理解各种复杂度

### O(1) vs O(n)

```
任务：找第 1 本书

O(1): 
书架：[书 A, 书 B, 书 C, ...]
      ↑
      直接拿！1 秒搞定

O(n):
书架：[书 A, 书 B, 书 C, ...]
从头到尾数到第 1 本...
1 本、2 本、3 本... 找到第 1 本
就算只有 1 本，也要看一遍

结果：O(1) 完胜！
```

### O(n) vs O(n²)

```
任务：让所有人互相握手

n 个人：

O(n): 
每个人握一次手
总共握 n 次 ✓

O(n²):
每个人要和所有人握手
第 1 个人：握 n 次
第 2 个人：握 n 次
...
第 n 个人：握 n 次
总共：n × n = n² 次 ❌

n=10: 100 次握手，还行
n=1000: 1,000,000 次握手，累死！
```

### O(log n) 有多快？

```
猜数字游戏（1-100 万）：

笨方法（O(n)）：
1, 2, 3, 4, ... 一个一个猜
最坏要猜 100 万次！😵

聪明方法（O(log n)）：
"是 50 万吗？" "大了"
"是 25 万吗？" "小了"
"是 37.5 万吗？" ...

每次排除一半！
只要猜 log₂100 万 ≈ 20 次！✨

20 次 vs 100 万次
快了 5 万倍！🚀
```

**1. 判断复杂度**
```javascript
// 下面这些函数分别是 O 多少？

(1) function foo(arr) {
    return arr.length;
}

(2) function bar(arr) {
    let sum = 0;
    for (let x of arr) sum += x;
    return sum;
}

(3) function baz(arr) {
    let count = 0;
    for (let i = 0; i < arr.length; i++) {
        for (let j = 0; j < arr.length; j++) {
            count++;
        }
    }
    return count;
}

【答案】
(1) O(1) - 直接返回长度
(2) O(n) - 遍历一遍
(3) O(n²) - 双重循环
```

```

B: O(n²)
C: O(log n)
D: O(2ⁿ)

从快到慢排序：___ < ___ < ___ < ___

【答案】
C < A < B < D
O(log n) < O(n log n) < O(n²) < O(2ⁿ)
```

### 常见问题

**Q: O(1)、O(log n)、O(n) 的区别？**

```
✅ 标准答案：

O(1): 常数时间，与数据量无关
例：数组随机访问 arr[i]

O(log n): 对数时间，每次排除一半
例：二分查找

O(n): 线性时间，遍历所有元素
例：找最大值

关系：O(1) < O(log n) < O(n)
```

**Q: 为什么快速排序平均是 O(n log n)？**

```
✅ 标准答案：

快速排序也是分治法：
1. 选一个 pivot
2. 分成两半
3. 递归排序

如果每次都能平分：
- 切 log n 刀
- 每刀处理 n 个元素
- 总共：n × log n

所以是 O(n log n)！
```

### 大厂真题

```
【腾讯 2023】

以下哪个算法的时间复杂度最低？

A. 冒泡排序
B. 归并排序  ✓
C. 插入排序
D. 选择排序

解析：选 B
冒泡、插入、选择都是 O(n²)
只有归并是 O(n log n)
```

```
【字节跳动 2023】

分析下面代码的时间复杂度：

for (int i = 0; i < n; i *= 2) {
    for (int j = 0; j < n; j++) {
        printf("%d ", j);
    }
}

A. O(n)
B. O(n log n)  ✓
C. O(n²)
D. O(log n)

解析：选 B
外层循环：i = 1,2,4,8,... → log n 次
内层循环：每次都执行 n 次
总共：n × log n
```

### 核心要点（必记）

```
✓ O 记号表示增长趋势（不是精确时间）
✓ 忽略低阶项和常数（n 很大时它们不重要）
✓ 常见复杂度：O(1) < O(log n) < O(n) < O(n log n) < O(n²)
✓ 数据越大，复杂度差距越恐怖
```
