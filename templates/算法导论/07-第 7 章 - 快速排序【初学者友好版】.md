## 👀 快速排序详解

### 核心思想（三步走）

```
快速排序三步骤：

1. 分解（Divide）：
   选一个 pivot（枢轴）
   把数组分成两部分：
   - 左边：都 ≤ pivot
   - 右边：都 ≥ pivot
   
2. 解决（Conquer）：
   递归地对左右两部分排序
   
3. 合并（Combine）：
   不需要！已经有序了！✨
```

### 一步一步演示

```
对 [10, 80, 30, 90, 40, 50, 70] 排序
选最后一个元素 70 作为 pivot

初始：[10, 80, 30, 90, 40, 50, 70]
                  ↑              ↑
                 j               pivot=70
                 i=-1

j=0: arr[0]=10 < 70
     i=0, 交换 arr[0] 和 arr[0]（不变）
     [10, 80, 30, 90, 40, 50, 70]
      ↑
      i

j=1: arr[1]=80 > 70
     不交换，j 继续
     [10, 80, 30, 90, 40, 50, 70]
          ↑
          j

j=2: arr[2]=30 < 70
     i=1, 交换 arr[1] 和 arr[2]
     [10, 30, 80, 90, 40, 50, 70]
               ↑
               i

j=3: arr[3]=90 > 70
     不交换
     [10, 30, 80, 90, 40, 50, 70]
                    ↑
                    j

j=4: arr[4]=40 < 70
     i=2, 交换 arr[2] 和 arr[4]
     [10, 30, 40, 90, 80, 50, 70]
                    ↑
                    i

j=5: arr[5]=50 < 70
     i=3, 交换 arr[3] 和 arr[5]
     [10, 30, 40, 50, 80, 90, 70]
                         ↑
                         i

最后：交换 arr[i+1] 和 pivot
     交换 arr[4] 和 arr[6]
     [10, 30, 40, 50, 70, 90, 80]
                      ↑
                   pivot 归位！

现在：
左边 [10, 30, 40, 50] 都 ≤ 70
右边 [90, 80] 都 ≥ 70

递归处理左右两边...
最终：[10, 30, 40, 50, 70, 80, 90] ✓
```

## 💻 代码实现

```javascript
/**
 * 快速排序
 * 
 * @param {number[]} arr - 待排序数组
 * @param {number} low - 起始索引
 * @param {number} high - 结束索引
 */
function quickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        // 分区操作
        const pivotIndex = partition(arr, low, high);
        
        // 递归排序左右两部分
        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }
    
    return arr;
}

/**
 * 分区操作（Lomuto 方案）
 * 选最后一个元素作为 pivot
 */
function partition(arr, low, high) {
    const pivot = arr[high];  // 选最后一个作为 pivot
    let i = low - 1;          // i 指向小于 pivot 区域的最后一个元素
    
    for (let j = low; j < high; j++) {
        // 如果当前元素小于等于 pivot
        if (arr[j] <= pivot) {
            i++;
            // 把它交换到小于区域
            [arr[i], arr[j]] = [arr[j], arr[i]];
        }
    }
    
    // 把 pivot 放到中间（小于区和大于区之间）
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
    
    return i + 1;  // 返回 pivot 的位置
}

// 🧪 测试
const arr = [10, 80, 30, 90, 40, 50, 70];
console.log('排序前:', arr);
quickSort(arr);
console.log('排序后:', arr);
// 输出：[10, 30, 40, 50, 70, 80, 90] ✓
```

### ❗ 常见错误

```javascript
// ❌ 错误 1：pivot 选择不对
const pivot = arr[0];  // 选了第一个
// 但分区逻辑是按最后一个写的，会出错！

// ❌ 错误 2：忘记交换 pivot
[arr[i], arr[high]] = [arr[high], arr[i]];  // 漏了这行
// pivot 还在最后，没有归位！

// ❌ 错误 3：递归条件错了
if (low <= high) {  // 应该是 <
    // 会导致无限递归！
}

// ❌ 错误 4：返回值错了
return i;  // 应该是 i + 1
// pivot 在 i+1 位置，不是 i！
```

## ⏱️ 复杂度分析（重点！）

### 最好情况（运气爆棚）

```
每次都正好选到中位数：

[10, 30, 40, 50, 70, 80, 90]
         ↑
       pivot

左边 3 个，右边 3 个，完美平分！

T(n) = 2T(n/2) + O(n)
根据主定理：T(n) = O(n log n)

就像归并排序一样快！⚡
```

### 最坏情况（倒霉透顶）

```
每次都是最小或最大的元素：

已排序数组：[1, 2, 3, 4, 5, 6, 7]
选最后一个：↑
            pivot

左边 6 个，右边 0 个，极不平衡！

T(n) = T(n-1) + O(n)
展开：T(n) = O(n²)

比插入排序还慢！❌
```

### 平均情况（正常发挥）

```
随机情况下，有时平衡，有时不平衡

但可以证明：期望时间复杂度 = O(n log n)

数据对比：
n=1000:
最好：O(n log n) ≈ 10000 次
最坏：O(n²) = 1000000 次
平均：O(n log n) ≈ 10000 次

所以平均来说很快！✨
```

## 🎲 随机化快速排序（避免最坏情况）

### 核心思想

```
既然最坏情况是选的 pivot 不好
那就随机选 pivot！

这样就不会被针对
期望时间复杂度 = O(n log n)
```

### 代码实现

```javascript
/**
 * 随机化快速排序
 */
function randomizedQuickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        // 随机选择 pivot
        const randomPivotIndex = Math.floor(Math.random() * (high - low + 1)) + low;
        
        // 把 pivot 换到最后
        [arr[randomPivotIndex], arr[high]] = [arr[high], arr[randomPivotIndex]];
        
        // 正常的快速排序
        const pivotIndex = partition(arr, low, high);
        
        randomizedQuickSort(arr, low, pivotIndex - 1);
        randomizedQuickSort(arr, pivotIndex + 1, high);
    }
    
    return arr;
}

// ⏱️ 期望时间复杂度：O(n log n)
// 避免了最坏情况 O(n²)
```
