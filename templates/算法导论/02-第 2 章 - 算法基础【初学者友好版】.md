## 🚶 一步一步跟着做

### 示例：归并排序 [38, 27, 43, 3, 9, 82, 10]

#### 第 1 阶段：分解（Divide）

```
初始：[38, 27, 43, 3, 9, 82, 10]
            ↓ 从中间切开
左：[38, 27, 43, 3]  右：[9, 82, 10]
      ↓                   ↓
[38, 27] [43, 3]    [9, 82] [10]
  ↓        ↓          ↓
[38][27] [43][3]   [9][82]

现在都是单个元素了！可以开始合并。
```

#### 第 2 阶段：合并（Combine）

```
合并 [38] 和 [27]：
左边：38，右边：27
27 < 38，先拿 27，再拿 38
→ [27, 38] ✓

合并 [43] 和 [3]：
左边：43，右边：3
3 < 43，先拿 3，再拿 43
→ [3, 43] ✓

合并 [9] 和 [82]：
左边：9，右边：82
9 < 82，先拿 9，再拿 82
→ [9, 82] ✓

[10] 单独，不用合并
→ [10]
```

#### 第 3 阶段：继续合并

```
合并 [27, 38] 和 [3, 43]：
左：[27, 38]  右：[3, 43]

步骤：
1. 比：27 vs 3 → 拿 3     结果：[3]
2. 比：27 vs 43 → 拿 27   结果：[3, 27]
3. 比：38 vs 43 → 拿 38   结果：[3, 27, 38]
4. 右还有 43 → 拿 43      结果：[3, 27, 38, 43] ✓

合并 [9, 82] 和 [10]：
左：[9, 82]  右：[10]

步骤：
1. 比：9 vs 10 → 拿 9     结果：[9]
2. 比：82 vs 10 → 拿 10   结果：[9, 10]
3. 左还有 82 → 拿 82      结果：[9, 10, 82] ✓
```

#### 最终合并

```
合并 [3, 27, 38, 43] 和 [9, 10, 82]：

左：[3, 27, 38, 43]
右：[9, 10, 82]

1. 3 vs 9 → 拿 3      [3]
2. 27 vs 9 → 拿 9     [3, 9]
3. 27 vs 10 → 拿 10   [3, 9, 10]
4. 27 vs 82 → 拿 27   [3, 9, 10, 27]
5. 38 vs 82 → 拿 38   [3, 9, 10, 27, 38]
6. 43 vs 82 → 拿 43   [3, 9, 10, 27, 38, 43]
7. 右还有 82 → 拿 82  [3, 9, 10, 27, 38, 43, 82] ✓

完成！🎉
```

## 💻 代码实现（逐行解释）

```javascript
/**
 * 归并排序 - 分治法的经典例子
 * 
 * @param {number[]} arr - 待排序数组
 * @returns {number[]} 排好序的数组
 * 
 * 💡 记忆口诀：
 * "一切为二，分别排序，再合起来"
 */
function mergeSort(arr) {
    // 基本情况：只有一个元素或空数组
    // 一个元素天然有序，不需要排序
    if (arr.length <= 1) {
        return arr;
    }
    
    // 1. 分解（Divide）：从中间切开
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);      // 左半部分
    const right = arr.slice(mid);        // 右半部分
    
    // 2. 解决（Conquer）：递归排序左右两边
    // 这里最巧妙！假设 mergeSort 已经能排序
    const sortedLeft = mergeSort(left);   // 排好左边
    const sortedRight = mergeSort(right); // 排好右边
    
    // 3. 合并（Combine）：把两个有序数组合成一个
    return merge(sortedLeft, sortedRight);
}

/**
 * 合并两个有序数组
 * 就像合并两摞已经排好序的扑克牌
 * 
 * @param {number[]} left - 左边的有序数组
 * @param {number[]} right - 右边的有序数组
 * @returns {number[]} 合并后的有序数组
 */
function merge(left, right) {
    const result = [];  // 存放合并结果
    let i = 0;  // left 的指针
    let j = 0;  // right 的指针
    
    // 两边都还有牌时，比较着拿
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            // 左边的小，拿左边的
            result.push(left[i]);
            i++;  // 左指针往前走
        } else {
            // 右边的小，拿右边的
            result.push(right[j]);
            j++;  // 右指针往前走
        }
    }
    
    // ⚠️ 注意：可能有一边还没拿完
    // 把剩下的直接加到结果后面
    
    // 左边剩余的（如果有的话）
    while (i < left.length) {
        result.push(left[i]);
        i++;
    }
    
    // 右边剩余的（如果有的话）
    while (j < right.length) {
        result.push(right[j]);
        j++;
    }
    
    return result;  // 返回完整的结果
}

// 🧪 测试一下
const arr = [38, 27, 43, 3, 9, 82, 10];
console.log('排序前:', arr);
const sorted = mergeSort(arr);
console.log('排序后:', sorted);
// 输出：[3, 9, 10, 27, 38, 43, 82] ✓
```

### ❗ 常见错误提醒

```javascript
// ❌ 错误 1：忘记处理剩余元素
function wrongMerge(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i++]);
        } else {
            result.push(right[j++]);
        }
    }
    
    return result;  // ❌ 错了！可能漏掉元素
    // 比如 left=[1,3,5], right=[2]
    // 会漏掉 3 和 5！
}

// ❌ 错误 2：没有基本情况（无限递归）
function wrongMergeSort(arr) {
    // 缺少这个判断就死循环了！
    // if (arr.length <= 1) return arr;
    
    const mid = Math.floor(arr.length / 2);
    const left = wrongMergeSort(arr.slice(0, mid));
    const right = wrongMergeSort(arr.slice(mid));
    return merge(left, right);
}

// ❌ 错误 3：合并时比较错了
if (left[i] > right[j]) {  // 应该是 <=
    result.push(left[i]);  // 这样会排成降序
}

// ✅ 正确版本（对照上面的代码）

## ⏱️ 到底有多快？

### 直观理解 O(n log n)

**画一棵树来理解：**

```
对 8 个数排序：[38, 27, 43, 3, 9, 82, 10, 5]

第 0 层：[38, 27, 43, 3, 9, 82, 10, 5]  ← 8 个数
         ↓ 切一刀
第 1 层：[38, 27, 43, 3] [9, 82, 10, 5]  ← 每边 4 个
         ↓ 各切一刀
第 2 层：[38, 27] [43, 3] [9, 82] [10, 5]  ← 每边 2 个
         ↓ 各切一刀
第 3 层：[38][27][43][3][9][82][10][5]  ← 单个元素

层数：log₂8 = 3 层

每层都要合并一遍，总共 8 个元素
总工作量：8 × 3 = 24 次操作

一般化：n × log₂n = O(n log n)
```

### 对比插入排序

```
数据量 n = 100:
━━━━━━━━━━━━━━━━━━━━━━━━━━
插入排序：O(n²)
100² = 10,000 次操作
耗时：0.01 秒

归并排序：O(n log n)
100 × log₂100 ≈ 100 × 7 = 700 次
耗时：0.0007 秒

快了 14 倍！⚡
━━━━━━━━━━━━━━━━━━━━━━━━━━

10000² = 100,000,000 次
耗时：100 秒  ❌ 等到花儿都谢了

归并排序：O(n log n)
10000 × log₂10000 ≈ 10000 × 13 = 130,000 次
耗时：0.13 秒  ⚡ 秒杀！

快了 769 倍！！！🚀
━━━━━━━━━━━━━━━━━━━━━━━━━━

### 常见问题

**Q1: 归并排序和插入排序比有什么优缺点？**

```
✅ 标准答案：

优点：
1. 快！O(n log n) vs O(n²)
   大数据量时快几百倍
   
2. 稳定排序
   相等元素不会交换位置
   
3. 适合链表排序
   不需要随机访问

缺点：
1. 需要额外 O(n) 空间
   插入排序只需要 O(1)
   
2. 小数据量优势不明显
   n < 30 时差不多快
```

**Q2: 归并排序适合什么场景？**

```
✅ 标准答案：
1. 大数据量排序（n > 1000）
2. 要求稳定性（如多关键字排序）
3. 链表排序（不需要额外空间）
4. 外部排序（数据太大内存放不下）

实际应用：
- Java 的 Arrays.sort() 对对象排序时用归并
- 数据库 ORDER BY 用归并
- Git 的 diff 算法用归并思想
```
