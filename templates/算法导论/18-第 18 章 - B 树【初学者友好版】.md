## 🔑 核心概念

### 什么是 B 树？

```
B 树的特点（和二叉搜索树对比）：

二叉搜索树：
- 每个节点 1 个值
- 最多 2 个孩子
- 树容易不平衡

B 树：
- 每个节点多个值（比如 3-5 个）
- 多个孩子（比如 4-6 个）
- 自动保持平衡
- 所有叶子在同一层 ✨

关键参数：最小度数 t
- 每个节点至少有 t-1 个关键字
- 每个节点至多有 2t-1 个关键字
- 每个节点至少有 t 个孩子（除了叶子）
```

### B 树的规则（背下来！）

```javascript
/**
 * B 树的性质（t 是最小度数，t ≥ 2）：

1. 根节点：
   - 至少有 1 个关键字（除非是空树）
   - 至少有 2 个孩子（除非是叶子）

2. 非根节点：
   - 至少有 t-1 个关键字
   - 至少有 t 个孩子

3. 所有节点：
   - 至多有 2t-1 个关键字
   - 至多有 2t 个孩子

4. 所有叶子：
   - 都在同一层

例子：t = 2 的 B 树（2-3-4 树）
- 每个节点：1-3 个关键字
- 每个节点：2-4 个孩子
*/
```

## 🛠️ B 树的基本操作

### 节点定义

```javascript
/**
 * B 树节点
 */
class BTreeNode {
    constructor(t, leaf) {
        this.t = t;          // 最小度数
        this.leaf = leaf;    // 是否是叶子
        this.keys = [];      // 关键字数组
        this.children = [];  // 子节点数组
    }
    
    /**
     * 判断节点是否已满
     */
    isFull() {
        return this.keys.length === 2 * this.t - 1;
    }
}
```

### 查找操作

```javascript
/**
 * B 树查找
 * 
 * @param {BTreeNode} node - 当前节点
 * @param {number} key - 目标值
 * @returns {Object|null} {node, index} 或 null
 */
function bTreeSearch(node, key) {
    let i = 0;
    
    // 找到第一个大于等于 key 的位置
    while (i < node.keys.length && key > node.keys[i]) {
        i++;
    }
    
    // 如果找到了
    if (i < node.keys.length && key === node.keys[i]) {
        return { node, index: i };
    }
    
    // 如果是叶子，没找到
    if (node.leaf) {
        return null;
    }
    
    // 否则在子树中继续找
    return bTreeSearch(node.children[i], key);
}

// ⏱️ 时间复杂度：O(log n)
// 因为树高 h = O(log_t n)
```

### 插入操作（难点）

```javascript
/**
 * B 树插入
 * 
 * @param {BTreeNode} root - 根节点
 * @param {number} key - 要插入的值
 * @param {number} t - 最小度数
 * @returns {BTreeNode} 新的根节点
 */
function bTreeInsert(root, key, t) {
    // 如果树为空，创建新节点
    if (root === null) {
        return new BTreeNode(t, true);
    }
    
    // 如果根节点满了，需要分裂
    if (root.isFull()) {
        const newRoot = new BTreeNode(t, false);
        newRoot.children[0] = root;
        splitChild(newRoot, 0, t);
        return insertNonFull(newRoot, key, t);
    }
    
    return insertNonFull(root, key, t);
}

/**
 * 在非满节点中插入
 */
function insertNonFull(node, key, t) {
    let i = node.keys.length - 1;
    
    if (node.leaf) {
        // 叶子节点：直接插入并排序
        node.keys.push(key);
        node.keys.sort((a, b) => a - b);
    } else {
        // 非叶子节点：找到合适的子树
        while (i >= 0 && key < node.keys[i]) {
            i--;
        }
        i++;
        
        // 如果子节点满了，先分裂
        if (node.children[i].isFull()) {
            splitChild(node, i, t);
            
            // 确定往哪个子树插
            if (key > node.keys[i]) {
                i++;
            }
        }
        
        insertNonFull(node.children[i], key, t);
    }
    
    return node;
}

/**
 * 分裂子节点
 */
function splitChild(parent, index, t) {
    const child = parent.children[index];
    const newNode = new BTreeNode(t, child.leaf);
    
    // 把 child 的后一半移到 newNode
    const midIndex = t - 1;
    const midKey = child.keys[midIndex];
    
    // keys 分裂
    newNode.keys = child.keys.splice(midIndex + 1, t - 1);
    child.keys.splice(midIndex, 1);  // 中间的上移
    
    // children 分裂（如果不是叶子）
    if (!child.leaf) {
        newNode.children = child.children.splice(midIndex + 1, t);
    }
    
    // 把 midKey 插入到 parent
    parent.keys.splice(index, 0, midKey);
    parent.children.splice(index + 1, 0, newNode);
}
```

### 图解分裂过程

```
分裂前（t=2）：
父节点：[50]
        /   \
子节点：[20,30,40]  ← 满了（2t-1=3）

分裂后：
        [30]      ← 中间的上移
       /  |  \
[20]  [50]  [40]  ← 分成两个节点

父节点现在有 3 个子节点 ✓
```

## 🎯 B 树 vs 红黑树

### 性能对比

```
数据量：n = 1,000,000

红黑树（二叉树）：
- 树高：log₂n ≈ 20
- 磁盘 I/O：20 次 ❌

B 树（t=100）：
- 树高：log₁₀₀n ≈ 3
- 磁盘 I/O：3 次 ✅

快了 7 倍！🚀
```

### 为什么数据库用 B 树？

```
关键原因：减少磁盘 I/O！

红黑树：
- 节点小（1 个值）
- 树很高
- 每次 I/O 读一个节点
- 要 I/O 很多次

B 树：
- 节点大（几百个值）
- 树很矮
- 每次 I/O 读整个节点（一页）
- I/O 次数很少

数据库瓶颈在磁盘 I/O，
所以 B 树是最佳选择！✨
```

## 🎯 B+ 树（数据库实际用的）

### B+ 树 vs B 树

```
B 树：
- 每个节点都存数据
- 查找可能在中间节点结束

B+ 树（改进）：
- 只有叶子节点存数据
- 所有数据在叶子层排好序
- 叶子之间用链表连接
- 非叶子节点只存索引

优点：
✓ 范围查询超快（叶子是链表）
✓ 查询稳定（都要到叶子）
✓ 更适合外部存储
```

### B+ 树的结构

```
非叶子节点（索引层）：
[10, 20, 30]
 ↓    ↓    ↓
叶子节点（数据层）：
[1-9] → [11-19] → [21-29] → [31-...]
 ↑                      ↑
链表连接，方便范围查询
```

### 常见问题

**Q: 为什么 MySQL 用 B+ 树做索引？**

```
✅ 标准答案：

B+ 树的优点：

1. 磁盘 I/O 少
   - 树很矮（log₁₀₀n）
   - 每次 I/O 读一整页
   
2. 范围查询快
   - 叶子用链表连接
   - SELECT * FROM table WHERE id > 10
   - 直接遍历链表就行
   
3. 查询稳定
   - 所有查询都要到叶子
   - 没有最好最坏情况

4. 适合批量加载
   - 顺序写入叶子节点
   - 性能好

所以数据库索引首选 B+ 树！✨
```

**Q: B 树的高度怎么计算？**

```
✅ 标准答案：

对于 n 个关键字，最小度数 t 的 B 树：

树高 h ≤ log_t((n+1)/2)

例子：
n = 1,000,000, t = 100

h ≤ log₁₀₀(500,000) ≈ 3

所以最多 3-4 层！
查数据只需要 3-4 次磁盘 I/O！⚡
```
