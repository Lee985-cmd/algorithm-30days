# 📚 算法导论 Part VII - 第 25 章：文本搜索的艺术 - 字符串匹配算法

## 25.1 什么是字符串匹配？

### 生活中的例子

**场景 1：Ctrl+F 查找**

```
你在看一篇 10 万字的论文

想找"人工智能"这个词出现的所有位置

怎么做？

笨方法：
从第一个字开始，一个一个比对
看到"人工"就继续看是不是"智能"
一直看到文章结束

这就是朴素匹配算法！
```

**场景 2：敏感词过滤**

```
网站有 100 万个敏感词库

用户发了一条评论："今天天气真好"

要检查这句话里有没有敏感词

怎么做？

用字符串匹配算法！
快速扫描整句话
发现敏感词就屏蔽或替换
```

### 说人话定义

```
字符串匹配 = 在长文本中找短模式

输入：
- 文本串 T（很长）
- 模式串 P（较短）

输出：
- P 在 T 中出现的所有位置

例如：
T = "ababcabcacbab"
P = "abc"

答案：
P 在 T 的位置 2 和 6 出现（从 0 开始）
```

## 25.2 朴素算法 - 暴力匹配

### 核心思想

```
生活比喻：逐个尝试

朴素算法的做法：
1️⃣ 从 T 的第一个字符开始
2️⃣ 逐字符比较 P
3️⃣ 如果完全匹配，记录位置
4️⃣ 如果不匹配，从 T 的下一个位置重新开始
5️⃣ 重复 2-4，直到 T 结束

最简单，但可能很慢！
```

### 代码实现

```javascript
/**
 * 朴素字符串匹配算法
 */
function naiveStringMatch(text, pattern) {
    const n = text.length;
    const m = pattern.length;
    const positions = [];
    
    // 尝试每个可能的起始位置
    for (let i = 0; i <= n - m; i++) {
        let j = 0;
        
        // 逐字符比较
        while (j < m && text[i + j] === pattern[j]) {
            j++;
        }
        
        // 如果完全匹配
        if (j === m) {
            positions.push(i);
        }
    }
    
    return positions;
}

// 测试
const text = "ababcabcacbab";
const pattern = "abc";
console.log(naiveStringMatch(text, pattern));
// 输出：[2, 6] ✅
```

```
═══════════════════════════════════

```

### 时间复杂度

```
最好情况：O(n)
→ 每次都很快发现不匹配

最坏情况：O((n-m+1) × m) ≈ O(nm)
→ 例如：T="aaaa...", P="aaab"
→ 几乎每次都要比较到最后一个字符才失败

太慢了！❌
```

## 25.3 KMP 算法 - 聪明的回溯

### 核心思想

```
生活比喻：不要走回头路

KMP 算法（Knuth-Morris-Pratt）的做法：
1️⃣ 预处理模式串 P，计算一个"部分匹配表"
2️⃣ 匹配时，如果失败了，不用回到起点
3️⃣ 根据部分匹配表，跳过一些明显不会匹配的位置
4️⃣ 继续比较

关键洞察：
已经比较过的部分包含了信息
可以利用这些信息避免重复比较

就像：
你背单词 "apple"
背到 'l' 忘了
下次不用从 'a' 开始
直接从 'p' 继续！
```

### next 数组（部分匹配表）

```
什么是 next 数组？

next[i] 表示：
模式串 P[0...i] 的最长公共前后缀的长度

说人话：
P 的前面部分和后面部分，最长有多少个字符是一样的

例如：P = "abcabc"
前缀：a, ab, abc, abca, abcab
后缀：c, bc, abc, cabc, bcabc

公共前后缀："abc"（长度 3）
所以 next[5] = 3

这个信息告诉我们：
如果在位置 5 失配
可以直接跳到位置 3 继续比较
不用从 0 开始！
```

### 计算 next 数组

```javascript
/**
 * 计算 KMP 的 next 数组
 */
function computeNext(pattern) {
    const m = pattern.length;
    const next = new Array(m).fill(0);
    
    let len = 0;  // 当前最长公共前后缀的长度
    let i = 1;
    
    while (i < m) {
        if (pattern[i] === pattern[len]) {
            // 匹配，长度 +1
            len++;
            next[i] = len;
            i++;
        } else {
            if (len !== 0) {
                // 不匹配，回退到上一个位置
                len = next[len - 1];
            } else {
                // 已经是 0 了，next[i]=0
                next[i] = 0;
                i++;
            }
        }
    }
    
    return next;
}

// 测试
const P = "abcabc";
console.log(computeNext(P));
// 输出：[0, 0, 0, 1, 2, 3]
```

### KMP 完整代码

```javascript
/**
 * KMP 字符串匹配算法
 */
function kmpSearch(text, pattern) {
    const n = text.length;
    const m = pattern.length;
    const positions = [];
    
    // 预处理：计算 next 数组
    const next = computeNext(pattern);
    
    let i = 0;  // text 的索引
    let j = 0;  // pattern 的索引
    
    while (i < n) {
        if (text[i] === pattern[j]) {
            // 匹配，继续
            i++;
            j++;
            
            // 完全匹配
            if (j === m) {
                positions.push(i - j);
                j = next[j - 1];  // 继续找下一个匹配
            }
        } else {
            if (j !== 0) {
                // 利用 next 数组回退
                j = next[j - 1];
            } else {
                // j 已经是 0，只能移动 i
                i++;
            }
        }
    }
    
    return positions;
}

// 测试
const text = "ababcabcacbab";
const pattern = "abc";
console.log(kmpSearch(text, pattern));
// 输出：[2, 6] ✅
```

### 时间复杂度

```
预处理（计算 next）：O(m)
匹配过程：O(n)

总时间：O(n + m)

比朴素算法的 O(nm) 快多了！✅

空间复杂度：O(m)（存储 next 数组）
```

## 25.4 Rabin-Karp 算法 - 哈希过滤

### 核心思想

```
生活比喻：指纹识别

Rabin-Karp 算法的做法：
1️⃣ 计算模式串 P 的哈希值
2️⃣ 计算文本串 T 中每个长度为 m 的子串的哈希值
3️⃣ 如果哈希值相同，再逐字符比较（防止哈希冲突）
4️⃣ 如果不同，直接跳过

关键优化：
用"滚动哈希"快速计算下一个子串的哈希值
不用重新计算！

就像：
给每个单词拍张照（哈希）
先比对照片
照片一样再比对真人
```

### 代码实现

```javascript
/**
 * Rabin-Karp 字符串匹配算法
 */
function rabinKarp(text, pattern, prime = 101) {
    const n = text.length;
    const m = pattern.length;
    const positions = [];
    
    if (m > n) return positions;
    
    const BASE = 256;  // 字符集大小（ASCII）
    
    // 计算 h = BASE^(m-1) % prime
    let h = 1;
    for (let i = 0; i < m - 1; i++) {
        h = (h * BASE) % prime;
    }
    
    // 计算模式串的哈希值
    let patternHash = 0;
    for (let i = 0; i < m; i++) {
        patternHash = (patternHash * BASE + pattern.charCodeAt(i)) % prime;
    }
    
    // 计算文本串第一个子串的哈希值
    let textHash = 0;
    for (let i = 0; i < m; i++) {
        textHash = (textHash * BASE + text.charCodeAt(i)) % prime;
    }
    
    // 滑动窗口比较
    for (let i = 0; i <= n - m; i++) {
        // 如果哈希值相同，逐字符比较
        if (patternHash === textHash) {
            let match = true;
            for (let j = 0; j < m; j++) {
                if (text[i + j] !== pattern[j]) {
                    match = false;
                    break;
                }
            }
            
            if (match) {
                positions.push(i);
            }
        }
        
        // 滚动哈希：计算下一个子串的哈希值
        if (i < n - m) {
            // 去掉首字符，加上尾字符
            textHash = (BASE * (textHash - text.charCodeAt(i) * h) + 
                       text.charCodeAt(i + m)) % prime;
            
            // 处理负数
            if (textHash < 0) {
                textHash += prime;
            }
        }
    }
    
    return positions;
}

// 测试
const text = "ababcabcacbab";
const pattern = "abc";
console.log(rabinKarp(text, pattern));
// 输出：[2, 6] ✅
```

### 时间复杂度

```
最好情况：O(n + m)
平均情况：O(n + m)
最坏情况：O(nm)
→ 当有很多哈希冲突时

空间复杂度：O(1)
```

## 25.5 三种算法对比

| 特性 | 朴素算法 | KMP | Rabin-Karp |
|------|---------|-----|------------|
| 预处理时间 | 无 | O(m) | O(m) |
| 匹配时间 | O(nm) | O(n+m) | O(n+m) 平均 |
| 最坏时间 | O(nm) | O(n+m) | O(nm) |
| 空间复杂度 | O(1) | O(m) | O(1) |
| 实现难度 | 简单 | 中等 | 中等 |
| 适用场景 | 短文本 | 长文本、重复模式 | 多模式匹配 |
| 优点 | 简单易懂 | 稳定高效 | 可扩展到多模式 |
| 缺点 | 慢 | 需要理解 next 数组 | 可能有哈希冲突 |

```javascript
/**
 * 敏感词过滤
 */
function filterSensitiveWords(text, sensitiveWords) {
    let result = text;
    
    for (let word of sensitiveWords) {
        const regex = new RegExp(word, 'g');
        result = result.replace(regex, '*'.repeat(word.length));
    }
    
    return result;
}

// 测试
const text = "今天天气真好，我想去骂人";
```
