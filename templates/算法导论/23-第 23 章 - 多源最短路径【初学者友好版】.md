## 🔑 核心概念

### 什么是多源最短路径？

```
问题：给定带权图 G=(V,E)，求任意两点 u,v 之间的最短路径

vs 单源最短路径：

单源（Dijkstra）：从一个点出发到所有点
→ 跑 n 次 Dijkstra = O(n³)

多源（Floyd）：任意两点之间
→ 一次 Floyd = O(n³)

但 Floyd 代码更简单！✨
```

## 🎯 Floyd-Warshall 算法

### 核心思想（动态规划）

```javascript
/**
 * Floyd-Warshall 算法
 * 
 * @param {Array} graph - 邻接矩阵（graph[i][j] = i 到 j 的距离）
 * @returns {Array} dist - 所有点对的最短距离
 */
function floydWarshall(graph) {
    const n = graph.length;
    
    // 初始化距离矩阵
    const dist = JSON.parse(JSON.stringify(graph));
    
    // 三重循环（背下来！）
    for (let k = 0; k < n; k++) {           // 枚举中间点
        for (let i = 0; i < n; i++) {       // 枚举起点
            for (let j = 0; j < n; j++) {   // 枚举终点
                // 状态转移方程：
                // 从 i 到 j，可以直达，也可以经过 k
                // 取较小的那个
                dist[i][j] = Math.min(
                    dist[i][j],              // 不经过 k
                    dist[i][k] + dist[k][j]  // 经过 k
                );
            }
        }
    }
    
    return dist;
}

// ⏱️ 时间复杂度：O(V³)
// 空间复杂度：O(V²)
```

### 一步一步演示

```
初始图（4 个城市）：
     0  1  2  3
0 [  0, 5, ∞, 10]
1 [ ∞,  0, 3, ∞ ]
2 [ ∞, ∞,  0,  1]
3 [ ∞, ∞, ∞,  0 ]

∞ 表示没有直达路

k=0（允许经过节点 0）:
检查所有 i→j 是否可以通过 0 变短
i=1, j=2: dist[1][0]+dist[0][2] = ∞+∞ = ∞ （不变）
i=2, j=3: dist[2][0]+dist[0][3] = ∞+10 = ∞ （不变）
... 没有能更新的

k=1（允许经过节点 1）:
i=0, j=2: dist[0][1]+dist[1][2] = 5+3 = 8 < ∞ ✓
          dist[0][2] = 8
i=0, j=3: dist[0][1]+dist[1][3] = 5+∞ = ∞ （不变）
...

k=2（允许经过节点 2）:
i=0, j=3: dist[0][2]+dist[2][3] = 8+1 = 9 < 10 ✓
          dist[0][3] = 9
i=1, j=3: dist[1][2]+dist[2][3] = 3+1 = 4 < ∞ ✓
          dist[1][3] = 4

k=3（允许经过节点 3）:
没有能更新的了

最终结果：
     0  1  2  3
0 [  0, 5, 8, 9]
1 [ ∞,  0, 3, 4]
2 [ ∞, ∞,  0, 1]
3 [ ∞, ∞, ∞,  0 ]

解释：
0→2: 0→1→2 = 5+3 = 8
0→3: 0→1→2→3 = 5+3+1 = 9
1→3: 1→2→3 = 3+1 = 4
```

### 代码详解

```javascript
/**
 * 完整版本（包含路径重建）
 */
function floydWithPath(graph) {
    const n = graph.length;
    
    // 距离矩阵
    const dist = JSON.parse(JSON.stringify(graph));
    
    // 路径矩阵（记录下一个节点）
    const next = Array.from({ length: n }, () => new Array(n).fill(null));
    
    // 初始化 next 矩阵
    for (let i = 0; i < n; i++) {
        for (let j = 0; j < n; j++) {
            if (graph[i][j] !== Infinity && i !== j) {
                next[i][j] = j;  // 直接到 j
            }
        }
    }
    
    // Floyd 主算法
    for (let k = 0; k < n; k++) {
        for (let i = 0; i < n; i++) {
            for (let j = 0; j < n; j++) {
                if (dist[i][k] + dist[k][j] < dist[i][j]) {
                    dist[i][j] = dist[i][k] + dist[k][j];
                    next[i][j] = next[i][k];  // 更新路径
                }
            }
        }
    }
    
    return { dist, next };
}

/**
 * 重建从 i 到 j 的路径
 */
function reconstructPath(next, i, j) {
    if (next[i][j] === null) return [];
    
    const path = [i];
    while (i !== j) {
        i = next[i][j];
        path.push(i);
    }
    
    return path;
}

// 🧪 测试
const graph = [
    [0, 5, Infinity, 10],
    [Infinity, 0, 3, Infinity],
    [Infinity, Infinity, 0, 1],
    [Infinity, Infinity, Infinity, 0]
];

const result = floydWithPath(graph);
console.log('最短距离:', result.dist);
console.log('0 到 3 的路径:', reconstructPath(result.next, 0, 3));
// 输出：[0, 1, 2, 3]
```

## ⏱️ 复杂度分析

### 时间复杂度

```
三重循环：
for k: n 次
  for i: n 次
    for j: n 次
      常数操作

总时间：n × n × n = n³

n=100: 100 万次操作 → 很快 ✓
n=1000: 10 亿次操作 → 有点慢 ❌

所以 Floyd 适合小规模图（n < 500）
```

### 空间复杂度

```
需要两个 n×n 矩阵：
- dist 矩阵：O(n²)
- next 矩阵：O(n²)

总空间：O(n²)
```

## 🎯 实际应用

### 应用 1: 社交网络六度分隔

```javascript
/**
 * 计算两个人之间的最少介绍人数
 */
function sixDegrees(adjacencyMatrix) {
    const dist = floydWarshall(adjacencyMatrix);
    
    // 找出最远的两个人
    let maxDist = 0;
    let farthestPair = [0, 0];
    
    for (let i = 0; i < dist.length; i++) {
        for (let j = i + 1; j < dist.length; j++) {
            if (dist[i][j] < Infinity && dist[i][j] > maxDist) {
                maxDist = dist[i][j];
                farthestPair = [i, j];
            }
        }
    }
    
    return {
        maxDistance: maxDist,
        pair: farthestPair,
        message: `最多需要${maxDist}个人介绍`
    };
}
```

### 应用 2: 传递闭包（判断连通性）

```javascript
/**
 * Warshall 算法（Floyd 的变种）
 * 判断任意两点是否连通
 */
function warshall(graph) {
    const n = graph.length;
    const reach = JSON.parse(JSON.stringify(graph));
    
    for (let k = 0; k < n; k++) {
        for (let i = 0; i < n; i++) {
            for (let j = 0; j < n; j++) {
                // 如果 i→k 且 k→j，则 i→j 可达
                reach[i][j] = reach[i][j] || (reach[i][k] && reach[k][j]);
            }
        }
    }
    
    return reach;
}
```

### 常见问题

**Q: Floyd 和 Dijkstra 哪个更好？**

```
✅ 标准答案：

看情况！

Floyd 优点：
1. 代码超级简单（就三重循环）
2. 一次算出所有点对
3. 可以处理负权边（不能有负权环）

Floyd 缺点：
1. O(n³) 太慢，n 大了不行
2. 稠密图不划算

Dijkstra 优点：
1. 单源最快 O(E log V)
2. 适合稀疏图

选择建议：
- 小图（n<500）、稠密 → Floyd
- 大图、稀疏、单源 → Dijkstra
```

✓ 适合小规模稠密图
