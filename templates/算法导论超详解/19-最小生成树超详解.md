## 第 21 章：最省钱修路 - 最小生成树算法

### 21.1 什么是最小生成树？

#### 生活中的例子

**场景 1：修路问题**

```
有 5 个村庄：A, B, C, D, E

现在要修路把所有村庄连接起来

可能的路线和成本：
A--B: 10 万
A--C: 6 万
B--C: 3 万
B--D: 8 万
C--D: 4 万
C--E: 7 万
D--E: 5 万

问：怎么修路，能让所有村庄连通且总成本最低？

这就是最小生成树问题！
```

**场景 2：网络布线**

```
公司有 10 台电脑
要拉网线把它们连起来

每个位置可以拉线
但拉线有成本（距离、材料等）

问：怎么拉线，能让所有电脑连通且用的线最少？

这也是最小生成树问题！
```

**说人话定义：**

```
最小生成树（Minimum Spanning Tree, MST）= 
用最少的边，让所有点连通，且总权重最小

输入：
- 一个连通的无向图
- 每条边有权重（成本、距离等）

输出：
- 一棵树（没有环）
- 包含所有顶点
- 边的总权重最小

特点：
✓ n 个顶点，一定有 n-1 条边
✓ 一定连通
✓ 一定没有环
✓ 总权重最小
```

### 21.2 Kruskal 算法 - 贪心思想

#### 核心思想

```
生活比喻：挑最便宜的买

Kruskal 算法的做法：
1. 把所有边按权重从小到大排序
2. 依次考虑每条边
3. 如果这条边连接的两个点还不连通，就选它！
4. 重复 2-3，直到选了 n-1 条边

就像买东西：
总是先挑最便宜的
只要有用（能连接两个不连通的部分）就买

这就是贪心算法！
```

#### 代码实现

```javascript
/**
 * Kruskal 算法 - 最小生成树
 * @param {number} n - 顶点数
 * @param {number[][]} edges - 边列表，每个元素是 [u, v, weight]
 * @returns {{mst: number[][], totalWeight: number}} - MST 的边和总权重
 */
function kruskal(n, edges) {
    // 第 1 步：按权重排序
    edges.sort((a, b) => a[2] - b[2]);
    
    // 第 2 步：用并查集维护连通性
    const uf = new UnionFind(n);
    const mst = [];  // 存储 MST 的边
    let totalWeight = 0;
    
    // 第 3 步：依次考虑每条边
    for (let [u, v, weight] of edges) {
        // 如果 u 和 v 还不连通
        if (!uf.connected(u, v)) {
            // 就选这条边
            uf.union(u, v);
            mst.push([u, v, weight]);
            totalWeight += weight;
            
            // 如果已经选了 n-1 条边，结束
            if (mst.length === n - 1) {
                break;
            }
        }
    }
    
    return { mst, totalWeight };
}

// 并查集（前面已实现）
class UnionFind {
    constructor(n) {
        this.parent = new Array(n);
        this.rank = new Array(n).fill(0);
        for (let i = 0; i < n; i++) {
            this.parent[i] = i;
        }
    }
    
    find(x) {
        if (this.parent[x] === x) return x;
        const root = this.find(this.parent[x]);
        this.parent[x] = root;
        return root;
    }
    
    union(x, y) {
        const rootX = this.find(x);
        const rootY = this.find(y);
        if (rootX !== rootY) {
            if (this.rank[rootX] < this.rank[rootY]) {
                this.parent[rootX] = rootY;
            } else if (this.rank[rootX] > this.rank[rootY]) {
                this.parent[rootY] = rootX;
            } else {
                this.parent[rootY] = rootX;
                this.rank[rootX]++;
            }
        }
    }
    
    connected(x, y) {
        return this.find(x) === this.find(y);
    }
}

// 测试

const edges = [
    [0, 1, 10],
    [0, 2, 6],
    [1, 2, 3],
    [1, 3, 8],
    [2, 3, 4],
    [2, 4, 7],
    [3, 4, 5]
];

const result = kruskal(n, edges);
console.log('MST 的边:', result.mst);
console.log('总权重:', result.totalWeight);
```

### 21.3 Prim 算法 - 另一种贪心

#### 核心思想

```
生活比喻：滚雪球

Prim 算法的做法：
1. 随便选一个起点
2. 从已选的点出发，找最近的未选点
3. 把那个点加入
4. 重复 2-3，直到所有点都选了

就像滚雪球：
从一个点开始
越滚越大
每次都找最近的点加入

这也是贪心算法！
```

#### 代码实现

```javascript
/**
 * Prim 算法 - 最小生成树
 * @param {number[][]} graph - 邻接矩阵表示的图
 * @returns {{mst: number[][], totalWeight: number}}
 */
function prim(graph) {
    const n = graph.length;
    
    // minDist[i] 表示从已选集合到 i 的最小距离
    const minDist = new Array(n).fill(Infinity);
    const parent = new Array(n).fill(null);  // MST 的父节点
    const inMST = new Array(n).fill(false);  // 是否已选
    
    // 从顶点 0 开始
    minDist[0] = 0;
    
    let totalWeight = 0;
    const mst = [];
    
    // 需要选 n 个点
    for (let count = 0; count < n; count++) {
        // 第 1 步：选距离最小的未选点
        let u = -1;
        let minVal = Infinity;
        
        for (let v = 0; v < n; v++) {
            if (!inMST[v] && minDist[v] < minVal) {
                minVal = minDist[v];
                u = v;
            }
        }
        
        if (u === -1) break;  // 不连通
        
        // 第 2 步：把 u 加入 MST
        inMST[u] = true;
        totalWeight += minVal;
        
        // 如果有父节点，记录这条边
        if (parent[u] !== null) {
            mst.push([parent[u], u, graph[parent[u]][u]]);
        }
        
        // 第 3 步：更新 u 的邻居
        for (let v = 0; v < n; v++) {
            if (graph[u][v] > 0 && !inMST[v] && graph[u][v] < minDist[v]) {
                minDist[v] = graph[u][v];
                parent[v] = u;
            }
        }
    }
    
    return { mst, totalWeight };
}

// 测试
const graph = [
    [0, 10, 6, 0, 0],
    [10, 0, 3, 8, 0],
    [6, 3, 0, 4, 7],
    [0, 8, 4, 0, 5],
    [0, 0, 7, 5, 0]
];

const result = prim(graph);
console.log('MST 的边:', result.mst);
console.log('总权重:', result.totalWeight);
```

### 21.4 Kruskal vs Prim 对比

| 特性 | Kruskal | Prim |
|------|---------|------|
| 核心思想 | 贪心选边 | 贪心选点 |
| 数据结构 | 并查集 | 优先队列 |
| 时间复杂度 | O(E log E) | O(V²) 或 O(E log V) |
| 适用场景 | 稀疏图（边少） | 稠密图（边多） |
| 实现难度 | 简单 | 中等 |
| 是否需要排序 | ✓ 需要 | ✗ 不需要 |
| 起始点 | 不需要 | 需要任选一个 |

### 21.5 时间复杂度分析

#### Kruskal 算法

```
排序边：O(E log E)
并查集操作：O(E × α(V)) ≈ O(E)

总时间：O(E log E)

空间复杂度：O(V + E)
```

#### Prim 算法

```
朴素实现（上面的代码）：
- 外层循环：V 次
- 内层选最小：O(V)
- 更新邻居：O(V)
总时间：O(V²)

适合：稠密图（E ≈ V²）

优化实现（用优先队列/最小堆）：
- 每次取最小：O(log V)
- 总共处理 E 条边
总时间：O(E log V)

适合：稀疏图（E << V²）

空间复杂度：O(V)
```

### 21.7 完整代码模板（背诵版）

```javascript
/**
 * Kruskal 算法
 */
function kruskal(n, edges) {
    // 排序
    edges.sort((a, b) => a[2] - b[2]);
    
    const uf = new UnionFind(n);
    const mst = [];
    let totalWeight = 0;
    
    for (let [u, v, w] of edges) {
        if (!uf.connected(u, v)) {
            uf.union(u, v);
            mst.push([u, v, w]);
            totalWeight += w;
            
            if (mst.length === n - 1) break;
        }
    }
    
    return { mst, totalWeight };
}

/**
 * Prim 算法
 */
function prim(graph) {
    const n = graph.length;
    const minDist = new Array(n).fill(Infinity);
    const parent = new Array(n).fill(null);
    const inMST = new Array(n).fill(false);
    
    minDist[0] = 0;
    let totalWeight = 0;
    const mst = [];
    
    for (let count = 0; count < n; count++) {
        let u = -1, minVal = Infinity;
        for (let v = 0; v < n; v++) {
            if (!inMST[v] && minDist[v] < minVal) {
                minVal = minDist[v];
                u = v;
            }
        }
        
        if (u === -1) break;
        
        inMST[u] = true;
        totalWeight += minVal;
        
        if (parent[u] !== null) {
            mst.push([parent[u], u, graph[parent[u]][u]]);
        }
        
        for (let v = 0; v < n; v++) {
            if (graph[u][v] > 0 && !inMST[v] && graph[u][v] < minDist[v]) {
                minDist[v] = graph[u][v];
                parent[v] = u;
            }
        }
    }
    
    return { mst, totalWeight };
}

/**
 * 并查集（辅助）
 */
class UnionFind {
    constructor(n) {
        this.parent = new Array(n);
        this.rank = new Array(n).fill(0);
        for (let i = 0; i < n; i++) {
            this.parent[i] = i;
        }
    }
    
    find(x) {
        if (this.parent[x] === x) return x;
        const root = this.find(this.parent[x]);
        this.parent[x] = root;
        return root;
    }
    
    union(x, y) {
        const rootX = this.find(x);
        const rootY = this.find(y);
        if (rootX !== rootY) {
            if (this.rank[rootX] < this.rank[rootY]) {
                this.parent[rootX] = rootY;
            } else if (this.rank[rootX] > this.rank[rootY]) {
                this.parent[rootY] = rootX;
            } else {
                this.parent[rootY] = rootX;
                this.rank[rootX]++;
            }
        }
    }
    
    connected(x, y) {
        return this.find(x) === this.find(y);
    }
}
```
