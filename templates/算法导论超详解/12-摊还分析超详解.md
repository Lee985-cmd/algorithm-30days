# 📚 算法导论 Part V - 摊还分析超详解

## 第 16 章：看平均成本 - 摊还分析

### 16.1 什么是摊还分析？

#### 生活中的例子

**场景 1：AA 制吃饭**

```
你和朋友去餐厅吃饭

有人点了很贵的菜（100 元）
有人点了便宜的菜（20 元）

如果各付各的：
→ 有人花 100，有人花 20
→ 不公平！

如果 AA 制：
→ 总花费 480 元，6 个人
→ 每人 80 元
→ 大家都能接受！

这就是"摊还"的思想！
把贵的成本平摊到所有人头上
```

**场景 2：信用卡年费**

```
信用卡有年费 200 元/年

但你一年刷卡 100 次

平均每次的成本：
200 ÷ 100 = 2 元

虽然年费是一次性交的
但摊到每次消费上就很便宜！

这也是摊还分析！
```

**说人话定义：**

```
摊还分析 = 计算一系列操作的平均成本

为什么要摊还分析？
因为有些操作：
- 大部分时候很快（O(1)）
- 偶尔很慢（O(n)）

如果只看最坏情况：
→ 每次都按 O(n) 算
→ 太悲观了！

用摊还分析：
→ 考虑整体平均
→ 发现其实很快（O(1)）

更公平、更准确！
```

### 16.2 三种分析方法

#### 方法 1：聚合分析

```
核心思想：
先算 n 个操作的总成本 T(n)
然后除以 n，得到每个操作的摊还成本

公式：
摊还成本 = T(n) / n

简单粗暴！
```

**例子：栈操作**

```javascript
/**
 * 栈的三种操作
 */
class Stack {
    constructor() {
        this.items = [];
    }
    
    // push: O(1)
    push(x) {
        this.items.push(x);
    }
    
    // pop: O(1)
    pop() {
        return this.items.pop();
    }
    
    // Multipop: O(k)
    multipop(k) {
        while (k > 0 && this.items.length > 0) {
            this.pop();
            k--;
        }
    }
}

// 问题：
// 执行 n 次操作（push/pop/multipop 的任意组合）
// 最坏时间复杂度是多少？
```

**聚合分析过程：**

```
分析：
- 每次 push 放入一个元素
- 每次 pop 或 multipop 弹出一个元素
- 要弹出元素，必须先 push 进去

关键观察：
n 次操作中，push 的次数最多 n 次
所以 pop 的次数也最多 n 次

因此：
n 次操作的总成本 ≤ 2n
（n 次 push + n 次 pop）

摊还成本：
2n / n = 2 = O(1)

结论：
每个操作的摊还成本是 O(1)！

虽然 multipop 单次可能是 O(n)
但平均下来是 O(1)！
```

---

#### 方法 2：记账方法

```
核心思想：
给每个操作分配一个"摊还成本"

操作时：
- 收的钱 > 实际花费 → 存起来（信用）
- 收的钱 < 实际花费 → 用存款补贴

关键：
要保证存款永远不为负！

如果做到了：
总摊还成本 ≥ 总实际成本
摊还成本就是实际成本的上界！
```

**记账方法分析栈操作**

```
定义摊还成本：
- push: 收费 3 元
- pop: 收费 0 元
- multipop: 收费 0 元

钱花在哪里？
- 每次 push 实际花费 1 元
- 每次 pop 实际花费 1 元

多收的钱存起来：
- push 收 3 元，花 1 元，存 2 元
- 这 2 元用来支付将来的 pop

为什么够？
- 每个元素 push 时存 2 元
- pop 该元素时花 1 元
- 永远够用！

验证：
n 次操作的总摊还成本：
假设 push 了 p 次
总摊还成本 = 3p

总实际成本：
push 花费 p
pop 花费 ≤ p（不能 pop 没 push 过的东西）
总实际成本 ≤ 2p

3p ≥ 2p ✓

所以摊还成本 O(1) 是实际上界！
```

#### 方法 3：势能方法

```
核心思想：
把"存款"看作系统的"势能"

就像物理中的势能：
- 位置高 → 势能大
- 位置低 → 势能小

数据结构也有势能：
- 栈里元素多 → 势能大
- 栈里元素少 → 势能小

摊还成本公式：
âᵢ = cᵢ + Φ(Dᵢ) - Φ(Dᵢ₋₁)

其中：
âᵢ = 第 i 个操作的摊还成本
cᵢ = 第 i 个操作的实际成本
Φ(Dᵢ) = 操作后的势能
Φ(Dᵢ₋₁) = 操作前的势能

说人话：
摊还成本 = 实际成本 + 势能变化
```

**势能方法分析栈操作**

```
定义势能函数：
Φ(D) = 栈中元素的个数

初始状态：空栈，Φ(D₀) = 0

分析各个操作：

1. push(x):
   实际成本：cᵢ = 1
   势能变化：Φ(Dᵢ) - Φ(Dᵢ₋₁) = 1
   摊还成本：âᵢ = 1 + 1 = 2

2. pop():
   实际成本：cᵢ = 1
   势能变化：Φ(Dᵢ) - Φ(Dᵢ₋₁) = -1
   摊还成本：âᵢ = 1 + (-1) = 0

3. multipop(k):
   实际成本：cᵢ = k（最多 pop k 次）
   势能变化：Φ(Dᵢ) - Φ(Dᵢ₋₁) = -k
   摊还成本：âᵢ = k + (-k) = 0

验证：
- 所有操作的摊还成本都是 O(1)
- 总摊还成本 = Σâᵢ = O(n)
- 势能始终非负 ✓

结论：
每个操作的摊还成本是 O(1)！
```

### 16.3 经典例子：动态表扩张

#### 问题描述

```
动态数组（如 JavaScript 的 Array）：
- 初始容量很小
- 满了就自动扩容
- 通常扩容为原来的 2 倍

例如：
容量：1 → 2 → 4 → 8 → 16 ...

问题：
插入 n 个元素的总成本是多少？
```

#### 详细分析

```javascript
/**
 * 动态表的简单实现
 */
class DynamicArray {
    constructor() {
        this.array = new Array(1);
        this.size = 0;
        this.capacity = 1;
    }
    
    insert(val) {
        // 如果满了，扩容
        if (this.size === this.capacity) {
            this.resize(this.capacity * 2);
        }
        
        this.array[this.size] = val;
        this.size++;
    }
    
    resize(newCapacity) {
        // 创建新数组
        const newArray = new Array(newCapacity);
        
        // 复制旧数据
        for (let i = 0; i < this.size; i++) {
            newArray[i] = this.array[i];
        }
        
        this.array = newArray;
        this.capacity = newCapacity;
    }
}

// 问题：insert n 次的总成本？
```

**聚合分析：**

```
观察：
第 i 次插入：
- 如果不需扩容：成本 1
- 如果需扩容：成本 i（复制 i-1 个元素 + 插入 1 个）

什么时候需要扩容？
当 i 是 2 的幂时：i = 1, 2, 4, 8, 16...

n 次插入的总成本：
T(n) = Σ(不需扩容的成本) + Σ(扩容的成本)
     = n + Σ(2^k)，其中 2^k ≤ n
     = n + (1 + 2 + 4 + ... + 2^⌊log n⌋)
     = n + (2^(⌊log n⌋+1) - 1)
     < n + 2n
     = 3n

摊还成本：
T(n) / n = 3n / n = 3 = O(1)

结论：
每次插入的摊还成本是 O(1)！
```

**记账方法分析：**

```
定义：
- 每次插入收费 3 元

钱花在哪里？
- 插入本身：1 元
- 将来复制自己：1 元
- 复制别人：1 元（这个解释有点牵强）

更好的策略：
- 插入收费 3 元
- 当前插入花费 1 元
- 存 2 元作为"复制基金"
  - 1 元用于复制自己
  - 1 元用于复制前面的元素

验证：
第 i 次插入：
- 如果 i 不是 2 的幂：花费 1 元，存 2 元
- 如果 i 是 2 的幂：
  - 复制 i-1 个元素，花费 i-1 元
  - 插入当前元素，花费 1 元
  - 之前存的钱：(i-1) × 2 = 2i-2 元
  - 够用！✓

总摊还成本：3n = O(n)
摊还成本：O(1) ✓
```

**势能方法分析：**

```
定义势能函数：
Φ(T) = 2 × size - capacity

其中：
- size：当前元素个数
- capacity：当前容量

验证非负性：
- 刚扩容后：size = capacity/2
  Φ = 2×(capacity/2) - capacity = 0 ✓
- 满的时候：size = capacity
  Φ = 2×capacity - capacity = capacity ≥ 0 ✓

分析插入操作：

1. 不扩容的情况：
   size' = size + 1
   capacity' = capacity
   Φ' = 2(size+1) - capacity = Φ + 2
   
   实际成本：cᵢ = 1
   摊还成本：âᵢ = 1 + 2 = 3

2. 扩容的情况（size = capacity）：
   size' = size + 1
   capacity' = 2 × capacity
   
   Φ' = 2(size+1) - 2×capacity
       = 2size + 2 - 2size
       = 2
   
   Φ = 2size - capacity = 2size - size = size
   
   势能变化：Φ' - Φ = 2 - size
   
   实际成本：cᵢ = size + 1（复制 size 个 + 插入 1 个）
   
   摊还成本：âᵢ = (size + 1) + (2 - size) = 3

结论：
无论是否扩容，摊还成本都是 3 = O(1)！
```

### 16.5 实际应用

#### 应用 1：动态数组的选择

```javascript
// JavaScript 的 Array
const arr = [];
arr.push(1);  // 摊还 O(1)
arr.push(2);  // 摊还 O(1)
// ...
arr.push(10000);  // 还是摊还 O(1)

// 不用每次扩容都担心
因为摊还成本是 O(1)！
```

#### 应用 2：哈希表的扩容

```
哈希表装满了怎么办？
→ 扩容为原来的 2 倍
→ 重新散列所有元素

虽然单次扩容是 O(n)
但摊还下来每次插入仍是 O(1)！

这就是为什么哈希表这么快！
```

#### 应用 3：并查集的优化

```
并查集的路径压缩 + 按秩合并：

单次 Find 操作：最坏 O(log n)
但 n 次操作的摊还成本：O(α(n)) ≈ O(1)

α(n) 是反阿克曼函数
增长极其缓慢！
```

### 16.6 完整代码模板

```javascript
/**
 * 动态数组 - 支持摊还分析
 */
class DynamicArray {
    constructor() {
        this.array = new Array(1);
        this.size = 0;
        this.capacity = 1;
    }
    
    insert(val) {
        if (this.size === this.capacity) {
            this.resize(this.capacity * 2);
        }
        
        this.array[this.size++] = val;
    }
    
    resize(newCapacity) {
        const newArray = new Array(newCapacity);
        for (let i = 0; i < this.size; i++) {
            newArray[i] = this.array[i];
        }
        this.array = newArray;
        this.capacity = newCapacity;
    }
    
    // 摊还分析：
    // 插入 n 次的总成本：O(n)
    // 每次插入的摊还成本：O(1)
}

// 测试
const arr = new DynamicArray();
for (let i = 0; i < 1000; i++) {
    arr.insert(i);
    // 每次都是摊还 O(1)！
}
```
