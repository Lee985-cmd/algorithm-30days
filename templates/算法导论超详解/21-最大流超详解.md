# 📚 算法导论 Part VI - 最大流超详解

## 第 24 章：网络的极限容量 - 最大流问题

### 24.1 什么是最大流？

#### 生活中的例子

**场景 1：水管网络**

```
有一个自来水厂（源点 s）
要供水到一个小区（汇点 t）

中间有很多水管和水泵站：

        5         3
    s ────→ A ────→ t
    │       │       ↑
  4 │     2 │     4 │
    ↓       ↓       │
    B ────→ C ──────┘
        3

每条边有容量限制（数字）
问：最多能输送多少水到小区？

这就是最大流问题！
```

**场景 2：交通流量**

```
城市交通网络：

起点：火车站
终点：机场

道路有车道数限制（容量）

问：每小时最多能通过多少辆车？

这也是最大流问题！
```

**说人话定义：**

```
最大流 = 在容量限制下，从源点到汇点的最大流量

输入：
- 有向图 G = (V, E)
- 每条边有容量 c(u,v) ≥ 0
- 源点 s（出发点）
- 汇点 t（目的地）

输出：
- 从 s 到 t 的最大流量

约束：
1. 容量限制：f(u,v) ≤ c(u,v)
   流量不能超过容量
   
2. 流量守恒：流入 = 流出
   中间节点不能囤积或产生流量
   （源点和汇点除外）
```

### 24.2 Ford-Fulkerson 方法 - 核心思想

#### 基本思路

```
生活比喻：不断找路改进

Ford-Fulkerson 方法的做法：
1. 初始时，所有边的流量都是 0
2. 找一条从 s 到 t 的"增广路径"
   （还能增加流量的路径）
3. 沿着这条路增加流量
4. 重复 2-3，直到找不到增广路径

关键概念：
✓ 残留网络：还剩多少容量可用
✓ 增广路径：还能改进的路径
✓ 反向边：允许"反悔"，把流量退回去
```

#### 重要概念详解

```
1. 残留网络（Residual Network）

原图的容量：c(u,v)
当前流量：f(u,v)
残留容量：c_f(u,v) = c(u,v) - f(u,v)

残留网络就是用残留容量构建的新图
表示还能走多少流量

2. 反向边

如果原图有边 u→v，流量为 f
那么残留网络中有反向边 v→u，容量为 f

意义：
允许"反悔"
可以把已经分配的流量退回去
重新分配

这太重要了！
没有反向边就找不到最优解！
```

#### 代码实现

```javascript
/**
 * Ford-Fulkerson 算法 - 最大流
 * @param {number[][]} capacity - 容量矩阵
 * @param {number} s - 源点
 * @param {number} t - 汇点
 * @returns {number} - 最大流
 */
function fordFulkerson(capacity, s, t) {
    const n = capacity.length;
    
    // 残留网络（初始等于原容量）
    const residual = capacity.map(row => [...row]);
    
    // 记录最大流
    let maxFlow = 0;
    
    // 不断寻找增广路径
    while (true) {
        // BFS 找增广路径
        const parent = new Array(n).fill(-1);
        const queue = [s];
        parent[s] = s;
        
        while (queue.length > 0) {
            const u = queue.shift();
            
            for (let v = 0; v < n; v++) {
                if (parent[v] === -1 && residual[u][v] > 0) {
                    parent[v] = u;
                    queue.push(v);
                    
                    if (v === t) break;  // 找到汇点
                }
            }
        }
        
        // 如果没有增广路径，结束
        if (parent[t] === -1) {
            break;
        }
        
        // 计算这条路径能增加的流量
        let pathFlow = Infinity;
        let v = t;
        while (v !== s) {
            const u = parent[v];
            pathFlow = Math.min(pathFlow, residual[u][v]);
            v = u;
        }
        
        // 更新残留网络
        v = t;
        while (v !== s) {
            const u = parent[v];
            residual[u][v] -= pathFlow;  // 正向边减少
            residual[v][u] += pathFlow;  // 反向边增加
            v = u;
        }
        
        // 累加流量
        maxFlow += pathFlow;
    }
    
    return maxFlow;
}

// 测试
const capacity = [
    [0, 5, 4, 0],  // s(0) → A(1), B(2)
    [0, 0, 2, 3],  // A(1) → B(2), t(3)
    [0, 0, 0, 4],  // B(2) → t(3)
    [0, 0, 0, 0]   // t(3)
];

console.log(fordFulkerson(capacity, 0, 3));
// 输出：7 ✓
```

### 24.3 Edmonds-Karp 实现 - BFS 优化

```
Ford-Fulkerson 只是一个方法框架
具体实现有不同版本

Edmonds-Karp 是其中一种：
用 BFS 找增广路径
每次找最短的增广路径（边数最少）

时间复杂度：O(VE²)
比一般的 Ford-Fulkerson 更稳定
```

### 24.4 最大流最小割定理

#### 什么是割？

```
割（Cut）：
把图分成两部分
源点 s 在一部分
汇点 t 在另一部分

割的容量：
从 s 所在部分到 t 所在部分的所有边的容量之和

例如：
        5         3
    s ────→ A ────→ t
    │       │       
  4 │     2 │       
    ↓       ↓       
    B ────→ C       

一个可能的割：
{s, A} 和 {B, C, t}

割的容量 = c(s,B) + c(A,t) = 4 + 3 = 7
```

#### 定理内容

```
最大流最小割定理：
最大流的值 = 最小割的容量

这意味着：
✓ 找出最大流，就知道了最小割
✓ 找出最小割，就知道了最大流

证明思路：
当算法结束时
残留网络中没有从 s 到 t 的路径
此时 s 能到达的点构成集合 S
t 和剩下的点构成集合 T
(S, T) 就是一个最小割
```

### 24.5 实际应用

#### 应用 1：二分图匹配

```
问题：
有 n 个男生和 m 个女生
某些男生和女生互相有好感
问：最多能撮合多少对情侣？

解法：
1. 构造源点 s，连向所有男生，容量 1
2. 构造汇点 t，所有女生连向 t，容量 1
3. 互相有好感的男女之间连边，容量 1
4. 求 s 到 t 的最大流

最大流的值就是最大匹配数！
```

#### 应用 2：棒球淘汰问题

```
问题：
棒球联赛，每队还有若干场比赛
问：某队是否还有可能获得第一？

解法：
1. 构造网络
2. 源点连向所有未打的比赛
3. 比赛连向参赛的两队
4. 球队连向汇点，容量为该队还能赢的场数
5. 求最大流

如果最大流等于剩余比赛总数
则该队未被淘汰
否则被淘汰！
```

#### 应用 3：图像分割

```
计算机视觉中的图像分割：

前景像素连向源点
背景像素连向汇点
相邻像素之间连边

边的容量根据颜色、纹理等确定

求最小割
就把图像分成了前景和背景！
```

### 24.6 完整代码模板（背诵版）

```javascript
/**
 * Edmonds-Karp 算法 - 最大流
 */
function edmondsKarp(capacity, s, t) {
    const n = capacity.length;
    const residual = capacity.map(row => [...row]);
    let maxFlow = 0;
    
    while (true) {
        // BFS 找增广路径
        const parent = new Array(n).fill(-1);
        const queue = [s];
        parent[s] = s;
        
        while (queue.length > 0) {
            const u = queue.shift();
            for (let v = 0; v < n; v++) {
                if (parent[v] === -1 && residual[u][v] > 0) {
                    parent[v] = u;
                    queue.push(v);
                    if (v === t) break;
                }
            }
        }
        
        if (parent[t] === -1) break;
        
        // 计算路径流量
        let pathFlow = Infinity;
        let v = t;
        while (v !== s) {
            const u = parent[v];
            pathFlow = Math.min(pathFlow, residual[u][v]);
            v = u;
        }
        
        // 更新残留网络
        v = t;
        while (v !== s) {
            const u = parent[v];
            residual[u][v] -= pathFlow;
            residual[v][u] += pathFlow;
            v = u;
        }
        
        maxFlow += pathFlow;
    }
    
    return maxFlow;
}

/**
 * 获取最小割
 */
function getMinCut(residual, s) {
    const n = residual.length;
    const visited = new Array(n).fill(false);
    const queue = [s];
    visited[s] = true;
    
    // BFS 找 s 能到达的点
    while (queue.length > 0) {
        const u = queue.shift();
        for (let v = 0; v < n; v++) {
            if (!visited[v] && residual[u][v] > 0) {
                visited[v] = true;
                queue.push(v);
            }
        }
    }
    
    // 收集割边
    const cutEdges = [];
    for (let u = 0; u < n; u++) {
        if (visited[u]) {
            for (let v = 0; v < n; v++) {
                if (!visited[v] && residual[u][v] === 0) {
                    cutEdges.push([u, v]);
                }
            }
        }
    }
    
    return { visited, cutEdges };
}
```

### 24.7 时间复杂度分析

```
Edmonds-Karp 算法：

每次 BFS：O(E)
最多进行 O(VE) 次增广

总时间复杂度：O(VE²)

对于稀疏图（E ≈ V）：O(V³)
对于稠密图（E ≈ V²）：O(V⁵)

空间复杂度：O(V²)（邻接矩阵）
```
