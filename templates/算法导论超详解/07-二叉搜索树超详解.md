# 📚 算法导论 Part III - 二叉搜索树超详解

## 第 12 章：会思考的字典 - 二叉搜索树

### 12.1 什么是二叉搜索树？为什么这么快？

#### 生活中的例子

**场景 1：猜数字游戏**

```
我想一个 1-100 之间的数字，你来猜

你每次猜一个数，我会告诉你是"大了"还是"小了"

聪明的猜法：
第一次：50（中间）
如果我说"大了" → 下次猜 25（左半边中间）
如果我说"小了" → 下次猜 75（右半边中间）

这样每次排除一半！
最多猜 7 次就能猜到！

这就是二叉搜索的思想！
```

**场景 2：查字典**

```
查"apple"这个单词：

笨方法：
从第一页开始一页一页翻 → O(n)

聪明方法：
1. 根据首字母 a，直接翻到前面部分
2. 在 A 开头的部分继续找
3. 很快就能找到！

这也是二叉搜索的思想！
```

**说人话定义：**

```
二叉搜索树（BST）= 会思考的字典

特点：
✓ 左子树的所有节点 < 根节点
✓ 右子树的所有节点 > 根节点
✓ 左右子树也是二叉搜索树

查找、插入、删除都很快！
平均时间复杂度：O(log n)
```

### 12.3 查找操作 - 最简单的操作

#### 代码实现

```javascript
class BinarySearchTree {
    /**
     * 查找某个值
     * @param {number} val - 要查找的值
     * @returns {TreeNode|null} - 找到返回节点，否则返回 null
     */
    search(val) {
        return this._searchRecursive(this.root, val);
    }
    
    /**
     * 递归版本
     */
    _searchRecursive(node, val) {
        // 没找到（到了空节点）
        if (node === null) {
            return null;
        }
        
        // 找到了
        if (node.val === val) {
            return node;
        }
        
        // 去左边找
        if (val < node.val) {
            return this._searchRecursive(node.left, val);
        }
        
        // 去右边找
        if (val > node.val) {
            return this._searchRecursive(node.right, val);
        }
    }
    
    /**
     * 迭代版本（推荐）
     */
    searchIterative(val) {
        let current = this.root;
        
        while (current !== null) {
            if (val === current.val) {
                return current;  // 找到了
            }
            
            if (val < current.val) {
                current = current.left;  // 往左走
            } else {
                current = current.right;  // 往右走
            }
        }
        
        return null;  // 没找到
    }
}

// 测试
const bst = new BinarySearchTree();
// 构建树：
//       8
//      / \
//     3   10
//    / \    \
//   1   6    14

console.log(bst.search(6));   // 返回值为 6 的节点 ✓
console.log(bst.search(100)); // null（没找到）✓
```

#### 详细执行过程

```
查找 6：

初始：current = 8

═══════════════════════════════════
第 1 步
═══════════════════════════════════
current = 8
6 < 8? ✓
→ 往左走
current = current.left = 3

═══════════════════════════════════
第 2 步
═══════════════════════════════════
current = 3
6 == 3? ✗
6 < 3? ✗
6 > 3? ✓
→ 往右走
current = current.right = 6

═══════════════════════════════════
第 3 步
═══════════════════════════════════
current = 6
6 == 6? ✓
→ 找到了！返回！

总共走了 3 步
如果是链表，可能需要走很多步
BST 就是这么快！
```

#### 时间复杂度

```
最好情况：O(1)
→ 第一次就找到了

最坏情况：O(h)
→ h 是树的高度
→ 如果树退化成链表，h = n
→ O(n) ❌

平均情况：O(log n)
→ 每次都排除一半
→ 就像二分查找
```

### 12.4 插入操作 - 找到位置就放进去

#### 代码实现

```javascript
class BinarySearchTree {
    /**
     * 插入一个新值
     * @param {number} val - 要插入的值
     */
    insert(val) {
        const newNode = new TreeNode(val);
        
        // 如果是空树，新节点就是根
        if (this.root === null) {
            this.root = newNode;
            return;
        }
        
        // 找位置
        let current = this.root;
        let parent = null;
        
        while (current !== null) {
            parent = current;
            
            if (val < current.val) {
                current = current.left;
            } else if (val > current.val) {
                current = current.right;
            } else {
                // 值已存在，不插入
                return;
            }
        }
        
        // 找到位置了，挂到父节点下面
        if (val < parent.val) {
            parent.left = newNode;
        } else {
            parent.right = newNode;
        }
    }
}

// 测试
```

### 12.5 删除操作 - 最复杂的情况

#### 三种情况

```
删除节点有三种情况：

情况 1：叶子节点（没有孩子）
→ 直接删除

情况 2：只有一个孩子
→ 让孙子继承爷爷的位置

情况 3：有两个孩子
→ 找替死鬼（后继或前驱）
```

#### 代码实现

```javascript
class BinarySearchTree {
    /**
     * 删除某个值
     * @param {number} val - 要删除的值
     */
    delete(val) {
        this.root = this._deleteRecursive(this.root, val);
    }
    
    _deleteRecursive(node, val) {
        // 没找到
        if (node === null) {
            return null;
        }
        
        // 找节点
        if (val < node.val) {
            node.left = this._deleteRecursive(node.left, val);
        } else if (val > node.val) {
            node.right = this._deleteRecursive(node.right, val);
        } else {
            // 找到了，开始删除
            
            // 情况 1 & 2：没有孩子或只有一个孩子
            if (node.left === null) {
                return node.right;  // 右孩子继承
            } else if (node.right === null) {
                return node.left;   // 左孩子继承
            }
            
            // 情况 3：有两个孩子
            // 找右子树的最小节点（后继）
            const minNode = this._findMin(node.right);
            
            // 用后继的值替换当前节点
            node.val = minNode.val;
            
            // 删除后继节点
            node.right = this._deleteRecursive(node.right, minNode.val);
        }
        
        return node;
    }
    
    /**
     * 找最小节点
     */
    _findMin(node) {
        while (node.left !== null) {
            node = node.left;
        }
        return node;
    }
}

// 测试
const bst = new BinarySearchTree();
// 树：
//       8
//      / \
//     3   10
//    / \    \
//   1   6    14

bst.delete(3);  // 删除有两个孩子的节点
console.log(bst.search(3));  // null（删除成功）
```

#### 详细执行过程（重点！）

```

═══════════════════════════════════

右子树是以 6 为根的子树
最小值 = 6（6 没有左孩子）

所以后继是 6

═══════════════════════════════════
第 3 步：替换
═══════════════════════════════════

```

#### 为什么要找后继？

```
问题：为什么不用前驱？

答案：都可以！

后继 = 右子树的最小值
前驱 = 左子树的最大值

通常用后继，但用前驱也可以

关键是要保证：
替换后仍然满足 BST 性质！

后继一定比左子树大，比右子树小
所以用它替换最合适！
```

### 12.6 遍历操作 - 访问所有节点

#### 中序遍历（最重要）

```javascript
class BinarySearchTree {
    /**
     * 中序遍历 - 从小到大输出
     * @returns {number[]} - 有序的数组
     */
    inorderTraversal() {
        const result = [];
        this._inorderRecursive(this.root, result);
        return result;
    }
    
    _inorderRecursive(node, result) {
        if (node === null) return;
        
        // 左 → 根 → 右
        this._inorderRecursive(node.left, result);
        result.push(node.val);
        this._inorderRecursive(node.right, result);
    }
}

// 测试
const bst = new BinarySearchTree();
// 树：
//       8
//      / \
//     3   10
//    / \    \
//   1   6    14

console.log(bst.inorderTraversal());
// 输出：[1, 3, 6, 8, 10, 14]
// 自动排好序了！✓
```

#### 其他遍历方式

```javascript
// 前序遍历：根 → 左 → 右
preorderTraversal() {
    const result = [];
    this._preorderRecursive(this.root, result);
    return result;
}

_preorderRecursive(node, result) {
    if (node === null) return;
    result.push(node.val);  // 根
    this._preorderRecursive(node.left, result);  // 左
    this._preorderRecursive(node.right, result); // 右
}

// 后序遍历：左 → 右 → 根
postorderTraversal() {
    const result = [];
    this._postorderRecursive(this.root, result);
    return result;
}

_postorderRecursive(node, result) {
    if (node === null) return;
    this._postorderRecursive(node.left, result);  // 左
    this._postorderRecursive(node.right, result); // 右
    result.push(node.val);  // 根
}
```

#### 遍历对比

```
树：
       8
      / \
     3   10
    / \
   1   6

中序（左根右）：[1, 3, 6, 8, 10]
→ 有序！这是 BST 的重要特性！

前序（根左右）：[8, 3, 1, 6, 10]
→ 可以用来复制树

后序（左右根）：[1, 6, 3, 10, 8]
→ 可以用来删除树（先删孩子再删根）
```

### 12.7 最大最小值

#### 代码实现

```javascript
class BinarySearchTree {
    /**
     * 找最小值
     */
    findMin() {
        if (this.root === null) return null;
        
        let current = this.root;
        while (current.left !== null) {
            current = current.left;
        }
        return current.val;
    }
    
    /**
     * 找最大值
     */
    findMax() {
        if (this.root === null) return null;
        
        let current = this.root;
        while (current.right !== null) {
            current = current.right;
        }
        return current.val;
    }
}

// 测试
const bst = new BinarySearchTree();
// 树：
//       8
//      / \
//     3   10
//    / \    \
//   1   6    14

console.log(bst.findMin());  // 1 ✓
console.log(bst.findMax());  // 14 ✓
```

#### 为什么这么快？

```
找最小值：一直往左走
找最大值：一直往右走

时间复杂度：O(h)
h 是树的高度

平衡的情况下：O(log n)
退化成链表：O(n)
```

### 12.8 BST 的优缺点

#### 优点 ✅

```
✓ 查找快：平均 O(log n)
✓ 插入快：O(log n)
✓ 删除快：O(log n)
✓ 有序：中序遍历得到有序序列
✓ 支持范围查询：找 [a, b] 之间的所有数
```

#### 缺点 ❌

```
❌ 可能退化：最坏情况变成 O(n)
❌ 需要额外空间：每个节点要存两个指针
❌ 不平衡时性能差
```

#### 解决方案

```
问题：树不平衡怎么办？

解决：自平衡二叉搜索树
→ AVL 树
→ 红黑树（下一章讲）
→ 2-3 树
→ B 树

这些树能保证始终平衡！
始终保持 O(log n)！
```

### 12.9 完整代码（背诵版）

```javascript
/**
 * 二叉搜索树节点
 */
class TreeNode {
    constructor(val) {
        this.val = val;
        this.left = null;
        this.right = null;
    }
}

/**
 * 二叉搜索树
 */
class BinarySearchTree {
    constructor() {
        this.root = null;
    }
    
    // 插入
    insert(val) {
        const newNode = new TreeNode(val);
        
        if (this.root === null) {
            this.root = newNode;
            return;
        }
        
        let current = this.root;
        let parent = null;
        
        while (current !== null) {
            parent = current;
            
            if (val < current.val) {
                current = current.left;
            } else if (val > current.val) {
                current = current.right;
            } else {
                return;  // 已存在
            }
        }
        
        if (val < parent.val) {
            parent.left = newNode;
        } else {
            parent.right = newNode;
        }
    }
    
    // 查找
    search(val) {
        let current = this.root;
        
        while (current !== null) {
            if (val === current.val) {
                return current;
            }
            
            if (val < current.val) {
                current = current.left;
            } else {
                current = current.right;
            }
        }
        
        return null;
    }
    
    // 删除
    delete(val) {
        this.root = this._deleteRecursive(this.root, val);
    }
    
    _deleteRecursive(node, val) {
        if (node === null) return null;
        
        if (val < node.val) {
            node.left = this._deleteRecursive(node.left, val);
        } else if (val > node.val) {
            node.right = this._deleteRecursive(node.right, val);
        } else {
            if (node.left === null) {
                return node.right;
            } else if (node.right === null) {
                return node.left;
            }
            
            const minNode = this._findMin(node.right);
            node.val = minNode.val;
            node.right = this._deleteRecursive(node.right, minNode.val);
        }
        
        return node;
    }
    
    _findMin(node) {
        while (node.left !== null) {
            node = node.left;
        }
        return node;
    }
    
    // 中序遍历
    inorder() {
        const result = [];
        this._inorderRecursive(this.root, result);
        return result;
    }
    
    _inorderRecursive(node, result) {
        if (node === null) return;
        this._inorderRecursive(node.left, result);
        result.push(node.val);
        this._inorderRecursive(node.right, result);
    }
    
    // 找最小值
    findMin() {
        if (this.root === null) return null;
        let current = this.root;
        while (current.left !== null) {
            current = current.left;
        }
        return current.val;
    }
    
    // 找最大值
    findMax() {
        if (this.root === null) return null;
        let current = this.root;
        while (current.right !== null) {
            current = current.right;
        }
        return current.val;
    }
}

// 使用示例
const bst = new BinarySearchTree();
bst.insert(8);
bst.insert(3);
bst.insert(10);
bst.insert(1);
bst.insert(6);
bst.insert(14);

console.log(bst.search(6));       // 找到
console.log(bst.inorder());       // [1, 3, 6, 8, 10, 14]
console.log(bst.findMin());       // 1
console.log(bst.findMax());       // 14

bst.delete(3);
console.log(bst.search(3));       // null（删除成功）
```
