### 22.1 什么是最短路径问题？

#### 生活中的例子

**场景 1：手机导航**

```
你要从北京到上海

有无数种走法：
- 飞机：3 小时，花费 1000 元
- 高铁：5 小时，花费 600 元
- 自驾：12 小时，油费 800 元
- 普快：20 小时，硬座 200 元

什么是"最短"？
→ 时间最短？
→ 费用最低？
→ 距离最近？
→ 综合最优？

这就是最短路径问题！
"短"可以指任何你想最小化的东西！
```

**场景 2：网络路由**

```
数据包要从你的电脑传到服务器

互联网是一张巨大的网
有无数的路径可选

路由器要决定：
走哪条路最快？
哪条路不堵车？
哪条路最可靠？

这也是最短路径问题！
```

**场景 3：游戏寻路**

```
游戏中角色要从 A 点走到 B 点

地图上有很多障碍物

AI 要计算：
怎么走不撞墙？
怎么走最近？
怎么避开敌人？

这还是最短路径问题！
```

**说人话定义：**

```
最短路径问题 = 找两点之间的最优路线

输入：
- 一张图（顶点 + 边）
- 每条边有权重（距离、时间、费用等）
- 起点和终点

输出：
- 从起点到终点的最短路径
- 或者最短距离

"最短"可以是：
✓ 距离最短
✓ 时间最少
✓ 费用最低
✓ 换乘最少

```

### 22.2 最短路径的分类

#### 问题的不同类型

```
类型 1：单源最短路径
从一个起点出发，到所有其他点的最短路径

例如：
从北京出发，到全国所有城市的最短路径

算法：
→ Dijkstra（无负权边）
→ Bellman-Ford（可有负权边）
→ SPFA（优化版 Bellman-Ford）

类型 2：单目标最短路径
从所有点出发，到一个终点的最短路径

解法：
把边反向，变成单源最短路径

类型 3：所有点对最短路径
任意两点之间的最短路径

例如：
全国任意两个城市之间的最短路径

算法：
→ Floyd-Warshall（动态规划）
→ n 次 Dijkstra

类型 4：带约束的最短路径
在限制条件下的最短路径

例如：
- 必须经过某些点
- 不能经过某些点
- 最多换乘 3 次
- 总费用不超过 1000

这种更难，通常用搜索算法
```

### 22.3 Dijkstra 算法 - 最经典的单源最短路径

#### 核心思想

```
生活比喻：水往低处流

想象起点是一个水源
水从起点往外流
先到达近的地方
再到达远的地方

Dijkstra 的做法：
1. 从起点开始
2. 每次选"当前能到达的最近的点"
3. 更新它邻居的距离
4. 重复 2-3，直到所有点都处理完

就像水波纹扩散！
```

#### 适用条件

```
✓ 边的权重必须非负（≥ 0）
✓ 可以是有向图或无向图
✓ 能找到单源到所有点的最短路径

✗ 不能有负权边（会出错）
✗ 不能有负权环（无意义）
```

#### 代码实现（完整版）

```javascript
/**
 * Dijkstra 算法 - 单源最短路径
 * @param {number[][]} graph - 邻接矩阵表示的图
 * @param {number} start - 起点
 * @returns {{distances: number[], prev: number[]}} - 最短距离和前驱节点
 */
function dijkstra(graph, start) {
    const n = graph.length;
    
    // distances[i] 表示从 start 到 i 的最短距离
    const distances = new Array(n).fill(Infinity);
    distances[start] = 0;
    
    // prev[i] 表示最短路径上 i 的前一个节点
    const prev = new Array(n).fill(null);
    
    // visited[i] 表示是否已经确定了到 i 的最短路径
    const visited = new Array(n).fill(false);
    
    // 需要处理 n 个顶点
    for (let count = 0; count < n; count++) {
        // 第 1 步：选未访问的顶点中距离最小的
        let minDist = Infinity;
        let u = -1;
        
        for (let v = 0; v < n; v++) {
            if (!visited[v] && distances[v] < minDist) {
                minDist = distances[v];
                u = v;
            }
        }
        
        // 如果所有点都不可达，结束
        if (u === -1) break;
        
        // 标记为已访问
        visited[u] = true;
        
        // 第 2 步：更新 u 的所有邻居
        for (let v = 0; v < n; v++) {
            // 如果 u 到 v 有边
            if (graph[u][v] > 0 && !visited[v]) {
                const newDist = distances[u] + graph[u][v];
                
                // 如果找到更短的路径，更新
                if (newDist < distances[v]) {
                    distances[v] = newDist;
                    prev[v] = u;  // 记录前驱
                }
            }
        }
    }
    
    return { distances, prev };
}

/**
 * 根据 prev 数组重建路径
 */
function reconstructPath(prev, start, end) {
    const path = [];
    let current = end;
    
    while (current !== null) {
        path.unshift(current);
        current = prev[current];
    }
    
    if (path[0] !== start) {
        return [];  // 不可达
    }
    
    return path;
}

// 测试
const graph = [
    // 0   1   2   3   4
    [0,  10,  3,  0,  0],  // 0
    [0,   0,  1,  2,  0],  // 1
    [0,   4,  0,  8,  2],  // 2
    [0,   0,  0,  0,  7],  // 3
    [0,   0,  0,  9,  0]   // 4
];

const result = dijkstra(graph, 0);
console.log('最短距离:', result.distances);
// 输出：[0, 7, 3, 9, 5]

console.log('到顶点 4 的最短路径:', 
    reconstructPath(result.prev, 0, 4));
// 输出：[0, 2, 4]
// 路径：0 → 2 → 4，距离 = 3 + 2 = 5 ✓
```

```

适合：稠密图（边很多）

- 选最小：O(log n)
- 总共处理 m 条边
总时间：O(m log n)

适合：稀疏图（边不多）

空间复杂度：O(n)
```

### 22.4 Bellman-Ford 算法 - 能处理负权边

#### 为什么需要 Bellman-Ford？

```
问题：如果有负权边怎么办？

例如：
A --5--> B
B --(-10)--> C
C --3--> D

从 A 到 D：
正常走：不知道多远
经过 B、C：5 + (-10) + 3 = -2

Dijkstra 能处理吗？
❌ 不能！
因为 Dijkstra 基于贪心，假设"越近的先确定"
但负权边会打破这个假设！

这时候要用 Bellman-Ford！
```

#### 核心思想

```
生活比喻：不断尝试改进

Bellman-Ford 的做法：
1. 初始化：起点距离为 0，其他都是∞
2. 松弛操作：尝试通过每条边改进距离
3. 重复第 2 步 V-1 次（V 是顶点数）
4. 检查是否有负权环

为什么 V-1 次就够了？
→ 因为最短路径最多经过 V-1 条边
→ 再多就一定有环了！
```

#### 代码实现

```javascript
/**
 * Bellman-Ford 算法
 * @param {number} n - 顶点数
 * @param {number[][]} edges - 边列表，每个元素是 [from, to, weight]
 * @param {number} start - 起点
 * @returns {{distances: number[], hasNegativeCycle: boolean}}
 */
function bellmanFord(n, edges, start) {
    // 初始化距离
    const distances = new Array(n).fill(Infinity);
    distances[start] = 0;
    
    // 第 1 步：松弛 V-1 次
    for (let i = 0; i < n - 1; i++) {
        let updated = false;
        
        for (let [from, to, weight] of edges) {
            if (distances[from] !== Infinity) {
                const newDist = distances[from] + weight;
                
                if (newDist < distances[to]) {
                    distances[to] = newDist;
                    updated = true;
                }
            }
        }
        
        // 如果一轮下来没有更新，提前结束
        if (!updated) break;
    }
    
    // 第 2 步：检查负权环
    let hasNegativeCycle = false;
    
    for (let [from, to, weight] of edges) {
        if (distances[from] !== Infinity) {
            const newDist = distances[from] + weight;
            
            if (newDist < distances[to]) {
                hasNegativeCycle = true;
                break;
            }
        }
    }
    
    return { distances, hasNegativeCycle };
}

// 测试
const n = 5;
const edges = [
    [0, 1, 6],
    [0, 3, 7],
    [1, 2, 5],
    [1, 3, 8],
    [1, 4, -4],
    [2, 1, -2],
    [3, 2, -3],
    [3, 4, 9],
    [4, 0, 2],
    [4, 2, 7]
];

const result = bellmanFord(n, edges, 0);
console.log('最短距离:', result.distances);
console.log('有负权环吗？', result.hasNegativeCycle);
```

但能处理负权边！

### 22.5 Floyd-Warshall 算法 - 所有点对的最短路径

#### 问题描述

```
不是求一个点到其他点
而是求任意两点之间的最短路径

例如：
全国有 100 个城市
想知道任意两个城市之间的最短距离

怎么做？
方法 1：对每个点跑一次 Dijkstra
→ O(n³)

方法 2：Floyd-Warshall 算法
→ 也是 O(n³)
→ 但代码超级简单！
→ 还能处理负权边！
```

#### 核心思想

```
动态规划的思想

设 d[k][i][j] 表示：
只允许经过顶点 0~k 时，从 i 到 j 的最短路径

状态转移方程：
d[k][i][j] = min(
    d[k-1][i][j],              // 不经过 k
    d[k-1][i][k] + d[k-1][k][j]  // 经过 k
)

说人话：
从 i 到 j，要么不经过 k
要么先从 i 到 k，再从 k 到 j
取两者中较小的

优化：
可以用二维数组，去掉第一维
```

#### 代码实现（简洁版）

```javascript
/**
 * Floyd-Warshall 算法 - 所有点对最短路径
 * @param {number[][]} graph - 邻接矩阵
 * @returns {number[][]} - 最短距离矩阵
 */
function floydWarshall(graph) {
    const n = graph.length;
    
    // 初始化 dist 为 graph 的副本
    const dist = graph.map(row => [...row]);
    
    // 三重循环
    for (let k = 0; k < n; k++) {      // 中间点
        for (let i = 0; i < n; i++) {  // 起点
            for (let j = 0; j < n; j++) {  // 终点
                // 如果经过 k 更短，就更新
                if (dist[i][k] !== Infinity && 
                    dist[k][j] !== Infinity) {
                    dist[i][j] = Math.min(
                        dist[i][j],
                        dist[i][k] + dist[k][j]
                    );
                }
            }
        }
    }
    
    return dist;
}

// 测试
const graph = [
    [0,   3,   8,   Infinity, -4],
    [Infinity, 0,   Infinity, 1,   7],
    [Infinity, 4,   0,   Infinity, Infinity],
    [2,   Infinity, -5,  0,   Infinity],
    [Infinity, Infinity, Infinity, 6,   0]
];

const result = floydWarshall(graph);
console.log('所有点对最短路径:');
for (let row of result) {
    console.log(row);
}
```

```

0    0   3   8   ∞  -4
1    ∞   0   ∞   1   7
2    ∞   4   0   ∞   ∞

4    ∞   ∞   ∞   6   0

═══════════════════════════════════

如果 i→0→j 比 i→j 短，就更新

3→0→2: 2+8=10 > -5 → 不更新
...

═══════════════════════════════════
k=1：允许经过顶点 0,1
═══════════════════════════════════
继续更新...

═══════════════════════════════════
k=2,3,4...
═══════════════════════════════════
依次允许经过每个顶点

最终得到所有点对的最短路径！
```

#### 时间复杂度

```
三重循环：
- 外层：n 次
- 中层：n 次
- 内层：n 次
总时间：O(n³)

空间复杂度：O(n²)

适合：顶点数不多的情况（n ≤ 100）
```

### 22.6 三种算法对比

| 特性 | Dijkstra | Bellman-Ford | Floyd-Warshall |
|------|---------|--------------|----------------|
| 问题类型 | 单源 | 单源 | 所有点对 |
| 时间复杂度 | O(n²) 或 O(m log n) | O(VE) | O(n³) |
| 空间复杂度 | O(n) | O(n) | O(n²) |
| 负权边 | ✗ 不能处理 | ✓ 可以 | ✓ 可以 |
| 负权环检测 | ✗ | ✓ 可以 | 较复杂 |
| 代码难度 | 中等 | 简单 | 超简单 |
| 适用场景 | 无负权边的单源 | 有负权边的单源 | 所有点对 |

/**
 * Dijkstra 算法
 */
function dijkstra(graph, start) {
    const n = graph.length;
    const dist = new Array(n).fill(Infinity);
    const prev = new Array(n).fill(null);
    const visited = new Array(n).fill(false);
    
    dist[start] = 0;
    
    for (let count = 0; count < n; count++) {
        let u = -1, minDist = Infinity;
        
        for (let v = 0; v < n; v++) {
            if (!visited[v] && dist[v] < minDist) {
                minDist = dist[v];
                u = v;
            }
        }
        
        if (u === -1) break;
        visited[u] = true;
        
        for (let v = 0; v < n; v++) {
            if (graph[u][v] > 0 && !visited[v]) {
                const newDist = dist[u] + graph[u][v];
                if (newDist < dist[v]) {
                    dist[v] = newDist;
                    prev[v] = u;
                }
            }
        }
    }
    
    return { dist, prev };
}

/**
 * Bellman-Ford 算法
 */
function bellmanFord(n, edges, start) {
    const dist = new Array(n).fill(Infinity);
    dist[start] = 0;
    
    for (let i = 0; i < n - 1; i++) {
        for (let [u, v, w] of edges) {
            if (dist[u] !== Infinity && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
            }
        }
    }
    
    // 检查负权环
    for (let [u, v, w] of edges) {
        if (dist[u] !== Infinity && dist[u] + w < dist[v]) {
            return { dist, hasNegativeCycle: true };
        }
    }
    
    return { dist, hasNegativeCycle: false };
}

/**
 * Floyd-Warshall 算法
 */
function floydWarshall(graph) {
    const n = graph.length;
    const dist = graph.map(row => [...row]);
    
    for (let k = 0; k < n; k++) {
        for (let i = 0; i < n; i++) {
            for (let j = 0; j < n; j++) {
                if (dist[i][k] !== Infinity && dist[k][j] !== Infinity) {
                    dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
                }
            }
        }
    }
    
    return dist;
}
