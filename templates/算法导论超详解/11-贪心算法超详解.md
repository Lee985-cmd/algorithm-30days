## 第 15 章：短视也能成功 - 贪心算法

### 15.1 什么是贪心算法？为什么有效？

#### 生活中的例子

**场景 1：找零钱**

```
问题：收银员要找给你 67 元

怎么找最少的纸币？

贪心方法：
每次都选不超过剩余金额的最大面值

第 1 步：最大能拿 50 元 → 剩余 17 元
第 2 步：最大能拿 10 元 → 剩余 7 元
第 3 步：最大能拿 5 元  → 剩余 2 元
第 4 步：最大能拿 2 元  → 剩余 0 元

结果：50+10+5+2 = 67 元
用了 4 张纸币 ✓

这就是贪心算法！
每次只看眼前最优！
```

**场景 2：背包问题（简单版）**

```
你有一个背包，最多装 10kg

地上有一堆物品，每个物品有价值：

物品 1: 3kg, 价值 10 元
物品 2: 4kg, 价值 12 元
物品 3: 5kg, 价值 20 元
物品 4: 2kg, 价值 8 元

问：怎么装价值最大？

贪心方法 1（按价值）：
先装价值最高的
→ 物品 3(20 元，5kg)
→ 物品 1(10 元，3kg)
→ 物品 2(12 元，4kg) → 装不下了！
总价值：20+10 = 30 元

贪心方法 2（按性价比）：
先装性价比最高的
物品 1: 10/3 ≈ 3.33 元/kg
物品 2: 12/4 = 3 元/kg
物品 3: 20/5 = 4 元/kg ← 最高
物品 4: 8/2 = 4 元/kg ← 最高

先装物品 3 和 4（共 7kg，28 元）
还能装物品 1（共 10kg）
总价值：20+8+10 = 38 元 ✓

不同的贪心策略，结果不同！
```

**说人话定义：**

```
贪心算法 = 短视的选择

核心思想：
每一步都选择当前看起来最优的
不考虑后果
不回头修改

希望：局部最优能导致全局最优

优点：快！通常比动态规划简单
缺点：不一定能得到最优解
```

### 15.2 活动选择问题 - 经典贪心

#### 问题描述

```
你要安排一天的会议

有 n 个活动，每个活动有开始时间和结束时间：

活动 1: [1, 4]   (1 点开始，4 点结束)
活动 2: [3, 5]
活动 3: [0, 6]
活动 4: [5, 7]
活动 5: [3, 9]
活动 6: [5, 9]
活动 7: [6, 10]
活动 8: [8, 11]
活动 9: [8, 12]
活动 10: [2, 14]
活动 11: [12, 16]

问：最多能参加多少个活动？
（同一时间只能参加一个活动）
```

#### 贪心策略

```
策略 1：选最早开始的？
→ 活动 3(0 点开始)，但它占了很长时间
→ 不好！❌

策略 2：选时间最短的？
→ 不一定最优
→ 不好！❌

策略 3：选最早结束的？✓
→ 这样能给后面留下更多时间
→ 这个策略是对的！✓
```

#### 代码实现

```javascript
/**
 * 活动选择问题 - 贪心算法
 * @param {number[][]} activities - 活动数组，每个元素是 [start, end]
 * @returns {number[]} - 选中的活动索引
 */
function activitySelector(activities) {
    const n = activities.length;
    
    // 按结束时间排序
    activities.sort((a, b) => a[1] - b[1]);
    
    // 选中第一个活动（结束最早的）
    const selected = [0];
    let lastFinishTime = activities[0][1];
    
    // 依次选择后面的活动
    for (let i = 1; i < n; i++) {
        // 如果这个活动的开始时间 >= 上一个活动的结束时间
        if (activities[i][0] >= lastFinishTime) {
            // 就选它！
            selected.push(i);
            lastFinishTime = activities[i][1];
        }
    }
    
    return selected;
}

// 测试
const activities = [
    [1, 4], [3, 5], [0, 6], [5, 7], [3, 9],
    [5, 9], [6, 10], [8, 11], [8, 12], [2, 14], [12, 16]
];

const result = activitySelector(activities);
console.log('选中的活动:', result);
// 输出：[0, 3, 7, 10] （可能因排序略有不同）
// 对应：[1,4], [5,7], [8,11], [12,16]
// 总共 4 个活动 ✓
```

### 15.3 Huffman 编码 - 数据压缩

#### 问题背景

```
计算机存储字符通常用固定长度：
ASCII: 每个字符 8 位（1 字节）

但有些字符出现频率高（如英文的 e, t, a）
有些字符出现频率低（如 z, q, x）

能不能：
高频字符用短编码
低频字符用长编码
这样总长度更短！

这就是 Huffman 编码的思想！
```

#### 例子

```
有一段文本："aaabbc"

固定长度编码：
每个字符 2 位（因为有 4 种字符）
a: 00
b: 01
c: 10

总长度：6 个字符 × 2 位 = 12 位

变长编码（Huffman）：
a(出现 3 次): 0      (1 位)
b(出现 2 次): 10     (2 位)
c(出现 1 次): 11     (2 位)

总长度：3×1 + 2×2 + 1×2 = 9 位

节省了：12 - 9 = 3 位！
压缩率：25% ✓
```

#### Huffman 树的构建

```
字符频率：
a: 3
b: 2
c: 1
d: 2
e: 5

构建步骤：

第 1 步：每个字符作为一个节点
   a(3)  b(2)  c(1)  d(2)  e(5)

第 2 步：选频率最小的两个合并
   选 c(1) 和 b(2)
   合并成节点 (3)
   
       (3)
      /   \
    c(1) b(2)

第 3 步：继续选最小的两个
   现在有：a(3), d(2), (3), e(5)
   选 d(2) 和 a(3)
   合并成 (5)
   
       (5)
      /   \
    d(2)  a(3)

第 4 步：继续
   现在有：(3), (5), e(5)
   选 (3) 和 (5)
   合并成 (8)
   
         (8)
        /   \
     (3)    (5)
    /   \   /   \
  c(1) b(2) d(2) a(3)

第 5 步：最后
   剩下 (8) 和 e(5)
   合并成 (13)
   
           (13)
          /    \
       (8)      e(5)
      /   \
   (3)    (5)
  /   \   /   \
c(1) b(2) d(2) a(3)

完成！这就是 Huffman 树！
```

#### 生成编码

```
从根到叶子，左边走 0，右边走 1：

路径：
c: 左左 → 00
b: 左右 → 01
d: 右左 → 10
a: 右右 → 11
e: 右 → 1

等等！有问题！
这样会混淆！

正确的做法：
必须保证没有一个编码是另一个的前缀！

重新构建（正确方法）：
每次选最小的两个，作为新节点的左右孩子

最终编码：
a: 11
b: 101
c: 100
d: 01
e: 0

验证：
没有一个是另一个的前缀 ✓
```

#### 代码实现

```javascript
/**
 * Huffman 编码树节点
 */
class HuffmanNode {
    constructor(char, freq) {
        this.char = char;      // 字符
        this.freq = freq;      // 频率
        this.left = null;      // 左孩子
        this.right = null;     // 右孩子
    }
}

/**
 * 构建 Huffman 树
 */
function buildHuffmanTree(chars, freqs) {
    // 创建优先队列（最小堆）
    const nodes = [];
    for (let i = 0; i < chars.length; i++) {
        nodes.push(new HuffmanNode(chars[i], freqs[i]));
    }
    
    // 按频率排序
    nodes.sort((a, b) => b.freq - a.freq);
    
    // 不断合并
    while (nodes.length > 1) {
        // 取出频率最小的两个
        const left = nodes.pop();
        const right = nodes.pop();
        
        // 创建父节点
        const parent = new HuffmanNode(null, left.freq + right.freq);
        parent.left = left;
        parent.right = right;
        
        // 放回队列
        nodes.push(parent);
        nodes.sort((a, b) => b.freq - a.freq);
    }
    
    return nodes[0];  // 返回根节点
}

/**
 * 生成 Huffman 编码
 */
function generateCodes(root) {
    const codes = {};
    
    function traverse(node, code) {
        if (node === null) return;
        
        // 如果是叶子节点
        if (node.char !== null) {
            codes[node.char] = code;
            return;
        }
        
        // 递归左右子树
        traverse(node.left, code + '0');
        traverse(node.right, code + '1');
    }
    
    traverse(root, '');
    return codes;
}

// 测试
const chars = ['a', 'b', 'c', 'd', 'e'];
const freqs = [3, 2, 1, 2, 5];

const root = buildHuffmanTree(chars, freqs);
const codes = generateCodes(root);

console.log('Huffman 编码:', codes);
// 输出类似：{ a: '11', b: '101', c: '100', d: '01', e: '0' }
```

### 15.4 贪心算法的正确性证明

#### 什么时候贪心是对的？

```
贪心算法不是永远正确的！

需要满足两个性质：

1. 贪心选择性质
   → 局部最优能导致全局最优
   
2. 最优子结构
   → 问题的最优解包含子问题的最优解
```

#### 如何证明？

```
证明贪心算法正确的常用方法：

1. 替换法
   假设存在更优解
   然后通过替换证明贪心解也一样优

2. 归纳法
   证明前 k 步贪心是对的
   然后推导第 k+1 步也对

3. 反证法
   假设贪心不对
   推出矛盾
```

### 15.5 贪心 vs 动态规划

#### 对比表

| 特性 | 贪心算法 | 动态规划 |
|------|---------|---------|
| 选择方式 | 当前最优 | 考虑所有选择 |
| 求解方向 | 自顶向下 | 自底向上 |
| 是否回溯 | 不回溯 | 可能需要 |
| 时间复杂度 | 通常较快 | 较慢 |
| 适用范围 | 有特殊性质 | 通用 |
| 正确性 | 需证明 | 自动保证 |

#### 如何选择

```
问题能用贪心吗？

先试试贪心：
→ 如果直觉上"显然对"，可能是贪心
→ 如果能举出反例，就不能用贪心
→ 如果证明不了正确性，就用动态规划

安全做法：
→ 先用动态规划（保证正确）
→ 如果发现可以优化，再用贪心
```

### 15.6 经典例题

#### 例题 1：分发饼干

```
问题：
有 g 个孩子，每个孩子有胃口值 g[i]
有 s 块饼干，每块饼干大小 s[j]

只有 s[j] >= g[i] 时，饼干 j 才能满足孩子 i

问：最多能满足几个孩子？

贪心策略：
用小饼干满足小胃口的孩子

代码：
function findContentChildren(g, s) {
    g.sort((a, b) => a - b);  // 孩子胃口从小到大
    s.sort((a, b) => a - b);  // 饼干大小从小到大
    
    let child = 0;
    let cookie = 0;
    
    while (child < g.length && cookie < s.length) {
        if (s[cookie] >= g[child]) {
            // 这块饼干能满足这个孩子
            child++;
        }
        cookie++;
    }
    
    return child;
}
```

#### 例题 2：跳跃游戏

```
问题：
数组每个元素表示你能跳的最大长度

问：能不能跳到最后一个位置？

贪心策略：
维护一个"最远能跳到的位置"

代码：
function canJump(nums) {
    let maxReach = 0;
    
    for (let i = 0; i < nums.length; i++) {
        if (i > maxReach) return false;  // 到不了这里
        
        maxReach = Math.max(maxReach, i + nums[i]);
        
        if (maxReach >= nums.length - 1) {
            return true;  // 已经能跳到最后了
        }
    }
    
    return true;
}
```
