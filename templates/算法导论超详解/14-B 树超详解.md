### 17.1 什么是 B 树？为什么这么重要？

#### 生活中的例子

**场景 1：图书馆索引**

```
大型图书馆有 100 万本书

怎么快速找到想要的书？

笨方法：
一本一本找 → O(n) 累死！

聪明方法：
按索书号分类：
- A 类：哲学（1 楼）

每层楼再细分：
- C 类：
  - C1: 数学
  - C2: 物理
  - C3: 化学

这就是 B 树的思想！
多级索引，快速定位！
```

**场景 2：字典查字**

```
新华字典有 1 万字

怎么快速找到"我"字？

1. 先看部首：扌部

```

**说人话定义：**

```
B 树 = 多叉平衡搜索树

特点：
✓ 一个节点可以有多个孩子（多叉）
✓ 所有叶子在同一层（平衡）
✓ 每个节点存多个关键字（高效）

应用：
✓ 数据库索引（MySQL、Oracle）
✓ 文件系统（NTFS、HFS+）
✓ 磁盘存储系统

为什么用 B 树？
→ 减少磁盘 I/O 次数！
→ 磁盘读写很慢，要尽量减少访问次数！
```

### 17.2 B 树的定义和性质

#### 形式化定义

```
一棵 m 阶 B 树满足以下性质：

性质 1：根节点至少有 2 个孩子（除非是叶子）

性质 2：每个非根内部节点至少有 ⌈m/2⌉ 个孩子

性质 3：每个节点最多有 m 个孩子

性质 4：所有叶子节点都在同一层

性质 5：如果一个节点有 k 个孩子，则有 k-1 个关键字

例如：3 阶 B 树（m=3）
- 每个节点最多 3 个孩子
- 非根节点至少 2 个孩子（⌈3/2⌉=2）
- 每个节点最多 2 个关键字
```

#### 图示说明

```
一棵 3 阶 B 树：

         [8, 12]
        /   |   \
    [2,5] [9,10] [14,16]
    
验证性质：
✓ 根节点有 3 个孩子
✓ 每个内部节点有 3 个孩子（≥⌈3/2⌉=2）
✓ 所有叶子在同一层
✓ 节点关键字数 = 孩子数 - 1

这是一棵合法的 3 阶 B 树！✓
```

### 17.3 B 树的基本操作

#### 查找操作

```javascript
/**
 * B 树节点
 */
class BTreeNode {
    constructor(isLeaf = false) {
        this.isLeaf = isLeaf;
        this.keys = [];      // 关键字数组
        this.children = [];  // 孩子指针数组
    }
}

/**
 * B 树
 */
class BTree {
    constructor(t) {
        this.t = t;  // 最小度数
        this.root = new BTreeNode(true);
    }
    
    /**
     * 查找关键字 k
     * @param {BTreeNode} node - 当前节点
     * @param {number} k - 要查找的关键字
     * @returns {object|null} - 找到返回节点和位置，否则返回 null
     */
    search(node, k) {
        let i = 0;
        
        // 找到第一个大于等于 k 的位置
        while (i < node.keys.length && k > node.keys[i]) {
            i++;
        }
        
        // 如果找到了
        if (i < node.keys.length && k === node.keys[i]) {
            return { node, index: i };
        }
        
        // 如果是叶子，没找到
        if (node.isLeaf) {
            return null;
        }
        
        // 否则在孩子节点中查找
        return this.search(node.children[i], k);
    }
}

// 测试
const tree = new BTree(3);  // 3 阶 B 树
// ... 插入一些元素
const result = tree.search(tree.root, 10);
console.log(result);  // 找到或 null
```

#### 查找详细过程

```
B 树结构：
         [8, 12]
        /   |   \
    [2,5] [9,10] [14,16]

查找关键字：10

═══════════════════════════════════
第 1 步：在根节点查找
═══════════════════════════════════
根节点：[8, 12]
比较：10 > 8 ✓
     10 < 12 ✓
所以在中间的孩子中查找

═══════════════════════════════════
第 2 步：在中间子树查找
═══════════════════════════════════
当前节点：[9, 10]
比较：10 == 10 ✓

找到了！返回 { node: [9,10], index: 1 }
```

### 17.4 插入操作 - 核心难点

#### 基本思想

```
插入的关键：
1. 新元素总是插入到叶子节点
2. 如果叶子节点满了，就要分裂
3. 分裂可能向上传播，甚至分裂根节点

满的定义：
对于 m 阶 B 树，节点最多有 m-1 个关键字
超过就满了！
```

#### 分裂操作

```javascript
class BTree {
    /**
     * 分裂节点
     * @param {BTreeNode} parent - 父节点
     * @param {number} index - 孩子在父节点中的位置
     * @param {BTreeNode} fullNode - 要分裂的满节点
     */
    splitChild(parent, index, fullNode) {
        const t = this.t;
        
        // 创建新节点（右半部分）
        const newNode = new BTreeNode(fullNode.isLeaf);
        
        // 中间关键字
        const midKey = fullNode.keys[t - 1];
        
        // 把后半部分移到新节点
        newNode.keys = fullNode.keys.splice(t);
        if (!fullNode.isLeaf) {
            newNode.children = fullNode.children.splice(t);
        }
        
        // 在父节点中插入中间关键字和新节点
        parent.keys.splice(index, 0, midKey);
        parent.children.splice(index + 1, 0, newNode);
    }
    
    /**
     * 插入关键字
     */
    insert(key) {
        const root = this.root;
        
        // 如果根节点满了，需要分裂
        if (root.keys.length === 2 * this.t - 1) {
            const newRoot = new BTreeNode(false);
            newRoot.children[0] = root;
            
            // 分裂根节点
            this.splitChild(newRoot, 0, root);
            
            // 新根成为新的根
            this.root = newRoot;
            
            // 在非满的根节点中插入
            this.insertNonFull(newRoot, key);
        } else {
            this.insertNonFull(root, key);
        }
    }
    
    /**
     * 在非满节点中插入
     */
    insertNonFull(node, key) {
        let i = node.keys.length - 1;
        
        if (node.isLeaf) {
            // 叶子节点，直接插入
            node.keys.push(key);
            node.keys.sort((a, b) => a - b);
        } else {
            // 找到插入位置
            while (i >= 0 && key < node.keys[i]) {
                i--;
            }
            i++;
            
            // 如果孩子满了，先分裂
            if (node.children[i].keys.length === 2 * this.t - 1) {
                this.splitChild(node, i, node.children[i]);
                
                // 确定插入哪个孩子
                if (key > node.keys[i]) {
                    i++;
                }
            }
            
            this.insertNonFull(node.children[i], key);
        }
    }
}
```

### 17.5 删除操作（简单了解）

```
删除比插入更复杂！

三种情况：

1. 删除叶子节点的键
   → 直接删除
   
2. 删除内部节点的键
   → 找前驱或后继替换
   
3. 删除后节点关键字太少
   → 向兄弟借
   → 或者与兄弟合并

由于非常复杂，面试很少考
知道思想即可！
```

### 17.6 B 树的高度分析

```
定理：
n 个关键字的 m 阶 B 树，高度 h ≤ log_(m/2)(n)

证明思路：
- 根至少 2 个孩子
- 其他节点至少 m/2 个孩子
- 第 h 层至少有 2×(m/2)^(h-1) 个节点
- 推出 h ≤ log_(m/2)(n)

意义：
B 树的高度很低！
例如：m=100, n=100 万
h ≤ log_50(10^6) ≈ 4

只需要 4 次磁盘 I/O！
这就是为什么 B 树适合磁盘存储！
```

### 17.7 实际应用

#### 应用 1：数据库索引

```sql
-- MySQL 的 InnoDB 引擎使用 B+ 树（B 树的变种）

CREATE TABLE users (
    id INT PRIMARY KEY,  -- 主键索引（B+ 树）
    name VARCHAR(100),
    email VARCHAR(100),
    INDEX idx_name (name)  -- 二级索引（B+ 树）
);

-- 查询：SELECT * FROM users WHERE name = '张三';
-- 通过 B+ 树索引，O(log n) 时间找到！
```

#### 应用 2：文件系统

```
NTFS（Windows 文件系统）：
- 用 B 树管理文件簇
- 快速定位文件数据

HFS+（macOS 文件系统）：
- 也用 B 树变体
- 高效管理大量文件
```

#### 应用 3：大数据存储

```
HBase、Cassandra 等 NoSQL 数据库：
- 用 LSM 树（B 树的变种）
- 支持海量数据的快速读写
```

### 17.8 B 树 vs 二叉搜索树

| 特性 | BST | AVL 树 | 红黑树 | B 树 |
|------|-----|--------|--------|------|
| 分支因子 | 2 | 2 | 2 | m（可达数百） |
| 高度 | O(n) | O(log n) | O(log n) | O(log_m n) |
| 磁盘 I/O | 多 | 多 | 多 | 少 ✓ |
| 适用场景 | 内存 | 内存 | 内存 | 磁盘 ✓ |
