# 📚 算法导论 Part III - 散列表超详解

## 第 11 章：散列表 - 世界上最快的查找方法

### 11.1 什么是散列表？为什么这么快？

#### 生活中的例子

**场景 1：储物柜**

```
学校有 1000 个储物柜，怎么快速找到你的柜子？

笨方法：
一个一个看号码 → O(n) 太慢！

聪明方法：
你的号码是 327 → 直接走到 327 号柜子 → O(1) 超快！

这就是散列的思想！
```

**场景 2：字典查单词**

```
查"apple"这个单词：

笨方法：
从第一页开始翻 → O(n) 累死

聪明方法：
根据首字母 a，直接翻到前面 → O(1) 秒找！

这也是散列！
```

**说人话定义：**

```
散列表 = 超级快的查找工具

原理：
把钥匙（key）通过一个函数转换
变成一个数组索引
然后直接访问那个位置

时间复杂度：O(1)
就是这么快！🚀
```

### 11.2 散列函数 - 核心中的核心

#### 什么是散列函数？

```
散列函数就像一个转换器：

输入：任意数据（字符串、数字、对象）
输出：一个数组索引

例如：
"apple" → 计算 → 5
"banana" → 计算 → 12
"orange" → 计算 → 8

这样就能存到数组的对应位置！
```

#### 简单的散列函数例子

```javascript
/**
 * 最简单的散列函数 - 除留余数法
 */
function simpleHash(key, tableSize) {
    // 如果是数字，直接取余数
    if (typeof key === 'number') {
        return key % tableSize;
    }
    
    // 如果是字符串，把所有字符的 ASCII 码加起来
    let sum = 0;
    for (let char of key) {
        sum += char.charCodeAt(0);
    }
    
    return sum % tableSize;
}

// 测试
const tableSize = 10;
console.log(simpleHash("apple", tableSize));   // 输出某个 0-9 的数字
console.log(simpleHash(123, tableSize));       // 123 % 10 = 3
```

#### 好的散列函数的特点

```
✓ 均匀分布
  不同的 key 应该分散到不同的位置
  
✓ 计算快速
  不能花太多时间计算
  
✓ 确定性
  同样的 key 永远得到同样的结果
  
✓ 少冲突
  尽量避免两个 key 得到相同的索引
```

### 11.3 冲突解决 - 不可避免的麻烦

#### 什么是冲突？

```
问题：
不同的 key，计算出相同的索引！

例如：
hash("apple") = 5
hash("pear") = 5

怎么办？两个都要存在位置 5！

这就是冲突！
```

#### 解决方法 1：链接法（拉链法）⭐推荐

```
思想：
每个数组位置是一个链表
冲突的元素都挂在这个链表上

图示：
索引：0   1   2   3   4   5      6   7
      ┌───┬───┬───┬───┬───┬──────┬───┬───┐
      │   │   │   │   │   │→app  │   │   │
      └───┴───┴───┴───┴───┤→pear │───┴───┘
                          └──────┘
                          
位置 5 有两个元素，用链表连起来
```

**代码实现：**

```javascript
/**
 * 散列表 - 链接法解决冲突
 */
class HashTableChaining {
    constructor(size = 16) {
        this.size = size;
        // 创建一个数组，每个位置是一个空数组（当作链表）
        this.buckets = new Array(size).fill(null).map(() => []);
        this.count = 0;
    }
    
    // 简单的散列函数
    _hash(key) {
        let hash = 0;
        const strKey = String(key);
        for (let i = 0; i < strKey.length; i++) {
            hash = ((hash << 5) - hash) + strKey.charCodeAt(i);
            hash = hash & hash;  // 转成 32 位整数
        }
        return Math.abs(hash) % this.size;
    }
    
    // 插入键值对
    set(key, value) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        // 先检查是否已存在
        for (let pair of bucket) {
            if (pair.key === key) {
                pair.value = value;  // 更新
                return;
            }
        }
        
        // 不存在就添加
        bucket.push({ key, value });
        this.count++;
        
        // 如果太满了，扩容
        if (this.count / this.size > 0.75) {
            this.resize();
        }
    }
    
    // 查找
    get(key) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let pair of bucket) {
            if (pair.key === key) {
                return pair.value;
            }
        }
        
        return undefined;  // 没找到
    }
    
    // 删除
    delete(key) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let i = 0; i < bucket.length; i++) {
            if (bucket[i].key === key) {
                bucket.splice(i, 1);
                this.count--;
                return true;
            }
        }
        
        return false;  // 没找到
    }
    
    // 扩容
    resize() {
        const oldBuckets = this.buckets;
        this.size *= 2;
        this.buckets = new Array(this.size).fill(null).map(() => []);
        this.count = 0;
        
        // 重新散列所有元素
        for (let bucket of oldBuckets) {
            for (let pair of bucket) {
                this.set(pair.key, pair.value);
            }
        }
    }
    
    // 获取大小
    size() {
        return this.count;
    }
}

// 测试
const hashTable = new HashTableChaining();
hashTable.set("apple", "苹果");
hashTable.set("pear", "梨");
hashTable.set("banana", "香蕉");

console.log(hashTable.get("apple"));  // "苹果" ✓
console.log(hashTable.get("pear"));   // "梨" ✓
```

#### 详细执行过程

```
初始：
buckets = [[], [], [], [], [], [], [], [], ...]
          ↑
       都是空的

═══════════════════════════════════
插入 ("apple", "苹果")
═══════════════════════════════════
hash("apple") = 5
存入 buckets[5]

buckets[5] = [{key: "apple", value: "苹果"}]

═══════════════════════════════════
插入 ("pear", "梨")
═══════════════════════════════════
hash("pear") = 5 （冲突了！）

把 "pear" 也放到 buckets[5]

buckets[5] = [
  {key: "apple", value: "苹果"},
  {key: "pear", value: "梨"}
]

这就是链接法！
冲突了就加到链表里！

═══════════════════════════════════
插入 ("banana", "香蕉")
═══════════════════════════════════
hash("banana") = 8
存入 buckets[8]

buckets[8] = [{key: "banana", value: "香蕉"}]

最终状态：
索引 5: [("apple","苹果"), ("pear","梨")]
索引 8: [("banana","香蕉")]
其他：[]
```

#### 解决方法 2：开放寻址法

```
思想：
冲突了就找下一个空位！

三种常见方式：
1. 线性探测：+1, +2, +3...
2. 二次探测：+1², -1², +2², -2²...
3. 双重散列：用另一个散列函数计算步长
```

**线性探测示例：**

```javascript
class HashTableOpenAddressing {
    constructor(size = 16) {
        this.size = size;
        this.keys = new Array(size);
        this.values = new Array(size);
        this.count = 0;
    }
    
    _hash(key) {
        let hash = 0;
        const strKey = String(key);
        for (let i = 0; i < strKey.length; i++) {
            hash = ((hash << 5) - hash) + strKey.charCodeAt(i);
            hash = hash & hash;
        }
        return Math.abs(hash) % this.size;
    }
    
    set(key, value) {
        let index = this._hash(key);
        
        // 线性探测：找空位
        while (this.keys[index] !== undefined) {
            if (this.keys[index] === key) {
                // 已存在，更新
                this.values[index] = value;
                return;
            }
            
            // 冲突了，找下一个位置
            index = (index + 1) % this.size;
        }
        
        // 找到空位，存入
        this.keys[index] = key;
        this.values[index] = value;
        this.count++;
    }
    
    get(key) {
        let index = this._hash(key);
        
        while (this.keys[index] !== undefined) {
            if (this.keys[index] === key) {
                return this.values[index];
            }
            index = (index + 1) % this.size;
        }
        
        return undefined;  // 没找到
    }
}

// 执行过程：
// hash("apple") = 5 → 存位置 5
// hash("pear") = 5 → 冲突！
//                  → 找位置 6 → 空的，存这里
// hash("banana") = 8 → 存位置 8
```

### 11.4 性能分析（为什么是 O(1)？）

#### 理想情况

```
没有冲突时：
- 计算哈希：很快
- 访问数组：瞬间
- 总计：O(1)

完美！
```

#### 最坏情况

```
所有元素都冲突（都在同一个链表里）：
- 要遍历整个链表
- O(n) ❌

什么时候发生？
→ 散列函数设计得很烂
→ 或者故意攻击（所有 key 都冲突）
```

#### 平均情况

```
好的散列函数 + 合适的负载因子：
- 期望链长 = n/m（元素数/桶数）
- 如果 m ≈ n，链长 ≈ 1
- 平均查找时间 = O(1) ✓

实际应用中：
非常快！比任何查找算法都快！
```

```javascript
/**
 * LRU 缓存 = 散列表 + 双向链表
 */
class LRUCache {
    constructor(capacity) {
        this.capacity = capacity;
        this.map = new Map();
    }
    
    get(key) {
        if (!this.map.has(key)) return -1;
        
        const value = this.map.get(key);
        // 移到最新使用的位置
        this.map.delete(key);
        this.map.set(key, value);
        
        return value;
    }
    
    put(key, value) {
        if (this.map.has(key)) {
            this.map.delete(key);
        } else if (this.map.size >= this.capacity) {
            // 删除最旧的
            const firstKey = this.map.keys().next().value;
            this.map.delete(firstKey);
        }
        
        this.map.set(key, value);
    }
}
```

### 11.6 完整代码（背诵版）

```javascript
/**
 * 散列表 - 链接法实现（推荐）
 */
class HashTable {
    constructor(size = 16) {
        this.size = size;
        this.buckets = new Array(size).fill(null).map(() => []);
        this.count = 0;
    }
    
    _hash(key) {
        let hash = 0;
        const strKey = String(key);
        for (let i = 0; i < strKey.length; i++) {
            hash = ((hash << 5) - hash) + strKey.charCodeAt(i);
            hash = hash & hash;
        }
        return Math.abs(hash) % this.size;
    }
    
    set(key, value) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let pair of bucket) {
            if (pair.key === key) {
                pair.value = value;
                return;
            }
        }
        
        bucket.push({ key, value });
        this.count++;
        
        if (this.count / this.size > 0.75) {
            this.resize();
        }
    }
    
    get(key) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let pair of bucket) {
            if (pair.key === key) {
                return pair.value;
            }
        }
        
        return undefined;
    }
    
    delete(key) {
        const index = this._hash(key);
        const bucket = this.buckets[index];
        
        for (let i = 0; i < bucket.length; i++) {
            if (bucket[i].key === key) {
                bucket.splice(i, 1);
                this.count--;
                return true;
            }
        }
        
        return false;
    }
    
    resize() {
        const oldBuckets = this.buckets;
        this.size *= 2;
        this.buckets = new Array(this.size).fill(null).map(() => []);
        this.count = 0;
        
        for (let bucket of oldBuckets) {
            for (let pair of bucket) {
                this.set(pair.key, pair.value);
            }
        }
    }
    
    has(key) {
        return this.get(key) !== undefined;
    }
    
    size() {
        return this.count;
    }
}

hashTable.set("name", "张三");
hashTable.set("age", 18);
hashTable.set("city", "北京");

console.log(hashTable.has("age"));   // true ✓
console.log('大小:', hashTable.size());  // 3 ✓
```
