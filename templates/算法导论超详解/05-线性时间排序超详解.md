### 8.1 为什么能比 O(n log n) 更快？

#### 之前的排序算法

```
插入排序：O(n²)
堆排序：O(n log n)
快速排序：平均 O(n log n)
归并排序：O(n log n)

问题：能不能更快？比如 O(n)？
```

#### 关键发现

```
之前的算法都有一个共同点：
→ 需要比较两个元素的大小
→ 通过比较来决定顺序

数学证明：
基于比较的排序，最快也要 O(n log n)

但是！
如果不比较呢？
→ 可以突破这个限制！
→ 可以达到 O(n)！

这就是线性时间排序！
```

### 8.2 计数排序（Counting Sort）- 统计票数

#### 生活中的例子

**场景：班级投票选班长**

```
问题：统计每个候选人的得票数

候选人：A, B, C
30 个同学投票

怎么做？

笨方法：
一张一张看，然后数 A 有几票，B 有几票...

聪明方法：
准备 3 个盒子，分别标上 A, B, C
拿到 A 的票 → 放 A 盒
拿到 B 的票 → 放 B 盒
拿到 C 的票 → 放 C 盒
最后数每个盒子有几张票

搞定！这就是计数排序的思想！
```

#### 说人话定义

```
计数排序的核心思想：

1. 统计每个值出现的次数
2. 根据统计结果，直接把元素放到正确位置

不需要比较谁大谁小
只需要知道"有多少个比我小"

就这么简单！
```

#### 代码实现

```javascript
/**
 * 计数排序
 * @param {number[]} arr - 待排序数组
 * @returns {number[]} - 排序后的数组
 */
function countingSort(arr) {
    if (arr.length === 0) return arr;
    
    // 找到最大值和最小值
    let max = arr[0];
    let min = arr[0];
    for (let num of arr) {
        max = Math.max(max, num);
        min = Math.min(min, num);
    }
    
    // 创建计数数组
    const range = max - min + 1;
    const count = new Array(range).fill(0);
    
    // 统计每个数出现的次数
    for (let num of arr) {
        count[num - min]++;
    }
    
    // 重建数组
    let index = 0;
    for (let i = 0; i < range; i++) {
        while (count[i] > 0) {
            arr[index++] = i + min;
            count[i]--;
        }
    }
    
    return arr;
}

// 测试
const arr = [4, 2, 2, 8, 3, 3, 1];
countingSort(arr);
console.log(arr);
// 输出：[1, 2, 2, 3, 3, 4, 8] ✓
```

#### 详细执行过程（动画版）

```

═══════════════════════════════════

═══════════════════════════════════

遇到 4 → count[4-1]++ → count[3] = 1
遇到 2 → count[2-1]++ → count[1] = 1
遇到 2 → count[1]++     → count[1] = 2
遇到 8 → count[8-1]++   → count[7] = 1
遇到 3 → count[3-1]++   → count[2] = 1
遇到 3 → count[2]++     → count[2] = 2

最终计数数组：
```

#### 时间复杂度分析

```
找最大最小：O(n)
统计次数：O(n)
重建数组：O(n + k)，k 是范围大小

总时间：O(n + k)

如果 k ≈ n（数字范围不大）
那么时间复杂度 = O(n) ✓

空间复杂度：O(k)（需要计数数组）
```

#### 适用场景

```
✓ 整数排序
✓ 数字范围已知且不大
✓ 有重复元素
✓ 要求稳定排序

✗ 浮点数排序
✗ 数字范围很大（如排电话号码）
✗ 字符串排序
```

### 8.3 基数排序（Radix Sort）- 扑克牌分类

#### 生活中的例子

**场景：整理一副乱序的扑克牌**

```
问题：把扑克牌按从小到大排好

怎么做？

聪明方法：
1. 先按花色分成 4 堆：♠ ♥ ♦ ♣
2. 每堆内部再按数字排序

或者：
1. 先按数字分成 13 堆：A, 2, 3, ..., K
2. 每堆内部再按花色排序

这就是基数排序的思想！
→ 分多次排序
→ 每次按一个关键字
```

#### 说人话定义

```
基数排序的核心思想：

1. 从最低位（个位）开始排序
2. 然后十位、百位...
3. 最高位排完，整体就有序了

每次排序用稳定的排序算法（通常用计数排序）

就像整理文件：
先按年份分 → 再按月份分 → 再按日期分
最后自然有序！
```

#### 代码实现

```javascript
/**
 * 基数排序
 * @param {number[]} arr - 待排序数组
 * @returns {number[]} - 排序后的数组
 */
function radixSort(arr) {
    if (arr.length === 0) return arr;
    
    // 找到最大值，确定有多少位
    let max = arr[0];
    for (let num of arr) {
        max = Math.max(max, num);
    }
    
    // 从个位开始，逐位排序
    let exp = 1;  // 1 表示个位，10 表示十位，100 表示百位...
    while (Math.floor(max / exp) > 0) {
        countingSortByDigit(arr, exp);
        exp *= 10;
    }
    
    return arr;
}

/**
 * 按指定位数进行计数排序
 */
function countingSortByDigit(arr, exp) {
    const n = arr.length;
    const output = new Array(n);
    const count = new Array(10).fill(0);  // 0-9
    
    // 统计当前位的数字出现次数
    for (let i = 0; i < n; i++) {
        const digit = Math.floor(arr[i] / exp) % 10;
        count[digit]++;
    }
    
    // 计算累积次数（为了稳定性）
    for (let i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }
    
    // 从后往前遍历，保证稳定性
    for (let i = n - 1; i >= 0; i--) {
        const digit = Math.floor(arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }
    
    // 复制回原数组
    for (let i = 0; i < n; i++) {
        arr[i] = output[i];
    }
}

// 测试
const arr = [170, 45, 75, 90, 802, 24, 2, 66];
radixSort(arr);
console.log(arr);
// 输出：[2, 24, 45, 66, 75, 90, 170, 802] ✓
```

### 8.4 桶排序（Bucket Sort）- 分班级考试

#### 生活中的例子

**场景：学校期末考试排名**

```
问题：给全校学生按成绩排名

怎么做？

聪明方法：
1. 按年级分成几个桶：
   - 一年级桶：0-100 分
   - 二年级桶：0-100 分
   - 三年级桶：0-100 分
   
2. 每个桶内部单独排序
   - 可以用任何排序算法
   
3. 把所有桶的结果连起来

这就是桶排序！
```

#### 说人话定义

```
桶排序的核心思想：

1. 把数据分到不同的桶里
2. 每个桶内部单独排序
3. 把所有桶的结果合并

关键点：
- 桶要设计得合理
- 数据要均匀分布
- 这样每个桶的元素都很少
```

#### 代码实现

```javascript
/**
 * 桶排序
 * @param {number[]} arr - 待排序数组
 * @param {number} bucketSize - 每个桶的大小
 * @returns {number[]} - 排序后的数组
 */
function bucketSort(arr, bucketSize = 5) {
    if (arr.length === 0) return arr;
    
    // 找到最大值和最小值
    let max = arr[0];
    let min = arr[0];
    for (let num of arr) {
        max = Math.max(max, num);
        min = Math.min(min, num);
    }
    
    // 计算需要多少个桶
    const bucketCount = Math.floor((max - min) / bucketSize) + 1;
    const buckets = new Array(bucketCount).fill(null).map(() => []);
    
    // 把元素分配到各个桶
    for (let num of arr) {
        const bucketIndex = Math.floor((num - min) / bucketSize);
        buckets[bucketIndex].push(num);
    }
    
    // 对每个桶内部排序
    const result = [];
    for (let bucket of buckets) {
        if (bucket.length > 0) {
            // 桶内用插入排序或快速排序
            bucket.sort((a, b) => a - b);
            result.push(...bucket);
        }
    }
    
    return result;
}

// 测试
const arr = [0.897, 0.565, 0.656, 0.1234, 0.665, 0.3434];
bucketSort(arr, 0.1);
console.log(arr);
// 输出：[0.1234, 0.3434, 0.565, 0.656, 0.665, 0.897] ✓
```

```

所有元素都在同一个桶里
退化成普通排序：O(n²)

平均情况：
O(n + k)，k 是桶的数量

空间复杂度：O(n + k)
```

#### 适用场景

```
✓ 数据均匀分布
✓ 浮点数排序
✓ 知道数据范围
✓ 可以设计合理的桶

✗ 数据分布不均匀
✗ 不知道数据范围
✗ 某些桶特别满
```

### 8.5 三种线性排序对比

| 特性 | 计数排序 | 基数排序 | 桶排序 |
|------|---------|---------|--------|
| 时间复杂度 | O(n+k) | O(d×n) | O(n+k) |
| 空间复杂度 | O(k) | O(n) | O(n+k) |
| 稳定性 | ✓ 稳定 | ✓ 稳定 | 取决于桶内排序 |
| 适用类型 | 整数 | 整数 | 浮点数 |
| 数据范围 | 范围小 | 范围大 | 均匀分布 |
| 核心思想 | 统计次数 | 按位排序 | 分桶处理 |

### 8.6 什么时候用哪种？

#### 选择指南

```
如果是整数，范围小，有重复：
→ 用计数排序

如果是整数，范围大，位数少：
→ 用基数排序

如果是浮点数，或数据均匀分布：
→ 用桶排序

如果不确定，或数据不规则：
→ 用快速排序/归并排序（O(n log n)）
```
